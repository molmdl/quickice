## Force Field Constants (Single Source of Truth)

**TIP4P-ICE LJ Parameters (`quickice/output/gromacs_writer.py`):**
- `TIP4P_ICE_ALPHA = 0.13458335` (line 52; also redefined at line 386 — same value, second definition shadows first)
- `TIP4P_ICE_OW_SIGMA = 3.16680e-1` nm (line 56)
- `TIP4P_ICE_OW_EPSILON = 8.82110e-1` kJ/mol (line 57)
- Reference: Abascal et al. 2005, DOI: 10.1063/1.1931662
- These module-level constants replace all inline LJ parameter values in the 6 TOP-writing functions
- All 6 TOP writers reference `TIP4P_ICE_OW_SIGMA` and `TIP4P_ICE_OW_EPSILON` via f-strings with `:.5e` formatting
- Reference data file: `quickice/data/tip4p-ice.itp` (canonical OW_ice/HW_ice/MW atomtypes + virtual_sites3)

**Combination Rule:**
- `comb-rule=2` (Lorentz-Berthelot) in all 6 TOP-writing functions
- `sigma_ij = (sigma_i + sigma_j) / 2`, `epsilon_ij = sqrt(eps_i * eps_j)`
- Matches AMBER/GAFF2 convention used by GROMACS-bundled AMBER force fields
- `[ defaults ]` writer locations in `quickice/output/gromacs_writer.py`: lines 962, 1557, 1959, 2630, 3219, 3929 (one per TOP writer)
- Each writes `1  2  yes  0.5  0.8333` (nbfunc=1, comb-rule=2, gen-pairs=yes, fudgeLJ=0.5, fudgeQQ=0.8333)

**Physical Constants:**
- `AVOGADRO = 6.02214076e23` mol⁻¹ (CODATA 2017) — single definition at `quickice/structure_generation/ion_inserter.py:29`
- Imported (never duplicated) by: `quickice/cli/pipeline.py:15`, `quickice/structure_generation/solute_inserter.py:28`, `quickice/structure_generation/custom_molecule_inserter.py:28`, `quickice/ranking/scorer.py:18`
- `WATER_VOLUME_NM3 = 0.0299` — defined in `quickice/structure_generation/types.py:39` (single source of truth; derivation documented in file comments)
- `WATER_ATOMS_PER_MOLECULE = 4` — defined in `quickice/structure_generation/types.py:22`
- `MOLECULE_TYPE_INFO` dict in `quickice/structure_generation/types.py:11` (atoms/res_name per molecule type)

## Input Validation

**CLI Validators (in `quickice/validation/validators.py`):**
- `validate_temperature(value)` → float in [0, 500] K
- `validate_pressure(value)` → float in [0, 10000] MPa
- `validate_nmolecules(value)` → int in [4, 100000]
- `validate_positive_float(value)` → float > 0
- `validate_box_dimension(value)` → float >= 1.0 nm
- `validate_concentration_range(value)` → float in [0.0, 5.0] mol/L (seawater ~0.6 to saturated ~5 mol/L)
- `validate_occupancy_range(value)` → float in [0.0, 100.0] % (cage occupancy)

**File Extension Validation:**
- `--custom-gro` file must have `.gro` extension (case-insensitive) — enforced in `quickice/cli/pipeline.py`
- `--custom-itp` file must have `.itp` extension (case-insensitive) — enforced in `quickice/cli/pipeline.py`

**Cross-Flag Validation (in `quickice/cli/parser.py:validate_pipeline_args()`):**
- `--hydrate + --nmolecules` mutually exclusive
- `--custom-gro` requires `--custom-itp` and vice versa
- `--custom-gro` requires `--interface`
- `--custom-placement custom` requires `--custom-positions-file`
- `--custom-placement random` requires `--custom-count` or `--custom-concentration` (not both)
- `--solute-type` requires `--solute-concentration` and `--interface`
- `--ion-concentration` requires `--interface`

## Configuration

**Environment:**
- Conda environment from `environment.yml` (development) or `environment-build.yml` (build)
- `setup.sh` activates conda env `quickice` and sets `PYTHONPATH` to project root + adds project root to PATH
- No `.env` files required; application has no required environment variable dependencies for core CLI operation
- No runtime configuration files; all parameters via CLI args or GUI inputs
- Display environment variables checked by `quickice/entry.py:_has_display()` on Linux: `DISPLAY`, `WAYLAND_DISPLAY`, `QT_QPA_PLATFORM`
- Optional `QUICKICE_FORCE_VTK` env var (see INTEGRATIONS.md)

**Build:**
- `quickice-gui.spec` - PyInstaller spec for executable build
  - Entry point: `quickice/__main__.py` (unified, routes to CLI or GUI)
  - `console=True` + `hide_console='hide-late'` — console allocated then hidden after GUI launches
  - Data collection: `quickice/data` + `collect_all()` for `iapws`, `genice2`, `genice_core`, `matplotlib`, `scipy`, `numpy`, `shapely`, `networkx`, `spglib`
  - Excludes: `*/tests/*`, `*/test/*`, `*/docs/*`, `*/__pycache__/*`, `*/.pytest_cache/*`, `*/egg-info/*`
- `environment-build.yml` - Conda environment for build (same deps + `pyinstaller==6.19.0`)
- `scripts/build-linux.sh` - Linux build script (requires `quickice` conda env activated; banner says "v4.5")
- `scripts/assemble-dist.sh` - Packages `dist/quickice-gui/` into versioned `quickice-{version}-linux-x86_64.tar.gz`
- `.github/workflows/build-windows.yml` - Windows build via GitHub Actions (manual `workflow_dispatch` only)

## Entry Points

**Unified entry (primary):**
- `python -m quickice` → `quickice/__main__.py` → `quickice/entry.py:main()` (argv-based router)
- Routes to CLI or GUI based on `--cli`/`--gui` flags and implicit pipeline-flag detection (`_has_pipeline_flags()`)
- GUI branch: `quickice/gui/main_window.py:run_app()`
- CLI branch: `quickice/main.py:main()` → `quickice/cli/parser.py:get_arguments()` → `quickice/cli/pipeline.py:CLIPipeline.execute()`

**Legacy entry (still works):**
- `python quickice.py` at project root — delegates to `quickice.entry.main()`
- `QuickIce.sh` / `QuickIce.bat` - Shell launchers for packaged binary

**PyInstaller entry:**
- `quickice-gui` binary → `quickice/__main__.py` (same unified entry)
