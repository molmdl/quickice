## Executive Summary

QuickIce v4.5 extends the existing interface generation system with two new capabilities:

1. **Tab 4 - Solute Insertion:** Add THF or CH₄ molecules to the liquid phase of an interface structure using concentration-based insertion. This is fundamentally different from Tab 2 (hydrate generation) — Tab 4 places molecules freely in liquid water, while Tab 2 places guests in specific cage positions within a hydrate lattice.

2. **Tab 5 - Custom Molecule Upload:** Allow users to upload arbitrary molecules (via .gro + .itp files) and insert them into the liquid phase with either random or user-specified placement (COM + rotation matrix).

Both features integrate with the existing interface infrastructure (Tab 3) and follow patterns established by the ion insertion system (Tab 6). The key challenge is distinguishing **hydrate guests** (CH4_HYD in cages) from **liquid solutes** (CH4_LIQ in water) in GROMACS output.

Research draws from: GROMACS `insert-molecules` tool documentation, existing QuickIce v4.0 codebase (ion insertion, GROMACS writer), and molecular simulation best practices.

## Tab 4: Solute Insertion — Feature Analysis

### What This Tab Does

Tab 4 inserts THF or CH₄ molecules into the **liquid water phase** of an existing interface structure. This is a post-processing operation that:

1. Takes an `InterfaceStructure` from Tab 3
2. Identifies the liquid region (atoms after `ice_atom_count`)
3. Calculates how many solute molecules to insert based on concentration
4. Places molecules randomly in liquid with overlap checking
5. Outputs modified structure with distinct residue naming (THF_LIQ, CH4_LIQ)

**Key distinction from Tab 2 (Hydrate):**
- Tab 2 generates clathrate hydrates with guests in **cage positions** (crystallographically defined)
- Tab 4 places solutes **freely in liquid** (random positions, no cage constraints)

### Table Stakes — Tab 4

Features users expect. Missing = product feels incomplete.

| Feature | Why Expected | Complexity | Notes | Dependency |
|---------|--------------|------------|-------|------------|
| Concentration input (mol/L) | Primary use case — scientists think in concentration | LOW | Numeric input, range 0.01-10 mol/L | None |
| Solute type selection (THF/CH₄) | Core functionality — must pick molecule type | LOW | Dropdown with 2 options initially | None |
| Auto-calculate molecule count | Users specify concentration, not count | MEDIUM | `n_molecules = conc × volume × NA`; uses water density | Concentration + interface volume |

### Expected Behavior — Tab 4

**Standard workflow (modeled on GROMACS insert-molecules):**

1. **Input validation:**
   - Concentration must be positive, reasonable range (0.001-10 mol/L)
   - Solute type must be selected
   - Interface structure must exist from Tab 3

2. **Molecule count calculation:**
   ```python
   # Calculate liquid volume
   liquid_volume_nm3 = box_volume - ice_volume
   # Or simpler: count liquid waters, convert to volume
   n_liquid_waters = total_atoms - ice_atom_count // 4  # TIP4P = 4 atoms
   liquid_volume_nm3 = n_liquid_waters * 18.015 / (water_density_g_cm3 * 6.022e23) * 1e21
   
   # Calculate solute count
   n_solute = concentration_mol_L * liquid_volume_nm3 * 1e-21 * 6.022e23
   ```

3. **Placement algorithm (random insertion):**
   ```python
   # For each solute molecule:
   for i in range(n_solute):
       # Try up to MAX_ATTEMPTS times
       for attempt in range(MAX_ATTEMPTS):
           # Random COM position in liquid region
           com = random_position_in_liquid()
           
           # Random orientation (rotation matrix or Euler angles)
           rotation = random_rotation_matrix()
           
           # Check overlap with existing atoms
           if not has_overlap(solute_atoms, com, rotation, vdw_cutoff=0.2 nm):
               place_molecule(solute, com, rotation)
               break
   ```

4. **Overlap checking:**
   - Use cKDTree from `scipy.spatial` (already used in `overlap_resolver.py`)
   - Check distance between solute atoms and all existing atoms
   - Cutoff: scaled VDW radii sum (typical: 0.57 × (r1 + r2))
   - Fallback: fixed 0.2 nm minimum distance

5. **Output ordering (GROMACS .gro convention):**
   ```
   SOL (ice + liquid water)
   Hydrate guests (if any) — e.g., CH4_HYD
   Liquid solutes — e.g., THF_LIQ, CH4_LIQ
   Custom molecules (Tab 5)
   NA, CL ions (Tab 6)
   ```

## Tab 5: Custom Molecule Upload — Feature Analysis

### What This Tab Does

Tab 5 allows users to upload arbitrary molecules via .gro + .itp file pairs and insert them into the liquid phase. This extends the solute insertion capability to any molecule the user can parameterize.

**Two placement modes:**
1. **Random:** Random position in liquid, random orientation
2. **Custom:** User-specified COM (x, y, z) + rotation matrix (or Euler angles)

**Validation is critical:** Invalid .gro/.itp pairs will cause GROMACS failures downstream.

### Table Stakes — Tab 5

| Feature | Why Expected | Complexity | Notes | Dependency |
|---------|--------------|------------|-------|------------|
| File upload (.gro + .itp) | Standard workflow for custom molecules | MEDIUM | File dialogs, file parsing | None |
| GRO file parsing | Must read atom positions | MEDIUM | Parse residue name, atom names, coordinates | File upload |
| ITP file parsing | Must read topology | MEDIUM | Extract atomtypes, bonds, angles, dihedrals | File upload |
| Atom count validation | GRO and ITP must match | LOW | Compare atom counts from both files | GRO + ITP parsing |
| Residue name validation | Consistent naming across files | LOW | Extract resname from GRO, check ITP moleculetype name | GRO + ITP parsing |
| Random placement mode | Simplest insertion method | MEDIUM | Random COM + random rotation, overlap check | File upload + validation |
| Overlap detection | Prevent physically impossible structures | MEDIUM | Reuse cKDTree from Tab 4 | Placement |
| GROMACS export with custom .itp | Must produce valid topology | MEDIUM | Copy user .itp to output directory, reference in .top | Multi-molecule export |
| 3D visualization | Users must see custom molecule | MEDIUM | VTK actor for custom molecule type | VTK viewer |
| Error messages for invalid files | Debugging aid for users | LOW | Specific error: atom count mismatch, missing sections, etc. | Validation |

### Expected Behavior — Tab 5

**File validation checklist:**

1. **GRO file validation:**
   ```python
   # Check format
   - Line 1: title string
   - Line 2: number of atoms
   - Lines 3 to N+2: atom records (resid, resname, atomname, x, y, z)
   - Last line: box dimensions
   
   # Extract from GRO:
   - resname (must be ≤ 5 characters, no spaces)
   - atom names (must match ITP)
   - coordinates (must be in nm, reasonable range)
   - atom count (must match ITP)
   ```

2. **ITP file validation:**
   ```python
   # Required sections:
   - [ moleculetype ] — name and nrexcl
   - [ atoms ] — atom types, charges, masses
   - Optional: [ bonds ], [ angles ], [ dihedrals ], [ impropers ]
   
   # Extract from ITP:
   - moleculetype name (must match GRO resname)
   - atom count (must match GRO)
   - atomtypes (unique atom types for GROMACS atomtypes section)
   - bonded parameters (for GROMACS topology)
   ```

3. **Consistency checks:**
   ```python
   # Must match:
   - GRO atom count == ITP atom count
   - GRO resname == ITP moleculetype name
   - GRO atom names match ITP atom names (order matters!)
   
   # Must be present:
   - All atomtypes in ITP must exist in force field or be defined
   - Coordinates must be finite (no NaN or Inf)
   - Box dimensions from interface must accommodate custom placement
   ```

4. **Atomtypes extraction for GROMACS export:**
   ```python
   # Extract unique atom types from ITP [ atoms ] section
   atomtypes = set()
   for atom in itp_atoms:
       atomtypes.add(atom.type)
   
   # In final .top file, include [ atomtypes ] section:
   [ atomtypes ]
   ; name  mass  charge  ptype  sigma  epsilon
   custom_C  12.01  0.0  A  0.34  0.36
   custom_H  1.008  0.0  A  0.25  0.13
   ...
   ```

**Placement modes:**

**Random mode:**
```python
# Same as Tab 4 solute insertion
com = random_position_in_liquid()
rotation = random_rotation_matrix()  # 3x3 matrix or quaternion
place_molecule(custom_mol, com, rotation)
```

**Custom mode:**
```python
# User specifies:
com_x = float(x_input)  # nm
com_y = float(y_input)  # nm
com_z = float(z_input)  # nm

# Rotation matrix (user provides 9 values or Euler angles)
rotation_matrix = [
    [r11, r12, r13],
    [r21, r22, r23],
    [r31, r32, r33]
]

# Alternative: Euler angles (more intuitive for users)
# phi, theta, psi = rotation angles in degrees
rotation_matrix = euler_to_rotation_matrix(phi, theta, psi)

place_molecule(custom_mol, com, rotation_matrix)
```

```

┌─────────────────────────────────────┐
```

```
                                    │
```

### GROMACS Export Order

All tabs contribute to final GROMACS output:

```
1. SOL — All water molecules (ice + liquid from Tab 3)
2. Hydrate guests — e.g., CH4_HYD, THF_HYD (from Tab 2)
3. Liquid solutes — e.g., CH4_LIQ, THF_LIQ (from Tab 4)
4. Custom molecules — User-provided (from Tab 5)
5. NA — Sodium ions (from Tab 6)
6. CL — Chloride ions (from Tab 6)
```

### Residue Naming Convention

| Source | Residue Name | Location | Notes |
|--------|--------------|----------|-------|
| Tab 2 (Hydrate) | CH4_HYD, THF_HYD | Inside hydrate cages | Distinct from liquid solutes |
| Tab 4 (Solute) | CH4_LIQ, THF_LIQ | In liquid water | Distinct from hydrate guests |
| Tab 5 (Custom) | User-defined | In liquid water | Must not conflict with reserved names |
| Tab 6 (Ion) | NA, CL | In liquid water | Replaces water molecules |

**Reserved residue names:**
- `SOL` — Water molecules
- `NA`, `CL` — Ions
- `*_HYD` — Hydrate guests
- `*_LIQ` — Liquid solutes

## Validation & Error Handling

### Common User Errors

| Error | Detection | User Message |
|-------|-----------|--------------|
| Negative concentration | Input validation | "Concentration must be positive" |
| Concentration too high | Physical check | "Concentration exceeds solubility limit for [solute]" |
| No interface structure | State check | "Generate interface in Tab 3 first" |
| GRO/ITP mismatch | File validation | "GRO file has [N] atoms, ITP has [M] atoms — must match" |
| Residue name conflict | Name check | "Residue name 'SOL' is reserved for water" |
| Invalid rotation matrix | Matrix check | "Rotation matrix must be orthogonal (det = ±1)" |
| COM outside liquid region | Position check | "Center of mass (x, y, z) is not in liquid region" |
| Overlap detected | Placement check | "Cannot place molecule without overlap. Try larger box or fewer molecules." |

### Warnings (Non-blocking)

| Warning | When | User Message |
|---------|------|--------------|
| High concentration | conc > 1 mol/L | "High concentration may cause GROMACS instability. Consider energy minimization." |
| Custom molecule untested | After upload | "Custom molecule not validated. Run test MD before production." |
| No energy minimization | After insertion | "Inserted molecules need energy minimization before production MD." |
