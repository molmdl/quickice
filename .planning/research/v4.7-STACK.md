### Filled Ice Guest Placement (VERIFIED)

Guest placement in filled ices uses the **cage type name** `Ne1` in the `guests` dict, NOT numeric cage IDs (like `12`, `14` for clathrates).

**Python verification:**
```python
from genice2.genice import GenIce
from genice2.plugin import safe_import
from genice2.valueparser import parse_guest
from collections import defaultdict

lat = safe_import('lattice', 'c0te').Lattice()
water = safe_import('molecule', 'tip4p').Molecule()
fmt = safe_import('format', 'gromacs').Format()

guests = defaultdict(dict)
parse_guest(guests, 'Ne1=co2')  # Use cage type name "Ne1"

ice = GenIce(lat)
gro_string = ice.generate_ice(formatter=fmt, water=water, guests=guests, depol='strict')
# Output: 33 atoms (24 water + 9 CO2 = 3 CO2 molecules × 3 atoms)
```

### ⚠️ CRITICAL: Water Model Changes Residue Name and Atom Names

**The current `_build_molecule_index()` in `hydrate_generator.py` ONLY recognizes:**
- Residue name `ICE` for water
- Atom names `OW/HW1/HW2/MW` (4-site TIP4P pattern)
- Atom names `O/H/H` (3-site pattern, with residue name check `!= 'THF'`)

**When switching to TIP3P, SPC/E, TIP5P, etc., the residue name changes from `ICE` to `SOL`:**
```
TIP4P output:      1ICE     OW    1   ...
TIP3P output:      1SOL      O    1   ...
SPC/E output:      1SOL     Ow    1   ...
TIP5P output:      1SOL     OW    1   ...
```

**This breaks `_build_molecule_index()` for ALL non-TIP4P water models.** The function won't match `SOL` residue water molecules. Fix: make water residue detection configurable or use `atom_name` pattern matching instead of residue name matching.

### Recommended Water Models for v4.7

For the v4.7 release, support only the **scientifically common** water models. The niche models (6-site, 7-site, NvdE, physical_water) can be added later if requested.

## What Does NOT Work (CRITICAL)

### 1. `assess_cages=True` on Filled Ices → NameError (BUG)

```python
# THIS CRASHES:
ice = GenIce(safe_import('lattice', 'c0te').Lattice())
gro = ice.generate_ice(formatter=fmt, water=water, guests={}, depol='strict', assess_cages=True)
# → NameError: name 'self' is not defined
# (genice2/genice.py line 1172: cagepos = self.cagepos)
```

**Root cause:** Bug in GenIce2's `dopants_info()` function (line 1172 uses `self.cagepos` instead of the correct parameter). This is a GenIce2 upstream bug.

**Impact:** Cannot auto-detect cage structure for filled ices. Must use known cage type names (`Ne1`) instead.

**Workaround:** Don't use `assess_cages=True` for filled ices. Hardcode the cage type mapping for each filled ice lattice.

### 2. `sTprime` Has No Cages — Guests Cannot Be Placed

```python
lat = safe_import('lattice', 'sTprime').Lattice()
print(lat.cagetype)  # → []
print(lat.cagepos)   # → []
# No standard way to add guests to sTprime
```

**Root cause:** The `sTprime` lattice module defines only `waters` and `bondlen` — no cage positions. It's a pure water network.

**Impact:** sTprime generates only water structures. If users need filled sT' with guests, they must:
1. Use a custom lattice plugin that adds cage definitions, OR
2. Place guests manually using the `spot_guests` parameter with pre-computed cage center positions, OR
3. Use `genice2-cif` with a CIF file that includes guest positions

### 3. Ethane Plugin is `et`, NOT `ethane`

```python
safe_import('molecule', 'ethane')  # → AssertionError: Nonexistent or failed to load the module: ethane
safe_import('molecule', 'et')     # → Works (united-atom ethane, 1 atom)
```

**Impact:** UI must use `et` as the internal name, not `ethane`. Display name can still be "Ethane".

### 4. Water Model ≠ TIP4P Breaks `_build_molecule_index()`

```python
# TIP4P (current): residue name "ICE", atoms "OW,HW1,HW2,MW" — works ✅
# TIP3P:           residue name "SOL", atoms "O,H,H"         — BROKEN ❌
# SPC/E:           residue name "SOL", atoms "Ow,Hw,Hw"       — BROKEN ❌
# TIP5P:           residue name "SOL", atoms "OW,HW1,HW2,MW1,MW2" — BROKEN ❌
```

**Impact:** `_build_molecule_index()` needs refactoring BEFORE water model selector is exposed. Must handle multiple residue name patterns and atom naming conventions.

### 5. `c1te` Has `fixed` H-Bond Orientations

```python
lat = safe_import('lattice', 'c1te').Lattice()
print(lat.fixed)  # → [[33,13], [0,13], [0,10], ...]  (72 pairs)
```

**Impact:** c1te is hydrogen-ordered. The `asis=False` (default) parameter allows GenIce2 to rearrange H-bonds, which may violate the C1 ordering. Consider setting `asis=True` for c1te, or at minimum warn users.
