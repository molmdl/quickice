# Executive Summary: License Analysis for Pre-built Small Molecules

**Verdict: GO — with BSD-3-Clause attribution strategy**

The license analysis reveals that redistributing pre-built small molecule files under MIT is **legally viable** using the phenix geostd repository as the data source. The PDB Chemical Component Dictionary is CC0 (public domain), and the phenix geostd repository explicitly distributes mol2/frcmod files — the same antechamber output as the AMBER geostd archive — under a BSD-3-Clause license, which is fully MIT-compatible. The FSF's GPL FAQ authoritatively states that the output of a GPL-licensed program is NOT automatically GPL; the copyright of output belongs to the user (or inherits from the input data), and this applies to antechamber/parmchk2-generated mol2/frcmod files just as it applies to GCC-compiled object code. GAFF2 atom type labels and individual force field parameters are likely not copyrightable as they are functional labels and scientific facts respectively, and the abcg2 charges in the mol2 files are computed output rather than verbatim copies of AmberTools' BCC parameter tables. The only real risk is the AMBER geostd archive itself, which lacks an explicit license on its data files; however, this risk is entirely avoidable by sourcing equivalent data from the BSD-3-Clause-licensed phenix geostd repository instead.

**Recommended strategy:** Source mol2/frcmod data from the phenix geostd repository (BSD-3-Clause), include the required BSD-3-Clause attribution and copyright notice, add scientific citations for GAFF2/abcg2/phenix geostd methodologies, and ship the QuickIce format converter (mol2/frcmod → .gro/.itp) under MIT. Provide a standalone converter function for user-supplied AMBER files as a secondary feature. Do NOT bundle data from the AMBER geostd archive (amber_geostd.tar.bz2) without first obtaining explicit redistribution permission from the AMBER team.
