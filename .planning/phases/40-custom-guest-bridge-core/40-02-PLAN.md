  <action>
Extend `transform_guest_itp(itp_content, guest_name, suffix="_H")` in `quickice/output/gromacs_writer.py`.

Currently the function does:
- Step 1: `comment_out_atomtypes_in_itp(itp_content)` — comments out `[ atomtypes ]`.
- Step 2: Renames `[ moleculetype ]` name to `new_name = f"{guest_name}{suffix}"` (e.g., "CH4" -> "CH4_H"), validates GRO name length via `validate_gro_residue_name`.

Add Step 3 AFTER Step 2 (after the moleculetype rename loop completes and `content` is the renamed string): rewrite the resname column in the `[ atoms ]` section to `new_name` (the same `{guest_name}{suffix}` value, e.g., "MOL_H").

GROMACS `[ atoms ]` section format (from etoh.itp):
```
[ atoms ]
;  Index   type   residue  resname   atom         cgnr     charge       mass
     1     hc         1      MOL     H              1    0.05772791    1.007941
```
The resname is the 4th column (index 3, 0-based) on each data line. Columns are whitespace-separated.

Implementation for Step 3:
- Search the `content` for the `[ atoms ]` section using the same regex pattern style as existing code: `re.search(r'\[\s*atoms\s*\](.*?)(?=\[\s*\w+\s*\]|$)', content, re.DOTALL | re.IGNORECASE)`.
- If no `[ atoms ]` section found, return content as-is (graceful no-op — some ITPs may lack it).
- For each line in the section body: skip blank lines and comment lines (starting with `;` or `#`). For data lines, split into fields; if `len(fields) >= 4`, replace `fields[3]` (the resname, 0-indexed) with `new_name`, then rejoin with single spaces. Preserve leading whitespace by using the original line's leading whitespace if practical (or just rejoin with the split fields joined by spaces — acceptable since GROMACS is whitespace-flexible in ITP).
- Reconstruct the content with the rewritten `[ atoms ]` section spliced back in.
- IMPORTANT: Do NOT modify the `[ moleculetype ]` section or `[ atomtypes ]` section in this step (they were already handled in Steps 1-2).

Add `import re` at the top of gromacs_writer.py if not already present (it likely is — check first).

Use `except (ValueError, re.error)` not bare `except` if wrapping in try/except (though this function should be pure string processing — no try/except needed if logic is sound).
