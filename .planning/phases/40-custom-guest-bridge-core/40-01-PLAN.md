  <files>quickice/structure_generation/itp_parser.py</files>
  <action>
Add a new function `parse_itp_defaults_comb_rule(content: str) -> int | None` to `quickice/structure_generation/itp_parser.py` (after the existing `parse_itp_file` function).

The GROMACS `[ defaults ]` section format:
```
[ defaults ]
; nbfunc  comb-rule  gen-pairs  fudgeLJ  fudgeQQ
     1        2         yes       0.5     0.8333
```
The comb-rule is the 2nd field (index 1) on the first non-comment, non-blank data line after the `[ defaults ]` header.

Implementation requirements:
- Use `re.search(r'\[\s*defaults\s*\](.*?)(?=\[\s*\w+\s*\]|$)', content, re.DOTALL | re.IGNORECASE)` to extract the `[ defaults ]` section body (up to the next `[ section ]` header or end of content).
- If no `[ defaults ]` section found, return `None` (this is VALID — etoh.itp has no `[ defaults ]`; the main `.top` supplies comb-rule=2).
- Iterate the section body lines; skip blank lines and comment lines (starting with `;` or `#`).
- On the first data line, split into fields; if `len(fields) >= 2`, try `int(fields[1])` and return it; on `ValueError` return `None`.
- If no data line found, return `None`.
- Add a module-level `re` import if not already present (it IS already imported at top of itp_parser.py — verify before adding).
- Use `except (ValueError, IndexError)` not bare `except`.
- Add a docstring with the format example above and the validation rule: "Reject comb-rule != 2 ONLY when this returns a non-None value; accept when None (main .top supplies comb-rule=2)."
  </action>
  <verify>`python -c "from quickice.structure_generation.itp_parser import parse_itp_defaults_comb_rule; print(parse_itp_defaults_comb_rule('[ defaults ]\n; nbfunc comb-rule\n1 2 yes 0.5 0.8333'))"` prints `2`</verify>
  <done>parse_itp_defaults_comb_rule is importable and returns 2 for comb-rule=2 input, 1 for comb-rule=1 input, None for absent [ defaults ]</done>
