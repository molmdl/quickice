<objective>
Extend `HydrateConfig` in types.py to carry `guest_residue_name` and `guest_gro_path`, and to accept custom guest types (not in `GUEST_MOLECULES`) when explicit metadata is provided.

Purpose: The custom guest bridge (40-04) and hydrate generator integration (40-05) need HydrateConfig to (a) distinguish the GenIce2 plugin name (`guest_type`, e.g. "etoh_custom") from the GRO residue name (`guest_residue_name`, e.g. "MOL"), (b) know the GRO file path to build the Molecule module from, and (c) accept custom guest types that aren't in the built-in `GUEST_MOLECULES` dict. Per decision [38-01], custom types must provide metadata explicitly (no auto-populate).

Output: Extended HydrateConfig with two new fields, an `is_custom_guest` property, relaxed `__post_init__` validation, and `from_dict` passthrough.
</objective>

  <name>Task 1: Add guest_residue_name, guest_gro_path fields + is_custom_guest property to HydrateConfig</name>
  <files>quickice/structure_generation/types.py</files>
  <action>
Modify the `HydrateConfig` dataclass in `quickice/structure_generation/types.py`.

1. Add two new fields AFTER `guest_itp_path` (at the end of the field list, before `__post_init__`):
   ```python
   guest_residue_name: str = ""
   guest_gro_path: str = ""
   ```
   These default to empty string (backward compatible — built-in guests don't need them).

2. Add a property `is_custom_guest` to the class (after `__post_init__` or after `from_dict`):
   ```python
   @property
   def is_custom_guest(self) -> bool:
       """True if guest_type is a custom guest (not in GUEST_MOLECULES)."""
       return self.guest_type not in GUEST_MOLECULES
   ```

3. Modify `__post_init__` to handle custom guests. Currently it does:
   ```python
   if self.guest_type not in GUEST_MOLECULES:
       raise ValueError(f"Unknown guest type: {self.guest_type}")
   ```
   Replace this single check with a branch:
   ```python
   if self.guest_type not in GUEST_MOLECULES:
       # Custom guest (Phase 40): require explicit metadata, do NOT auto-populate
       if not self.guest_residue_name:
           raise ValueError(
               f"Custom guest_type '{self.guest_type}' requires guest_residue_name "
               f"(the GRO residue name, <=3 chars base for _H suffix). "
               f"Set guest_residue_name explicitly."
           )
       if not self.guest_atom_labels:
           raise ValueError(
               f"Custom guest_type '{self.guest_type}' requires guest_atom_labels "
               f"(atom name sequence for one molecule, for metadata-driven identification). "
               f"Set guest_atom_labels explicitly."
           )
       if not self.guest_atom_count:
           raise ValueError(
               f"Custom guest_type '{self.guest_type}' requires guest_atom_count "
               f"(number of atoms per guest molecule). Set guest_atom_count explicitly."
           )
       if not self.guest_gro_path:
           raise ValueError(
               f"Custom guest_type '{self.guest_type}' requires guest_gro_path "
               f"(path to the custom guest .gro file for GenIce2 Molecule module building). "
               f"Set guest_gro_path explicitly."
           )
       # Custom guest: guest_name defaults to guest_residue_name if not set
       if not self.guest_name:
           self.guest_name = self.guest_residue_name
   else:
       # Built-in guest: auto-populate metadata from GUEST_MOLECULES (existing behavior)
       guest_info = GUEST_MOLECULES[self.guest_type]
       if not self.guest_name:
           self.guest_name = guest_info["name"]
       if not self.guest_atom_labels:
           self.guest_atom_labels = list(guest_info["atom_labels"])
       if not self.guest_atom_count:
           self.guest_atom_count = guest_info["atoms"]
   ```
   Keep ALL other validation unchanged (lattice_type, cage_occupancy, supercell).

4. Update `from_dict` to pass through the new fields:
   ```python
   guest_residue_name=d.get("guest_residue_name", ""),
   guest_gro_path=d.get("guest_gro_path", ""),
   ```
   Add these two lines to the `cls(...)` call in `from_dict`, after `guest_itp_path=...`.

5. Update the docstring of HydrateConfig to document the two new fields and `is_custom_guest`.
