**Module structure:**
```python
"""CLI pipeline executor for QuickIce.

Orchestrates multi-step structure generation: Source → Interface → Custom → Solute → Ion → Export.
Mirrors the GUI's MainWindow workflow but runs entirely in the CLI without PySide6/VTK.
"""
import sys
import csv
import logging
from pathlib import Path
import argparse

logger = logging.getLogger(__name__)


def report_progress(message: str) -> None:
    """Print progress message to stderr (keeps stdout clean for piping)."""
    print(f"[PROGRESS] {message}", file=sys.stderr)


class CLIPipeline:
    """Orchestrates multi-step structure generation pipeline."""
    
    def __init__(self, args: argparse.Namespace):
        self.args = args
        self._interface_result = None   # InterfaceStructure
        self._hydrate_result = None     # HydrateStructure
        self._custom_result = None      # CustomMoleculeStructure
        self._solute_result = None      # SoluteStructure
        self._ion_result = None         # IonStructure
        self._ice_candidate = None      # Candidate
        self._output_dir = None         # Path — created in execute()
    
    def execute(self) -> int:
        """Run all requested pipeline steps in order. Returns exit code."""
        # Create output directory
        self._output_dir = Path(self.args.output).resolve()
        self._output_dir.mkdir(parents=True, exist_ok=True)
        
        # Step 1: Generate source (ice candidate or hydrate)
        if self.args.interface or (getattr(self.args, 'hydrate', False)):
            self._run_source_step()
        
        # Step 2: Interface generation
        if self.args.interface:
            self._run_interface_step()
        
        # Step 3: Custom molecule insertion
        if getattr(self.args, 'custom_gro', None):
            self._run_custom_step()
        
        # Step 4: Solute insertion
        if getattr(self.args, 'solute_type', None):
            self._run_solute_step()
        
        # Step 5: Ion insertion
        if getattr(self.args, 'ion_concentration', None):
            self._run_ion_step()
        
        # Step 6: Export
        self._run_export_step()
        
        return 0
```

**Step method implementations:**

**`_run_source_step()`**: Generate ice candidate or hydrate.
- If `--hydrate`: Create `HydrateConfig` from args, call `HydrateStructureGenerator().generate(config)`, store as `self._hydrate_result`. Report: `"Generated {lattice_type} hydrate with {guest_count} {guest_type} guests"`.
- If `--interface` without `--hydrate`: Call `generate_candidates()` with `phase_info` (need `lookup_phase(args.temperature, args.pressure)`), `nmolecules=256`, `n_candidates=1`. Store first candidate as `self._ice_candidate`. Report: `"Generated ice candidate ({phase_id})"`.
- If `--hydrate` + `--interface`: Generate hydrate, then convert to candidate via `hydrate.to_candidate()`. Store as `self._ice_candidate`. Report both steps.

Use `getattr(self.args, 'hydrate', False)` and `getattr(self.args, 'solute_type', None)` for safe attribute access since flags are added by Plan 01.

**`_run_interface_step()`**: Generate interface structure.
- Create `InterfaceConfig` from args (same as existing main.py logic).
- Call `generate_interface(self._ice_candidate, config)`.
- Store as `self._interface_result`. Report: `"Interface: {ice_nmolecules} ice + {water_nmolecules} water molecules"`.
- Print report if available: `result.report` to stderr.

**`_run_custom_step()`**: Insert custom molecules.
- Validate `.gro` and `.itp` files exist (raise FileNotFoundError with clear message if not).
- Parse `CustomMoleculeConfig` from args:
  - If `--custom-placement random`:
    - If `--custom-count`: use that count
    - If `--custom-concentration`: calculate count via `CustomMoleculeInserter.calculate_molecule_count(concentration, liquid_volume)` where `liquid_volume = self._interface_result.water_nmolecules * 0.0299`
  - If `--custom-placement custom`:
    - Parse CSV file from `--custom-positions-file` using `self._parse_positions_csv(filepath)`
    - Create config with positions/rotations
- Create `CustomMoleculeInserter(config, seed=args.seed)` and call `place_random(structure, count)` or `place_custom(structure, positions, rotations)`.
- The structure input is always `self._interface_result`.
- Store as `self._custom_result`. Report: `"Custom molecules: {count} placed"`.

**`_run_solute_step()`**: Insert solutes.
- Get source structure based on `--solute-source`:
  - "interface": `self._interface_result`
  - "custom": `self._custom_result` (must not be None, error if missing)
- Create `SoluteConfig(concentration_molar=args.solute_concentration, solute_type=args.solute_type, seed=args.seed)`.
- Create `SoluteInserter(config)` and call `inserter.insert_solutes(source, config)`.
- If source is "custom" (CustomMoleculeStructure), the inserter handles it via `getattr(structure, 'water_nmolecules', None)` fallback (already in SoluteInserter code).
- Register hydrate guest if source interface has guests (mirror GUI pattern: `registry = MoleculetypeRegistry(); registry.register_hydrate_guest(guest_upper)` before calling insert_solutes). Actually, SoluteInserter creates its own registry internally, so this step may not be needed — check by reading SoluteInserter.__init__.
- Store as `self._solute_result`. Report: `"Solute: {n_molecules} {solute_type} molecules inserted"`.

**`_run_ion_step()`**: Insert ions — MOST COMPLEX step due to source selection attribute propagation.
- Get source name from `--ion-source`.
- Source resolution logic (mirrors GUI MainWindow._on_insert_ions lines 820-929):
  - "interface": `source = self._interface_result`
  - "custom": `source = self._custom_result`; then get `interface = source.interface_structure`; attach custom molecule attributes to interface:
    ```python
    interface.custom_molecule_positions = source.positions[source.ice_atom_count + source.water_atom_count:]
    interface.custom_molecule_atom_names = source.atom_names[source.ice_atom_count + source.water_atom_count:]
    interface.custom_molecule_count = source.custom_molecule_count
    interface.custom_molecule_moleculetype = source.moleculetype_name
    interface.custom_gro_path = source.gro_path
    interface.custom_itp_path = source.itp_path
    source_for_ions = interface
    ```
  - "solute": `source = self._solute_result`; then get `interface = source.interface_structure`; attach solute attributes to interface:
    ```python
    interface.solute_type = source.solute_type
    interface.solute_positions = source.positions
    interface.solute_atom_names = source.atom_names
    interface.solute_n_molecules = source.n_molecules
    interface.solute_molecule_indices = source.molecule_indices
    interface.solute_registry = source.registry
    source_for_ions = interface
    ```
    ALSO: if custom molecules present in solute source, propagate custom attributes too:
    ```python
    if source.custom_molecule_count > 0 and source.custom_molecule_positions is not None:
        interface.custom_molecule_count = source.custom_molecule_count
        interface.custom_molecule_atom_count = source.custom_molecule_atom_count
        interface.custom_molecule_positions = source.custom_molecule_positions
        interface.custom_molecule_atom_names = source.custom_molecule_atom_names
        interface.custom_molecule_moleculetype = source.custom_molecule_moleculetype
        interface.custom_gro_path = source.custom_gro_path
        interface.custom_itp_path = source.custom_itp_path
    ```
- Calculate liquid volume: `liquid_volume = source_for_ions.water_nmolecules * 0.0299`
  - For custom/solute sources, get `water_nmolecules` from the INTERFACE (not the source structure directly): `getattr(interface, 'water_nmolecules', None)`; fallback: `water_atom_count // WATER_ATOMS_PER_MOLECULE`.
- Call `insert_ions(source_for_ions, args.ion_concentration, liquid_volume, seed=args.seed)`.
- Store as `self._ion_result`. Report: `"Ion insertion: {na_count} Na+, {cl_count} Cl-"`.

**`_run_export_step()`**: Export most downstream structure as GROMACS files.
- Determine most downstream structure (priority: ion > solute > custom > interface > ice):
  ```python
  if self._ion_result is not None:
      structure = self._ion_result
      step_name = "ion"
  elif self._solute_result is not None:
      structure = self._solute_result
      step_name = "solute"
  elif self._custom_result is not None:
      structure = self._custom_result
      step_name = "custom"
  elif self._interface_result is not None:
      structure = self._interface_result
      step_name = "interface"
  else:
      # ice-only (existing main.py path)
      ...
  ```
- For each structure type, call the appropriate writer functions from `quickice/output/gromacs_writer.py`:
  - Ice candidate: `write_gro_file()`, `write_top_file()`
  - Interface: `write_interface_gro_file()`, `write_interface_top_file()`
  - Custom: `write_custom_molecule_gro_file()`, `write_custom_molecule_top_file()`
  - Solute: `write_solute_gro_file()`, `write_solute_top_file()`
  - Ion: `write_ion_gro_file()`, `write_ion_top_file()`
- Output filenames: `{step_name}.gro`, `{step_name}.top` (per CONTEXT.md decision)
- Generate `ion.itp` if ion step present: `write_ion_itp(str(self._output_dir / "ion.itp"))`
- Copy all ITP files using `quickice/cli/itp_helpers.py` (see Task 2)
- Check `--no-overwrite` flag: if set and any output file exists, print error to stderr and return 1
- Report: `"Exported GROMACS files: {step_name}.gro, {step_name}.top, {itp_count} ITP files → {output_dir}"`

**`_get_source_structure(source_name)`**: Helper to return stored structure by name.
- "interface" → `self._interface_result`
- "custom" → `self._custom_result`
- "solute" → `self._solute_result`
- Raise ValueError for unknown source names

**`_parse_positions_csv(filepath)`**: Parse CSV file for custom placement.
- Read file, skip lines starting with `#`
- Parse each row as: x, y, z, alpha, beta, gamma (all floats)
- Validate 6 columns per row
- Return `(positions: list[tuple[float,float,float]], rotations: list[tuple[float,float,float]])`

**Error handling pattern:**
- Each `_run_*_step()` method wraps its logic in try/except
- On `ImportError` (GenIce2 missing): print error to stderr, return 1
- On `ValueError` (bad config): print error to stderr, return 1
- On `InsertionError` (overlap failure): print error to stderr, return 1
- On `InterfaceGenerationError`: print error to stderr, return 1
- The `execute()` method checks return values and stops on first failure

**IMPORTANT:** Use `from quickice.structure_generation.types import WATER_ATOMS_PER_MOLECULE` for the water atom count constant. Do NOT hardcode `// 4` or `* 0.0299` as a magic number — use `WATER_ATOMS_PER_MOLECULE` and the existing `0.0299` nm³ per TIP4P molecule constant (this is in SoluteInserter already as `_WATER_VOLUME_PER_MOLECULE_NM3 = 0.0299`; import it or define locally).

4. **`copy_itp_files_for_structure(output_dir: Path, structure, step_name: str, hydrate_config=None) -> list[str]`**

   - Use `shutil.copy(source, dest)` for direct copies
   - For custom ITP: read source content, apply `comment_out_atomtypes_in_itp()`, write to dest
   - Handle the hydrate guest detection: if `structure` has `guest_nmolecules > 0` or `guest_atom_count > 0`, detect guest type from molecule_index or atom count heuristic (5 atoms=ch4, 13 atoms=thf). Use `detect_guest_type_from_atoms` from `quickice.output.gromacs_writer` if available.
