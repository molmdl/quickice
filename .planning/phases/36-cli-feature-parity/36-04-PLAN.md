Replace the `_run_ion_step()` stub with a full implementation. This is the most complex step due to source selection and attribute propagation.

**`_run_ion_step() -> int`**: Insert ions — mirrors GUI MainWindow._on_insert_ions (lines 820-929).

- Get source name from `--ion-source` (default: "interface")
- Source resolution logic (each case mirrors the GUI's MainWindow code):

  **Case "interface":** `source_for_ions = self._interface_result`

  **Case "custom":** 
  ```python
  source = self._custom_result
  if source is None:
      print("Error: No custom molecule structure available. Run --custom-gro step first.", file=sys.stderr)
      return 1
  interface = source.interface_structure
   # Attach custom molecule attributes to interface (mirrors GUI)
   # CustomMoleculeStructure positions: ice + water + guests + custom (see custom_molecule_inserter.py:748-753)
   # GUI code (main_window.py:898) uses ice + water + guest_atom_count offset
   interface.custom_molecule_positions = source.positions[source.ice_atom_count + source.water_atom_count + source.guest_atom_count:]
   interface.custom_molecule_atom_names = source.atom_names[source.ice_atom_count + source.water_atom_count + source.guest_atom_count:]
  interface.custom_molecule_count = source.custom_molecule_count
  interface.custom_molecule_moleculetype = source.moleculetype_name
  interface.custom_gro_path = source.gro_path
  interface.custom_itp_path = source.itp_path
  source_for_ions = interface
  ```

  **Case "solute":**
  ```python
  source = self._solute_result
  if source is None:
      print("Error: No solute structure available. Run --solute-type step first.", file=sys.stderr)
      return 1
  interface = source.interface_structure
  # Attach solute attributes to interface (mirrors GUI)
  interface.solute_type = source.solute_type
  interface.solute_positions = source.positions
  interface.solute_atom_names = source.atom_names
  interface.solute_n_molecules = source.n_molecules
  interface.solute_molecule_indices = source.molecule_indices
  interface.solute_registry = source.registry
  source_for_ions = interface
  # ALSO: if custom molecules present in solute source, propagate custom attributes
  if hasattr(source, 'custom_molecule_count') and source.custom_molecule_count > 0:
      if hasattr(source, 'custom_molecule_positions') and source.custom_molecule_positions is not None:
          interface.custom_molecule_count = source.custom_molecule_count
          interface.custom_molecule_atom_count = source.custom_molecule_atom_count
          interface.custom_molecule_positions = source.custom_molecule_positions
          interface.custom_molecule_atom_names = source.custom_molecule_atom_names
          interface.custom_molecule_moleculetype = source.custom_molecule_moleculetype
          interface.custom_gro_path = source.custom_gro_path
          interface.custom_itp_path = source.custom_itp_path
  ```

- Calculate liquid volume: `liquid_volume = source_for_ions.water_nmolecules * 0.0299`
  - For custom/solute sources, `water_nmolecules` comes from the INTERFACE (attached via `interface_structure`), so `getattr(interface, 'water_nmolecules', None)` works. Fallback: `water_atom_count // WATER_ATOMS_PER_MOLECULE` where `water_atom_count = getattr(interface, 'water_atom_count', 0)`.
- Call `insert_ions(source_for_ions, concentration_molar=args.ion_concentration, liquid_volume_nm3=liquid_volume, seed=args.seed)`
- Store as `self._ion_result`. Report: `"Ion insertion: {na_count} Na+, {cl_count} Cl-"`

**Important:** The attribute propagation pattern is duck-typing (setting attributes on InterfaceStructure at runtime). This is the EXACT same pattern used by the GUI's MainWindow — do NOT try to create a "cleaner" abstraction. The inserter code expects these attributes to exist on the interface structure when it receives it.

**Type annotation note:** `insert_ions()` type annotation says `IonStructure` but it actually accepts `InterfaceStructure` via duck typing — the same pattern used throughout the GUI codebase. This is a known pattern documented in e2e-api-workflow tests.
