  <action>
Add 5 new argument groups to `create_parser()` and create `validate_pipeline_args()` function.

**New argument groups (add AFTER the existing interface_group, BEFORE --version):**

1. **hydrate generation** group:
   - `--hydrate` action="store_true"
   - `--lattice-type` choices=["sI", "sII", "sH"], default="sI"
   - `--guest` choices=["CH4", "THF"], default="CH4"
   - `--supercell-x`, `--supercell-y`, `--supercell-z` type=int, default=1
   - `--cage-occupancy-small` type=float, default=100.0
   - `--cage-occupancy-large` type=float, default=100.0

2. **custom molecule insertion** group:
   - `--custom-gro` type=str
   - `--custom-itp` type=str
   - `--custom-placement` choices=["random", "custom"], default="random"
   - `--custom-count` type=int
   - `--custom-concentration` type=float
   - `--custom-positions-file` type=str

3. **solute insertion** group:
   - `--solute-type` choices=["CH4", "THF"]
   - `--solute-concentration` type=float
   - `--solute-source` choices=["interface", "custom"], default="interface"

4. **ion insertion** group:
   - `--ion-concentration` type=float
   - `--ion-source` choices=["interface", "custom", "solute"], default="interface"

5. **output options** extension: `--no-overwrite` action="store_true", default=False

**Create `validate_pipeline_args(parser, args)` (called after `parser.parse_args()`):**
Keep existing `validate_interface_args` call. Add new validation rules (use `parser.error()` for exit code 2):
- `--hydrate` + `--nmolecules` → invalid (hydrate uses supercell)
- `--custom-gro` requires `--custom-itp` and vice versa
- `--custom-gro` requires `--interface`
- `--custom-placement custom` requires `--custom-positions-file`
- `--custom-placement random` requires `--custom-count` or `--custom-concentration` (not both)
- `--solute-type` requires `--solute-concentration`
- `--solute-type` requires `--interface`
- `--solute-source custom` requires `--custom-gro`
- `--ion-concentration` requires `--interface`
- `--ion-source custom` requires `--custom-gro`
- `--ion-source solute` requires `--solute-type`

**Update `create_parser()` epilog** with v4.5 pipeline examples.
**Update `get_arguments()` docstring** to document new Namespace attributes.
