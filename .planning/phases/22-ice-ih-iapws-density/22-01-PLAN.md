Create `quickice/phase_mapping/ice_ih_density.py` implementing IAPWS R10-06(2009) Ice Ih density calculation using `iapws._iapws._Ice(T, P)`.

Key implementation details:
- Import `from iapws._iapws import _Ice` — this function ALREADY implements IAPWS R10-06(2009)
- `_Ice(T, P)` takes T in **Kelvin** and P in **MPa** (same as our codebase — NO unit conversion needed)
- `_Ice(T, P)` returns a dict with key `"rho"` in kg/m³
- Convert kg/m³ → g/cm³ by dividing by 1000

Two public functions:
1. `ice_ih_density_kgm3(T_K: float, P_MPa: float) -> float` — returns density in kg/m³, decorated with `@lru_cache(maxsize=256)`
2. `ice_ih_density_gcm3(T_K: float, P_MPa: float) -> float` — returns density in g/cm³, calls ice_ih_density_kgm3 and divides by 1000

Error handling:
- `_Ice` raises `NotImplementedError("Incoming out of bound")` when P > 208.566 MPa — catch this and return `FALLBACK_DENSITY_GCM3 * 1000`
- `_Ice` may also raise `ValueError` or `OverflowError` for extreme inputs — catch these too
- Define `FALLBACK_DENSITY_GCM3 = 0.9167` as module constant (the old hardcoded value at 273.15K, 1 atm)

Warning suppression:
- `_Ice` issues `warnings.warn("Metastable ice in liquid region")` or `warnings.warn("Metastable ice")` for conditions outside stable Ice Ih — suppress these with `warnings.filterwarnings("ignore", message="Metastable ice")` inside the cached function, or use `with warnings.catch_warnings():` context manager

Module docstring should reference IAPWS R10-06(2009) and the iapws library.

Do NOT add `gsw` as a dependency — iapws already has Ice Ih EOS.
