<name>Task 3: Implement all-atom overlap checking and insert_solutes()</name>
<files>quickice/structure_generation/solute_inserter.py</files>
<action>
Add all-atom overlap checking and main insert_solutes() method to SoluteInserter.

**_build_existing_atoms_tree() method** (follow ion_inserter.py lines 274-340):
- Args: structure (InterfaceStructure), exclude_water (bool, default False)
- Build cKDTree from existing atoms for overlap checking
- **CRITICAL**: Exclude MW atoms (virtual sites) to avoid false positives
- **CRITICAL**: Only use atoms from liquid region if exclude_water=True
- Return cKDTree or None

**_check_solute_overlap() method**:
- Args: solute_positions (np.ndarray), existing_tree (cKDTree), min_separation (float)
- Check ALL atoms in solute molecule (not center-of-mass)
- Return True if overlap detected, False if valid placement

**insert_solutes() method** (main entry point):
- Args: structure (InterfaceStructure), config (SoluteConfig | None)
- Steps:
  1. Calculate molecule count from concentration and liquid volume
  2. Load solute template using _load_solute_template()
  3. Build cKDTree from existing atoms (exclude water for replacement)
  4. **CRITICAL**: Only sample positions from liquid region (positions[ice_atom_count:ice_atom_count + water_atom_count])
  5. For each molecule:
     - Generate random position in liquid region bounds
     - Generate random rotation matrix
     - Rotate template and translate to position
     - Check overlap with _check_solute_overlap()
     - If no overlap: add to structure, update tree
     - If max_attempts reached: continue to next molecule (partial success)
  6. Create SoluteStructure with placed molecules
  7. Register with MoleculetypeRegistry: registry.register_liquid_solute(config.solute_type)
  8. Return SoluteStructure
- Handle partial success: log "Placed N of M molecules" if N < M
- Raise ValueError if concentration is invalid (negative, too high)

**Follow ion_inserter.py patterns:**
- Lines 350-415 for insertion loop
- Lines 406-415 for partial success handling
- Use self.rng for random numbers (reproducible)
</action>
