## Decisions Made

- **7 files need path edits (not 3 as research stated):** The 48.2-RESEARCH.md §2 said "14 zero-edit + 3 one-line edits" but empirically verified at plan time that 7 files use `Path(__file__).parent.parent` / `Path(__file__).resolve().parents[1]` / `Path(__file__).resolve().parent.parent` patterns that resolve to PROJECT ROOT from tests/ root but would resolve to tests/ (not project root) after the move → they'd point at tests/quickice/... which does NOT exist. Fixed by going one level deeper: `parent.parent` → `resolve().parents[2]`, `parents[1]` → `parents[2]`, and `parent` (sys.path) → `resolve().parent.parent`. The 3 files the research flagged (constants, gromacs, triclinic_pbc) needed the e2e_export_helpers sys.path fix; triclinic_pbc ALSO needed its em.mdp skipif path fixed.
- **SLOW full-suite baseline SKIPPED (hang avoidance):** The full ~1854-test suite deadlocks at the C level in this headless environment (GUI/Qt/VTK event loops; `DISPLAY=localhost:11.0` forwarded X11, `QT_QPA_PLATFORM` unset). `pytest --timeout=120` (SIGALRM) cannot interrupt C-level/Qt-event-loop hangs, so pytest-timeout is ineffective. A previous executor attempt HUNG FOR ~1 HOUR on this and was killed. Phase 48.1-14 SUMMARY documented the same trap and skipped the full suite there too. The FAST gates (global collect count + path-stripped test-ID diff) + the targeted scancode subset run (306 tests, 3.28s) are the load-bearing invariants for this pure-reorg plan (no production code, no test logic changes — just file moves + path fixes). The SLOW full-suite gate is deferred to plan 48.2-03's final task (phase-level slow gate). `/tmp/opencode/phase48_2_before_summary.txt` remains 0 bytes — documenting the skip.
- **em.mdp skipif path fix is CRITICAL:** `test_scancode_bugs_triclinic_pbc.py:550` has `@pytest.mark.skipif(not (Path(__file__).parent / "em.mdp").exists(), ...)`. After the move, `Path(__file__).parent` is `tests/scancode/` where `em.mdp` does NOT exist (it's at `tests/em.mdp`). Without the fix, `TestCRIT01GromppDryRun::test_grompp_on_sH_hydrate_export` would flip from RUNNING to SKIPPED. Since gmx IS available here, this would change the skip count (a behavioral regression for a pure-move plan). Fixed: `Path(__file__).parent` → `Path(__file__).resolve().parent.parent` (reaches tests/ root where em.mdp lives). Verified: the test RAN (appears in the warnings summary with the genice2 DeprecationWarning, confirming it executed, not skipped).
- **Reused on-disk before-snapshots (Task 1 pre-done):** The previous executor attempt already captured the 3 FAST before-snapshots before hanging on the SLOW suite. These files were valid on disk: `phase48_2_before_count.txt` (1854 tests collected in 1.47s), `phase48_2_before_ids.txt` (114789 bytes, sorted test IDs), `phase48_2_before_scancode_subset.txt` (306 passed, 3 warnings in 3.61s). Reused them as the before-baseline — did NOT re-run them (would re-trigger the SLOW hang for the summary). The SLOW `phase48_2_before_summary.txt` (0 bytes) documents the skip.
