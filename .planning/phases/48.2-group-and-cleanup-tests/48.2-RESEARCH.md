# Phase 48.2: Group and cleanup tests — Research

**Researched:** 2026-07-22
**Domain:** pytest test-suite reorganization / hygiene (internal, no new deps)
**Confidence:** HIGH (move mechanics empirically verified in-tree; LOW for full-suite runtime numbers)

## Summary

Phase 48.2 is a pure test-tree reorganization. No new libraries, no source-code changes, no new
tests. Success = the same 1847 tests collect and pass after the move/dedup/rename. There is no
upstream `CONTEXT.md` (no locked decisions) — the researcher has full discretion within the phase
description, which itself marks the e2e grouping as **optional**.

The test tree is a pytest-9 suite with **default discovery** (no `pytest.ini`/`pyproject.toml`).
`tests/__init__.py` exists (making `tests` a Python package), and there is a **project-root
`conftest.py`** (`/share/home/nglokwan/quickice/conftest.py`, registers the `slow` marker).
`tests/conftest.py` holds the real-GenIce2 **module-scoped** fixtures + the `run_quickice` helper
+ the `gmx_skipif` marker. `tests/test_output/conftest.py` holds **synthetic function-scoped**
fixtures (distinct names — no overlap with root). `e2e_export_helpers.py` (non-collected helper)
lives at `tests/` root and is imported by ~31 test files via an explicit per-file
`sys.path.insert(0, str(Path(__file__).parent))` hack.

**The single most important mechanic (verified empirically):** test files import two things that
are sensitive to file location — (1) `from tests.conftest import run_quickice/gmx_skipif` is an
**absolute package import** that works from ANY subdirectory (confirmed: `tests/test_cli/` files
already do this); (2) `from e2e_export_helpers import ...` is a **rootdir-relative top-level
import** that ONLY works because each consuming file does `sys.path.insert(0, str(Path(__file__).parent))`
at the top. **Moving a file that uses pattern (2) into a subdir requires changing that line from
`Path(__file__).parent` → `Path(__file__).resolve().parent.parent`** (one-line edit). This is the
established subdir pattern already used by `tests/test_cli/test_mixed_cage_cli.py` and
`tests/test_output/test_gromacs_export_chain.py`. I built a throwaway probe in `tests/scancode/`
(both with and without `__init__.py`) and confirmed collection + the exact import pattern works,
then cleaned it up — suite returned to exactly 1847.

**Primary recommendation:** Do the **scancode move** (17 files → `tests/scancode/`, 3 one-line
edits) and the **CLI move** (3 files → `tests/test_cli/`, 1 file with 2 edits) — both high-value,
low-risk. **Skip the e2e grouping** in this phase (42 files, ~32 one-line edits, fuzzy category
boundaries, low marginal value after scancode+CLI already remove 20 files from root) — or split it
into its own phase. Do the **rename + scaffolding drop** (drop `phase_48_1_` prefixes, fix `sH`/`sh`
casing) as a final wave. **Defer fixture dedup** to a follow-up unless quick wins are confirmed
identical (see Findings §5).

## Findings

### 1. Move mechanics & safety (pytest 9, default discovery) — HIGH confidence, empirically verified

- **Subdirectory auto-discovery:** pytest collects test files in subdirs automatically. No
  `__init__.py` is *required* in a subdir for collection. Proven by `tests/test_cli/` (4 test files,
  **NO `__init__.py`**, NO `conftest.py`, all currently pass) and by my throwaway probe. **However**
  `tests/test_output/` DOES have both `__init__.py` and `conftest.py`. The precedent is split (see
  Open Questions).
- **`tests.conftest` import (fixtures + `run_quickice` + `gmx_skipif`):** This is an **absolute
  package import** (`from tests.conftest import run_quickice`). It works because `tests/__init__.py`
  exists AND the project root is on `sys.path` (via `PYTHONPATH=/share/home/nglokwan/quickice` per
  `setup.sh`, and pytest's rootdir insertion). **Verified:** `tests/test_cli/test_mixed_cage_cli.py`
  line 52 already does `from tests.conftest import gmx_skipif` from a subdir. → **Moving a file into a
  subdir does NOT break `tests.conftest` imports. No edit needed for these.**
- **`e2e_export_helpers` import (the BREAK RISK):** `tests/e2e_export_helpers.py` is imported as a
  **top-level module** (`from e2e_export_helpers import (...)`), NOT as `tests.e2e_export_helpers`.
  This works ONLY because each consuming file has an explicit
  `sys.path.insert(0, str(Path(__file__).parent))` at the top (inserts `tests/`). After moving a file
  into a subdir, `Path(__file__).parent` becomes the *subdir* (e.g. `tests/scancode/`), so the import
  **breaks**. **Fix (one line):** change to `sys.path.insert(0, str(Path(__file__).resolve().parent.parent))`
  (inserts `tests/`). This is exactly the pattern `tests/test_cli/test_mixed_cage_cli.py:49-50` and
  `tests/test_output/test_gromacs_export_chain.py:46-47` already use.
  - **Cleaner alternative:** since `tests/__init__.py` exists, `from tests.e2e_export_helpers import ...`
    works as an absolute import from anywhere (verified: `python -c "from tests import e2e_export_helpers"`
    succeeds with project root on path). The plan MAY prefer switching moved files to this absolute
    form and dropping the `sys.path` hack entirely — but that's a larger per-file edit and changes
    the established convention. **Recommendation: use the minimal `.parent.parent` fix to match the
    existing `test_cli/` + `test_output/` convention.**
- **Conftest cascade:** `tests/conftest.py` (root) fixtures are visible to ALL tests including
  subdirs (cascade downward). `tests/test_output/conftest.py` fixtures are visible ONLY to
  `tests/test_output/` (and its subdirs). A subdir MAY add its own `conftest.py` to extend/override.
  For `tests/scancode/` and `tests/test_cli/`, **no new conftest is needed** — root conftest
  cascades down and scancode/cli tests use only `run_quickice`/`gmx_skipif`/`e2e_export_helpers`
  (none require a subdir conftest).
- **No cross-imports between test modules** except ONE: `tests/_phase_48_1_capture_baseline.py:76`
  does `from test_phase_48_1_gro_top_byte_equivalence import (...)`. Renaming that target test
  breaks the helper (see Findings §7). All other test modules import only `quickice.*`, stdlib,
  `tests.conftest`, and `e2e_export_helpers` — no `from test_X import`.

### 5. Fixture dedup — MEDIUM confidence (candidates identified; content diff not done)

  `simple_interface`) from `test_output/conftest.py` UP to `tests/conftest.py` (root) so ALL tests
  share them, then delete the local re-definitions IF byte-identical. **CAUTION:** each local
  `simple_candidate` may differ subtly (different atom counts/positions). The plan MUST diff each
  local def against the shared one before deleting — only delete if identical. Hoisting is safe
  (function-scoped, no GenIce2); deleting locals requires verified equivalence.

  setup) — **do NOT remove without comparing**. Flag, don't auto-dedup.
- **Per-file module-scoped e2e fixtures (~30+):** each e2e file defines its own module-scoped
  GenIce2 structure for its specific scenario. These are **NOT duplicates** (each is a distinct
  structure). Keep local. **Dedup must NOT merge these** (would cause wrong structures or, if
  unified, could change GenIce2 call counts — though note: STATE.md says module-scoped fixtures
  amortize GenIce2, and merging to one shared fixture would *reduce* calls, not duplicate them).
- **Net:** fixture dedup has 1 real candidate (hoist `simple_candidate`/`simple_interface` to root)
  but requires careful per-file content verification. **Recommend: defer to a follow-up unless the
  planner wants a quick-win task that diffs-then-deletes only confirmed-identical locals.**
