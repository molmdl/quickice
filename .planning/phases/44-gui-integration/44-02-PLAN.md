### Part F â€” Pitfall 6 mitigation (restrict custom to one cage)

Pitfall 6 (verified by research): if the user selects the custom guest in 2 cages, `HydrateConfig.__post_init__` (types.py:711-720) raises `ValueError: Duplicate guest_residue_name 'MOL' across custom cage assignments`. `main_window.py:742,748` call `get_configuration()` with NO try/except â†’ uncaught crash. **An engine fix to relax Pitfall 6 is OUT OF SCOPE (42-01 design decision â€” the _H hydrate path does not disambiguate residue names).**

Implement the recommended mitigation (research Pitfall 1, option 1): when the user selects "Custom: {residue}" in a second cage, auto-clear the FIRST cage back to a built-in ("ch4"). This is the simplest, zero-crash approach and still satisfies GUI-06 (mixed 1-custom + 1-builtin works perfectly â€” verified: `has_custom_assignment=True`, no error).

Implement this in `_on_guest_changed` (`hydrate_panel.py:409-413`). The cleanest approach: after the existing ff-label + info-display + configuration_changed updates, scan `_cage_guest_combos` for cages whose `currentData()` equals `self._custom_guest['guest_type']` (when `_custom_guest is not None`); if more than one cage has the custom guest, reset all but the most-recently-changed one back to index 0 (first built-in, usually ch4). 

Since `_on_guest_changed` does not receive the changed combo, the simplest robust implementation is: connect each per-cage combo's `currentIndexChanged` to a small lambda that passes the cage_key (modify the `_rebuild_cage_rows` connect line to `combo.currentIndexChanged.connect(lambda _idx, ck=cage_key: self._on_cage_guest_changed(ck))`), and add a new `_on_cage_guest_changed(self, cage_key)` method that:
1. Calls `self._enforce_single_custom_cage(cage_key)` (the mitigation).
2. Calls `self._update_ff_label()` + `self._update_info_display()`.
3. Emits `configuration_changed`.

`_enforce_single_custom_cage(self, changed_cage_key)`:
- If `self._custom_guest is None`: return (no custom loaded).
- Find all cages whose `combo.currentData() == self._custom_guest['guest_type']`.
- If count <= 1: return.
- For every such cage EXCEPT `changed_cage_key`, set `combo.setCurrentIndex(0)` (the first built-in item, typically ch4). This silently clears the duplicate. (Do NOT emit a dialog â€” the auto-clear IS the feedback; the user sees the previous cage revert to ch4.)

Keep the existing `_on_guest_changed` method (hydrate_panel.py:409) for any legacy callers, but route the per-cage combo changes through the new `_on_cage_guest_changed` so the mitigation runs. **Test:** verify with `test_pitfall6_auto_clears_second_cage` (Task 2).
