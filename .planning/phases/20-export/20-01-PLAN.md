**1. compute_mw_position(o_pos, h1_pos, h2_pos):**
- Takes three numpy arrays (3,) — oxygen, hydrogen1, hydrogen2 positions in nm
- Computes MW = O + α*(H1-O) + α*(H2-O) where α = 0.13458335 (TIP4P-ICE specific, matching tip4p-ice.itp virtual_sites3 directive)
- Returns numpy array (3,) — MW position in nm
- Define TIP4P_ICE_ALPHA = 0.13458335 as a module-level constant (add near top of file after imports)

**2. write_interface_gro_file(iface: InterfaceStructure, filepath: str):**
- Accept InterfaceStructure (not Candidate like the existing write_gro_file)
- Title line: `f"Ice/water interface ({iface.mode}) exported by QuickIce"`
- Total atoms: (ice_nmolecules + water_nmolecules) * 4 (all molecules output as 4-atom TIP4P-ICE)
- ICE MOLECULES LOOP (mol_idx 0..ice_nmolecules-1):
  - base_idx = mol_idx * 3 (ice uses 3 atoms per molecule: O, H, H)
  - Get o_pos, h1_pos, h2_pos from iface.positions[base_idx], [base_idx+1], [base_idx+2]
  - Compute mw_pos = compute_mw_position(o_pos, h1_pos, h2_pos)
  - res_num = mol_idx + 1 (continuous numbering starting at 1)
  - Write 4 atoms: OW (o_pos), HW1 (h1_pos), HW2 (h2_pos), MW (mw_pos)
  - Use EXACT same .gro format string as existing write_gro_file: `{res_num:5d}SOL  {atom_name:>4s}{atom_num:5d}{x:8.3f}{y:8.3f}{z:8.3f}\n`
- WATER MOLECULES LOOP (mol_idx 0..water_nmolecules-1):
  - base_idx = iface.ice_atom_count + mol_idx * 4 (water starts after ice atoms, 4 atoms per molecule)
  - res_num = iface.ice_nmolecules + mol_idx + 1 (continue numbering after ice)
  - Write 4 atoms: OW, HW1, HW2, MW — positions come directly from iface.positions (no MW computation needed, already in data)
  - Use atom names ["OW", "HW1", "HW2", "MW"] for each water molecule
- Box vectors: same triclinic format as existing write_gro_file, from iface.cell

**3. write_interface_top_file(iface: InterfaceStructure, filepath: str):**
- Accept InterfaceStructure (not Candidate)
- Header: `"; Generated by QuickIce\n; TIP4P-ICE water model\n; Ice/water interface structure\n\n"`
- [defaults], [atomtypes], [moleculetype], [atoms], [settles], [virtual_sites3], [exclusions] — IDENTICAL to existing write_top_file (copy exactly)
- [system] name: `f"Ice/water interface ({iface.mode}) exported by QuickIce"`
- [molecules]: SINGLE combined count `f"SOL          {iface.ice_nmolecules + iface.water_nmolecules}\n"` — NOT separate entries for ice and water

**CRITICAL — Anti-patterns to avoid:**
- Do NOT split molecule count into separate SOL_ice / SOL_water entries (CONTEXT explicitly overrides EXP-02)
- Do NOT add chain identifiers (no chain A/B)
- Do NOT normalize InterfaceStructure's internal representation — normalization is ONLY at export time
- Do NOT use α values from other TIP4P variants — TIP4P-ICE uses exactly 0.13458335
- Ice atom stride is 3 (O, H, H), NOT 4 — water atom stride is 4 (OW, HW1, HW2, MW)
