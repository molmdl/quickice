Create tests/test_e2e_ion_export.py with 3 test classes.

**Scenario 7: Ion from Interface Export** (class `TestIonFromInterface`)
- Uses `interface_slab` fixture from conftest.py
- Build ion structure: `ion = _insert_ions(interface_slab, concentration=0.15)` (helper from e2e_export_helpers.py)
- Generate ion.itp: `from quickice.structure_generation.gromacs_ion_export import write_ion_itp; write_ion_itp(tmp_path / "ion.itp", ion.na_count, ion.cl_count)`
- Calls `write_ion_gro_file(ion, gro_path)` and `write_ion_top_file(ion, top_path)`
- Validates ALL 7 checks:
  1. GRO residue names: "SOL" first, then "NA", then "CL" — no interleaving
  2. GRO atom count: header matches actual lines (ice*4 + water*4 + na_count + cl_count)
  3. TOP [molecules]: `{"SOL": ice+water, "NA": na_count, "CL": cl_count}`
  4. TOP #include: `["tip4p-ice.itp", "ion.itp"]`
  5. ITP ion.itp exists (generated) and has [ moleculetype ]
  6. ITP tip4p-ice.itp exists in quickice/data/ and has [ moleculetype ]
  7. Atom conservation: total atoms match; na_count == cl_count (charge neutrality)

**Scenario 8: Ion from Custom Export** (class `TestIonFromCustom`)
- Uses `interface_slab` fixture from conftest.py
- Build chain: `custom = _insert_custom_molecules(interface_slab, n_molecules=3)`, then `ion = _insert_ions(custom, concentration=0.15)`
- Generate ion.itp: `write_ion_itp(tmp_path / "ion.itp", ion.na_count, ion.cl_count)`
- Calls `write_ion_gro_file(ion, gro_path)` and `write_ion_top_file(ion, top_path)`
- Validates ALL 7 checks:
  1. GRO residue names: "SOL" first, then "ETOH" (custom), then "NA", then "CL"
  2. GRO atom count: header matches actual lines
  3. TOP [molecules]: `{"SOL": ice+water, "ETOH": 3, "NA": na_count, "CL": cl_count}`
  4. TOP #include: `["tip4p-ice.itp", "etoh.itp", "ion.itp"]`
  5. ITP files validated: etoh.itp (in data/custom/) and ion.itp (generated)
  6. Custom molecule count preserved: `ion.custom_molecule_count > 0`
  7. Atom conservation + charge neutrality

**Scenario 9: Ion from Solute Export** (class `TestIonFromSolute`)
- Uses `interface_slab` fixture from conftest.py
- Build chain: `solute = _insert_solutes(interface_slab, solute_type='CH4', concentration=0.3)`, then `ion = _insert_ions_from_solute(solute, concentration=0.15)` — MUST use _solute_to_ion_source workaround
- Register registry: `registry.register_liquid_solute("CH4")` before insert_solutes
- Generate ion.itp: `write_ion_itp(tmp_path / "ion.itp", ion.na_count, ion.cl_count)`
- Calls `write_ion_gro_file(ion, gro_path)` and `write_ion_top_file(ion, top_path)`
- Validates ALL 7 checks:
  1. GRO residue names: "SOL" first, then "CH4_L" (solute), then "NA", then "CL"
  2. GRO atom count: header matches actual lines
  3. TOP [molecules]: `{"SOL": ice+water, "CH4_L": solute.n_molecules, "NA": na_count, "CL": cl_count}`
  4. TOP #include: `["tip4p-ice.itp", "ch4_liquid.itp", "ion.itp"]`
  5. ITP files validated: ch4_liquid.itp (in data/) and ion.itp (generated)
  6. Solute info preserved: `ion.solute_type == "CH4"`, `ion.solute_n_molecules > 0`
  7. Atom conservation + charge neutrality

**CRITICAL:** The `_solute_to_ion_source()` workaround is MANDATORY for Scenario 9. Without it, IonInserter raises `AttributeError: 'InterfaceStructure' object has no attribute 'solute_type'`. This is a known P0 bug (BUG I5). The workaround copies solute attributes to the interface_structure before passing to IonInserter.

**IMPORTANT:** Ion from Custom (Scenario 8) does NOT need the workaround — IonInserter can read custom molecule attrs directly from CustomMoleculeStructure. Only solute→ion needs the workaround.

**IMPORTANT:** ion.itp is GENERATED by `write_ion_itp()`, not copied from data/ like other ITPs. It must be generated to the tmp_path directory before write_ion_top_file references it. The write_ion_top_file #include "ion.itp" expects it to exist in the same directory as the .top file.

2. `test_ch4_hydrate_itp` — Path: quickice/data/ch4_hydrate.itp. Exists, has [ moleculetype ].

6. `test_etoh_itp` — Path: quickice/data/custom/etoh.itp. Exists, has [ moleculetype ].
7. `test_generated_ion_itp` — Use write_ion_itp() to generate ion.itp to tmp_path. Verify it has [ moleculetype ] and contains both NA and CL sections.
8. `test_no_duplicate_moleculetype_names` — Parse moleculetype names from tip4p-ice.itp, ch4_hydrate.itp, ch4_liquid.itp. Verify they're all different (no "SOL" appearing in both tip4p-ice.itp and ch4_hydrate.itp moleculetype sections).

**Implementation:**
- Import: `import quickice; DATA_DIR = Path(quickice.__file__).parent / "data"`
- Import: `from quickice.structure_generation.gromacs_ion_export import write_ion_itp`

- For test 8, use a simple regex `r'\[ moleculetype \]\s*\n\s*(\w+)'` or parse the line after [ moleculetype ] to extract the moleculetype name

**Why this test matters:** If an ITP file is missing or corrupted, ALL exports that reference it will produce invalid GROMACS files. This catches data directory issues early.
