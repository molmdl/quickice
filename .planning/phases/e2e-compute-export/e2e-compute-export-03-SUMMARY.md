## Key Findings

1. **Custom molecule residue name is MOL, not ETOH** â€” The etoh.itp bundled file defines its moleculetype as "MOL" (not "ETOH"). The GRO residue name and TOP molecule name are both "MOL".

2. **IonInserter doesn't propagate CustomMoleculeStructure custom molecule data** â€” `CustomMoleculeStructure` stores custom molecules in shared `positions`/`atom_names` arrays with `molecule_index` entries of `mol_type="custom"`, NOT in separate `custom_molecule_positions`/`custom_molecule_atom_names` attributes. The IonInserter's `getattr` fallback returned `None`, causing `write_ion_gro_file` to skip custom molecules entirely. Fixed by extracting custom molecule data from molecule_index when `custom_molecule_positions` is None.

3. **SOL count changes after ion insertion** â€” When water molecules are replaced by ions, the IonStructure's molecule_index has fewer water entries. Using the original interface's `ice_nmolecules + water_nmolecules` gives incorrect SOL count in TOP [molecules].

4. **BUG I5 workaround works correctly** â€” The `_insert_ions_from_solute()` helper applies `_solute_to_ion_source()` which copies solute attributes to the interface_structure, preventing the AttributeError on InterfaceStructure.solute_type.

5. **No duplicate moleculetype names** â€” tip4p-ice.itp has "SOL", ch4_hydrate.itp has "CH4_H", ch4_liquid.itp has "CH4_LIQ" â€” all unique, no GROMACS naming conflicts.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] Fixed IonInserter custom molecule propagation**

- **Found during:** Task 1 test execution (TestIonFromCustom)
- **Issue:** IonInserter.replace_water_with_ions() uses `getattr(structure, 'custom_molecule_positions', None)` which returns None for CustomMoleculeStructure (it doesn't have this attribute). Custom molecules were completely absent from exported .gro files.
- **Fix:** Added extraction logic: when `custom_molecule_positions` is None but `custom_molecule_count > 0` and structure has `molecule_index` with `mol_type="custom"` entries, build `custom_molecule_positions` and `custom_molecule_atom_names` from the shared positions/atom_names arrays. Also extract `moleculetype_name` and `itp_path` from CustomMoleculeStructure.
- **Files modified:** quickice/structure_generation/ion_inserter.py
- **Commit:** 28a1adb

**2. [Rule 1 - Bug] Fixed SOL count assertion using interface counts**

- **Found during:** Task 1 test execution (TestIonFromInterface)
- **Issue:** Test used `interface_slab.ice_nmolecules + interface_slab.water_nmolecules` for expected SOL count, but ion insertion replaces water molecules, reducing the count.
- **Fix:** Changed to `sum(1 for m in ion.molecule_index if m.mol_type in ("ice", "water"))` which gives the actual SOL count after ion insertion.
- **Files modified:** tests/test_e2e_ion_export.py (all 3 test classes)
- **Commit:** 28a1adb

**3. [Rule 1 - Bug] Adjusted custom molecule residue name from ETOH to MOL**

- **Found during:** Task 1 test execution (TestIonFromCustom)
- **Issue:** Plan specified "ETOH" as the custom molecule residue name, but etoh.itp actually defines moleculetype name as "MOL".
- **Fix:** Changed test assertions from "ETOH" to "MOL" to match actual ITP file content.
- **Files modified:** tests/test_e2e_ion_export.py
- **Commit:** 28a1adb
