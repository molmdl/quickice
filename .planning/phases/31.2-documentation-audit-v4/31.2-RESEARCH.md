## Common Pitfalls

### Pitfall 1: Inconsistent Tab Numbering
**What goes wrong:** gui-guide.md references Tab 2 as Interface, but v4.0 has Tab 2 as Hydrate Config
**Why it happens:** v4.0 added Hydrate tab BEFORE Interface tab, shifting Interface to Tab 3
**How to avoid:** 
1. Check main_window.py lines 165-168 for authoritative tab order
2. Check help_dialog.py lines 100-109 for workflow reference
3. Update ALL tab references in gui-guide.md consistently
**Warning signs:** "Tab 2" appearing without context in documentation

### Pitfall 2: Missing Feature Documentation
**What goes wrong:** v4.0 features (hydrate generation, ion insertion) undocumented
**Why it happens:** Documentation written for v3.0, not updated after v4.0 features added
**How to avoid:**
1. Add new section for Hydrate Config (Tab 2) in gui-guide.md
2. Add new section for Ion Insertion (Tab 4) in gui-guide.md
3. Update README.md feature list to include hydrate/ion
**Warning signs:** Users don't know v4.0 features exist

### Pitfall 3: Inconsistent Tooltip Coverage
**What goes wrong:** hydrate_panel.py has zero tooltips while ion_panel.py and interface_panel.py have comprehensive tooltips
**Why it happens:** hydrate_panel.py implemented later, tooltip addition overlooked
**How to avoid:** Add tooltips to all input controls in hydrate_panel.py following ion_panel.py pattern
**Warning signs:** Hovering over controls in Hydrate tab shows no help text

### Pitfall 4: Screenshot References Without Files
**What goes wrong:** Documentation references screenshots that don't exist
**Why it happens:** Documentation written before screenshots captured
**How to avoid:** Either capture screenshots OR add placeholder text "TODO: Add screenshot" with clear instructions
**Warning signs:** Image tags with missing src or broken image links
