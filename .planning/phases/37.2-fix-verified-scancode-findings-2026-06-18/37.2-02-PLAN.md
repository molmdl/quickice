<objective>
Fix overlap tree gap: custom molecule atoms are invisible to solute_inserter and custom_molecule_inserter overlap checking, allowing silently wrong simulation results (overlapping molecules).

Purpose: In multi-step pipelines (Interface→Custom→Solute→Ion), custom molecules placed by the CustomMoleculeInserter are not included in the SoluteInserter's overlap tree. This means solutes can be placed overlapping existing custom molecules — a silently wrong simulation. The same gap exists in CustomMoleculeInserter itself (custom molecules from prior steps are invisible).

Output: Both _build_existing_atoms_tree methods add custom_molecule_positions to the overlap checking set.
</objective>

  <name>Task 1: Add custom_molecule_positions to solute_inserter._build_existing_atoms_tree</name>
  <files>quickice/structure_generation/solute_inserter.py</files>
  <action>
In solute_inserter.py, the `_build_existing_atoms_tree` method (lines ~318-352) adds ice atoms, water atoms, and guest atoms to existing_positions. It does NOT add custom_molecule_positions from the structure. This means when a solute inserter receives a structure that already has custom molecules (e.g., from the Custom→Solute workflow), those custom atoms are invisible to overlap checking.

After the guest atoms loop (after line ~348), add a section for custom molecule atoms:

```python
# Add custom molecule atoms (from prior pipeline steps)
custom_molecule_positions = getattr(structure, 'custom_molecule_positions', None)
if custom_molecule_positions is not None and len(custom_molecule_positions) > 0:
    custom_atom_names = getattr(structure, 'custom_molecule_atom_names', None)
    for i, pos in enumerate(custom_molecule_positions):
        # Custom molecules don't have MW virtual sites, but check anyway
        if custom_atom_names is not None and i < len(custom_atom_names):
            if custom_atom_names[i] != "MW":
                existing_positions.append(pos)
        else:
            existing_positions.append(pos)
```

Use `getattr` with None default for backward compatibility (structures from older pipeline steps may not have custom_molecule_positions attribute). Follow the same pattern as existing ice/water/guest loops.
