## Decisions Made

1. **getattr with None default** — Backward-compatible access to custom_molecule_positions; structures from older pipeline steps (or fresh interfaces without custom molecules) won't have this attribute
2. **MW virtual site exclusion** — Applied to custom molecule atoms even though custom molecules typically don't have MW sites; defensive pattern matching existing ice/water/guest code
3. **Same loop pattern** — Identical enumerate+getattr pattern in both inserters for consistency and maintainability
