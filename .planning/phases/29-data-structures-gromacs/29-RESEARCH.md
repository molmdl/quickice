## Summary

This phase establishes data structures for handling variable atoms-per-molecule (ions=1, water=4, CH4=5, THF=12+) and multi-molecule GROMACS export. Key findings:

1. **CO2 + TIP4P-ice:** TraPPE-small IS compatible with TIP4P-ice and has been used in hydrate studies. The "Settle complexity" refers to water model constraints, not incompatibility. Can proceed with CO2 implementation.

2. **CH4/THF force fields:** GAFF/GAFF2 recommended over TraPPE for guest molecules when using TIP4P-ice water, based on literature showing GAFF provides better alkane-water interaction predictions.

3. **GenIce2 API:** Supports guest molecules via `--guest 14=me` (cage type) and `--anion 3=Cl` (ion replacement). Partial occupancy format: `co2*0.6+me*0.4`.

4. **GROMACS multi-moleculetype:** Standard approach uses `#include` directives to include separate .itp files per molecule type. Confirmed by GROMACS 2025 documentation.

**Primary recommendation:** Use tuple-based molecule indexing with separate type tracking. Include molecule type in tuple for downstream code simplification. GROMACS export uses main .top with `#include` for each molecule .itp.

## Architecture Patterns

### Recommended Data Structure: Molecule Index with Type Tracking

**Pattern:** Use separate molecule index list with per-molecule metadata

```python
from dataclasses import dataclass
from typing import TypedDict

class MoleculeType(TypedDict):
    """Enumeration of supported molecule types."""
    ICE: str       # 3 atoms: O, H, H (from GenIce)
    WATER: str    # 4 atoms: OW, HW1, HW2, MW (TIP4P)
    NA: str       # 1 atom: Na+
    CL: str       # 1 atom: Cl-
    CH4: str      # 5 atoms: C + 4H
    THF: str      # 12 atoms: C4O + 8H
    CO2: str      # 3 atoms: C + 2O
    H2: str       # 2 atoms: H + H

@dataclass
class MoleculeIndex:
    """Tracks molecule position in atom array.
    
    Attributes:
        start_idx: First atom index in positions array
        count: Number of atoms in this molecule
        mol_type: Molecule type string ('ice', 'water', 'na', 'ch4', etc.)
    """
    start_idx: int
    count: int
    mol_type: str
```

**Why separate type tracking:** Including molecule type in the tuple (as CONTEXT suggests) simplifies downstream code that needs to:
- Apply different rendering styles per molecule type
- Export different .itp files per molecule type
- Calculate properties per molecule type

**Usage pattern:**
```python
# positions: (N_atoms, 3) numpy array
# molecule_index: List[MoleculeIndex]

def get_molecule_positions(mol_idx: int) -> np.ndarray:
    """Get positions for specific molecule."""
    mol = molecule_index[mol_idx]
    return positions[mol.start_idx : mol.start_idx + mol.count]
```

### Anti-Patterns to Avoid

1. **Single global atoms_per_molecule:** The current codebase uses a single `atoms_per_molecule` parameter everywhere. This MUST change to per-molecule tracking.

2. **Hardcoded molecule counts in .top:** Don't hardcode SOL 512 - use dynamic [molecules] section based on molecule_index.

3. **Single moleculetype assumption:** Existing gromacs_writer.py assumes single SOL type. Multi-molecule export needs separate [moleculetype] sections.

## Don't Hand-Roll

Problems that look simple but have existing solutions:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| GROMACS topology format | Custom parser | GROMACS 2025 docs | Standard format is well-documented, #include pattern is standard |
| Guest molecule insertion | Custom algorithm | GenIce2 CLI/API | GenIce2 has --guest, --anion options already |
| Cage detection | Custom graph algorithm | genice2-cage plugin | PyPI package `genice2-cage` provides cage detection |
| Ion placement | Manual positioning | GenIce2 --anion option | GenIce2 supports ion replacement |

**Key insight:** GenIce2 already supports hydrate generation with guest molecules via `--guest` and `--anion` CLI options. The QuickIce wrapper should leverage this rather than reimplementing.

## Common Pitfalls

### Pitfall 1: Global atoms_per_molecule Assumption
**What goes wrong:** Code that iterates `for mol_idx in range(nmolecules): base = mol_idx * 3` breaks with ions (1 atom) or CH4 (5 atoms).
**Why it happens:** Current codebase hardcodes 3 (ice) or 4 (water) everywhere.
**How to avoid:** Use molecule_index list with per-molecule start_idx and count.
**Warning signs:** Any hardcoded `* 3` or `* 4` pattern when handling mixed molecule types.

### Pitfall 2: Single Molecule Type in GROMACS Export
**What goes wrong:** write_top_file() only handles SOL moleculetype.
**Why it happens:** Original design assumed single water type.
**How to avoid:** Refactor to support multiple moleculetypes with #include pattern.

### Pitfall 3: GenIce2 Guest Format Confusion
**What goes wrong:** Wrong format for guest specification leads to GenIce2 errors.
**Why it happens:** GenIce2 uses specific syntax: `--guest 12=co2*0.6+me*0.4`
**How to avoid:** Use correct syntax from GenIce2 documentation:
- Cage type: `--guest 14=me` (fill type 14 cages with methane)
- Partial occupancy: `--guest 12=co2*0.6+me*0.4` (60% CO2, 40% methane)

### Pitfall 4: Force Field Mixing Incompatibility
**What goes wrong:** TIP4P-ice + TraPPE guest gives poor hydration free energies.
**Why it happens:** TraPPE parameterized for vapor-liquid, not water interactions.
**How to avoid:** Use GAFF/GAFF2 for guest molecules when using TIP4P-ice water.

class MoleculeInfo:

    mol_type: str

def build_molecule_index(positions: np.ndarray, atom_types: list[str]) -> list[MoleculeInfo]:
    """Build molecule index from positions and atom types.
    
    Args:
        positions: (N_atoms, 3) position array
        atom_types: List of atom type strings ['O', 'H', 'H', 'C', 'H', ...]
    
    Returns:
        List of MoleculeInfo objects for each molecule

            molecule_index.append(MoleculeInfo(i, 3, 'ice'))
            i += 3

        else:
            i += 1
    
    return molecule_index

### GROMACS Multi-Moleculetype Topology

```python
# Source: GROMACS 2025 documentation - topology file format
# Main .top file structure

TOPOLOGY_TEMPLATE = '''; Generated by QuickIce
; TIP4P-ICE water model with guest molecules

[ defaults ]
; nbfunc  comb-rule  gen-pairs  fudgeLJ  fudgeQQ
1               2               yes             0.5     0.8333

; Include molecule definitions
#include "water.itp"
#include "na.itp"
#include "ch4.itp"

[ system ]
Hydrate structure exported by QuickIce

[ molecules ]
; Compound    #mols
SOL          {water_count}
NA           {na_count}
CH4          {ch4_count}
'''

def write_multi_molecule_top(molecule_index: list[MoleculeInfo], filepath: str):
    """Write GROMACS topology with multiple moleculetypes."""
    # Count molecules by type
    counts = {}
    for mol in molecule_index:
        counts[mol.mol_type] = counts.get(mol.mol_type, 0) + 1
    
    # Write main topology with molecule counts
    content = TOPOLOGY_TEMPLATE.format(
        water_count=counts.get('water', 0),
        na_count=counts.get('na', 0),
        ch4_count=counts.get('ch4', 0)
    )
    
    with open(filepath, 'w') as f:
        f.write(content)
```

## Open Questions

1. **CO2 implementation complexity**
   - What we know: TraPPE-small works with TIP4P-ice in hydrate studies
   - What's unclear: Whether Settle constraints cause practical issues in GROMACS
   - Recommendation: Include CO2 support but note in UI that it's "experimental" until tested

2. **GenIce2 Python API vs CLI**
   - What we know: CLI supports --guest, --anion options
   - What's unclear: Whether Python API provides same functionality cleanly
   - Recommendation: Use CLI for Phase 31-32, investigate Python API for potential integration

3. **Guest occupancy distribution algorithm**
   - What we know: GenIce2 supports partial occupancy via `co2*0.6+me*0.4` syntax
   - What's unclear: How to evenly distribute across cages when user specifies percentage
   - Recommendation: Let GenIce2 handle distribution, pass occupancy percentage directly
