1. Add `atom_labels` field to each entry in the GUEST_MOLECULES dict (lines 117-132):
   - ch4: `{"atom_labels": ["C", "H", "H", "H", "H"]}` — matches all-atom methane from GenIce2
   - thf: `{"atom_labels": ["O", "CA", "CA", "CB", "CB", "H", "H", "H", "H", "H", "H", "H", "H"]}` — matches THF from GenIce2 (13 atoms)
   
   NOTE: atom_labels represents the canonical atom name sequence for ONE molecule of this guest type, as produced by GenIce2. This is used for metadata-driven identification in `_build_molecule_index` (Plan 02). The first atom label is the "signature" atom used for quick identification.

2. Extend HydrateConfig dataclass (lines 342-393) with these new fields:
   ```python
   guest_name: str = ""              # Display name (auto-populated from GUEST_MOLECULES for built-in types)
   guest_atom_labels: list[str] = field(default_factory=list)  # Atom name sequence for one molecule
   guest_atom_count: int = 0         # Number of atoms per guest molecule
   guest_itp_path: str = ""         # Path to guest ITP file (for custom guests, Phase 40)
   ```

3. Update `HydrateConfig.__post_init__` to auto-populate metadata for built-in guest types:
   - After validating `guest_type in GUEST_MOLECULES`:
   - If `guest_name` is empty, set it from `GUEST_MOLECULES[self.guest_type]["name"]`
   - If `guest_atom_labels` is empty, set it from `GUEST_MOLECULES[self.guest_type]["atom_labels"]`
   - If `guest_atom_count` is 0, set it from `GUEST_MOLECULES[self.guest_type]["atoms"]`
   - This means built-in types auto-populate; custom types (Phase 40) must provide these explicitly
   - Do NOT auto-populate guest_itp_path (only relevant for custom guests)

4. Update `HydrateConfig.from_dict` to pass through the new fields if present in the input dict (with defaults for backward compatibility):
   ```python
   guest_name=d.get("guest_name", ""),
   guest_atom_labels=d.get("guest_atom_labels", []),
   guest_atom_count=d.get("guest_atom_count", 0),
   guest_itp_path=d.get("guest_itp_path", ""),
   ```

5. Create test file `tests/test_hydrate_config_metadata.py` with tests:
   - Test that built-in ch4 config auto-populates guest_name="Methane", guest_atom_labels=["C","H","H","H","H"], guest_atom_count=5
   - Test that built-in thf config auto-populates correctly
   - Test that explicit guest_name overrides auto-population
   - Test that explicit guest_atom_labels overrides auto-population
   - Test that explicit guest_atom_count overrides auto-population
   - Test from_dict with and without new fields (backward compatibility)
