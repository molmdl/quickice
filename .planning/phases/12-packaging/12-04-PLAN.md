<task type="auto">
  <name>Create Windows build GitHub Actions workflow</name>
  <files>.github/workflows/build-windows.yml</files>
  <action>
    Create GitHub Actions workflow file for building Windows executable.
    
    This workflow is NOT executed in Phase 12 - it's provided for users who want to build on Windows themselves.
    
    **Workflow content:**
    ```yaml
    name: Build Windows Executable
    
    on:
      workflow_dispatch:  # Manual trigger only
    
    jobs:
      build:
        runs-on: windows-latest
        
        steps:
          - name: Checkout code
            uses: actions/checkout@v4
          
          - name: Setup Miniconda
            uses: conda-incubator/setup-miniconda@v3
            with:
              environment-file: environment-build.yml
              activate-environment: quickice-build
          
          - name: Build executable
            shell: bash -el {0}
            run: |
              pyinstaller --name quickice-gui \
                          --onedir \
                          --noconsole \
                          --clean \
                          quickice/gui/__main__.py
          
          - name: Copy documentation and licenses
            shell: bash
            run: |
              mkdir -p dist/package
              cp -r dist/quickice-gui dist/package/
              cp -r dist/_internal dist/package/
              cp README.md dist/package/
              cp LICENSE dist/package/
              mkdir -p dist/package/docs
              cp docs/gui-guide.md dist/package/docs/
              cp -r licenses dist/package/
          
          - name: Create distribution
            shell: bash
            run: |
              cd dist
              7z a -tzip quickice-v2.0.0-windows-x86_64.zip package/
              mv quickice-v2.0.0-windows-x86_64.zip ../
          
          - name: Upload artifact
            uses: actions/upload-artifact@v4
            with:
              name: windows-executable
              path: quickice-v2.0.0-windows-x86_64.zip
    ```
    
    **Important notes:**
    - Uses `environment-build.yml` (NOT environment.yml or env_dev.yml)
    - Uses `--noconsole` for Windows (same as `--windowed` on Linux/macOS)
    - Uses 7z for creating zip on Windows (tar not available by default)
    - Entry point: `quickice/gui/__main__.py`
    - Output name: `quickice-gui.exe` on Windows
    
    Create file: `.github/workflows/build-windows.yml`
  </action>
