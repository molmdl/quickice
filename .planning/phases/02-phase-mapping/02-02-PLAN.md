<feature>
  <name>Ice Phase Lookup</name>
  <files>
    quickice/phase_mapping/lookup.py
    tests/test_phase_mapping.py
  </files>
  <behavior>
Given temperature (K) and pressure (MPa), return the correct ice polymorph.

Test cases:
- (273, 0) → ice_ih (atmospheric, near melting point)
- (250, 100) → ice_ih (normal conditions)
- (300, 2500) → ice_vii (high pressure, high temp)
- (100, 1500) → ice_vi (high pressure, low temp)
- (260, 300) → ice_iii (moderate pressure)
- (260, 500) → ice_v (moderate-high pressure)
- (500, 500) → UnknownPhaseError (outside known regions)
- (50, 10000) → ice_viii (very high pressure, low temp)

Returns dict with:
- phase_id: str (e.g., "ice_ih")
- phase_name: str (e.g., "Ice Ih")
- density: float (g/cm³)
- temperature: float (input, K)
- pressure: float (input, MPa)
  </behavior>
  <implementation>
Create IcePhaseLookup class:
- __init__: Load JSON data on initialization
- lookup(temperature, pressure): Check each phase's boundaries, return matching phase

Implementation approach (from RESEARCH.md):
- Iterate through phases ordered by specificity (high pressure phases first)
- Check if T,P falls within each phase's boundaries
- Return first match or raise UnknownPhaseError

Key insight from RESEARCH.md: Check phases in order of specificity - high pressure phases (VII, VIII, VI) before low pressure (Ih) because boundaries may overlap.
  </implementation>
</feature>
