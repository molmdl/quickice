**MW-01 (line 455):** Change `write_ice_gro_file()` to use `wrap_molecules_into_box` instead of `wrap_positions_into_box`. The ice candidate has no molecule_index attribute, so construct one from `nmol`:
```python
from quickice.structure_generation.types import MoleculeIndex

# Build molecule_index for molecule-aware wrapping
# NOTE: count=3 here (OW, HW1, HW2) is the WRAPPING set — MW is computed AFTER
# wrapping from these 3 atoms, so MW is not included in the molecule_index for wrapping.
# This is distinct from WATER_ATOMS_PER_MOLECULE=4 (total including MW) used elsewhere.
ice_molecule_index = [
    MoleculeIndex(start_idx=i * 3, count=3, mol_type="ice")
    for i in range(nmol)
]
wrapped_positions = wrap_molecules_into_box(candidate.positions, ice_molecule_index, candidate.cell)
```
Replace the existing line `wrapped_positions = wrap_positions_into_box(candidate.positions, candidate.cell)`. Note: `MoleculeIndex` is already imported at the top of the file (used by other writers). If not already imported, add the import.

**DEFLT-01 (lines 1710-1712, 2099-2101, 2595-2597):** Change fudgeLJ=0.0, fudgeQQ=0.0 to fudgeLJ=0.5, fudgeQQ=0.8333 in three multi-molecule TOP writers:

The simple writers (write_top_file line 529, write_interface_top_file line 961, write_hydrate_top_file line 1237) already use 0.5/0.8333 — leave them unchanged. The fix standardizes ALL writers to Amber forcefield defaults. For TIP4P-ICE water-only systems, fudgeLJ value is irrelevant (no 1-4 pairs), making 0.5 safe for all contexts. For GAFF2 systems (THF, CH4, custom molecules), 0.5/0.8333 is the correct Amber standard.
