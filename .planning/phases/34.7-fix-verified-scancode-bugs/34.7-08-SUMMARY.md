# Phase 34.7 Plan 08: Fix GUEST-01 Latent Design Concern Summary

**One-liner:** Remove dead CO2 code, add explicit guest_type parameter to count_guest_atoms and propagate through _detect_guest_atoms in all 3 mode files

## What Was Done

### Code Fix (commit 6d04262)

1. **Removed dead CO2 handler** from `molecule_utils.py` (lines 102-104):
   - `if first_atom == "C" and any(a == 'O' for a in sample[:3]): return 3` was unreachable
   - The earlier block at lines 84-90 intercepted any C/CA/CB-first molecule with O atoms, returning 13 (THF)
   - This dead code could never execute regardless of input

2. **Added `guest_type` parameter** to `count_guest_atoms()`:
   - Signature: `def count_guest_atoms(atom_names: list[str], start: int, guest_type: str | None = None) -> int:`
   - When `guest_type == "ch4"`: returns `CH4_ATOMS_PER_MOLECULE` (5)
   - When `guest_type == "thf"`: returns `THF_ATOMS_PER_MOLECULE` (13)
   - When `guest_type is None`: falls back to existing heuristic (backward compatible)

3. **Added module constants** in `molecule_utils.py`:
   - `CH4_ATOMS_PER_MOLECULE: int = 5`
   - `THF_ATOMS_PER_MOLECULE: int = 13`

4. **Updated callers** in all 3 mode files:
   - `_detect_guest_atoms()` now accepts `guest_type` parameter, passes to `count_guest_atoms()`
   - `_count_guest_molecules()` now accepts `guest_type` parameter, passes to `count_guest_atoms()`
   - `assemble_slab()`, `assemble_pocket()`, `assemble_piece()` extract guest_type from `candidate.metadata["guest_type_counts"]` and pass it

5. **Added documentation comment** on heuristic fallback path warning that it assumes C+O = THF and would need updating if new guest types are added

### Regression Tests (commit d25383f)

42 tests in `tests/test_guest_atom_counting.py`:
- `TestGuestAtomConstants`: 2 tests (CH4=5, THF=13 constants)
- `TestExplicitGuestType`: 5 tests (ch4→5, thf→13, realistic atoms, unknown type, fallback)
- `TestHeuristicFallback`: 7 tests (CH4 C-first/H-first, THF O-first/C-first, Me, H2, water, None=default)
- `TestDeadCO2CodeRemoved`: 2 tests (C+O+O never returns 3, returns 13 instead)
- `TestDetectGuestAtomsWithGuestType`: 9 tests (3 modes × CH4/THF/None)
- `TestCountGuestMoleculesWithGuestType`: 13 tests (3 modes × CH4/THF/2CH4/heuristic)
- `TestEdgeCases`: 4 tests (empty list, beyond length, Me atom)
