<objective>
Fix KDTree rebuild inefficiency in ion inserter (TREE-01 MEDIUM). Move the cKDTree rebuild inside the conditional block where an ion is successfully placed, instead of rebuilding on every loop iteration — including iterations where overlap rejection occurs.

Purpose: The current code rebuilds the ion_tree on every iteration of the placement loop (line 380), even when the previous iteration rejected the placement (overlap detected). This is wasteful — the tree only needs updating when the ion_positions list actually changes (i.e., when an ion is successfully placed). The custom_molecule_inserter and solute_inserter are acceptable (they rebuild only after each placed molecule, which IS every iteration for them), but the ion inserter has a rejection loop.

Output: Optimized ion_inserter.py with conditional KDTree rebuild.
</objective>

  <action>
In `quickice/structure_generation/ion_inserter.py`, the `replace_water_with_ions()` method has a loop (starting around line 363) that iterates over selected water positions to place ions. Currently at line 379-380:

```python
if ion_positions:
    ion_tree = cKDTree(np.array(ion_positions))
    min_ion_dist = ion_tree.query(water_pos, k=1)[0]
    if min_ion_dist < MIN_SEPARATION:
        # Too close to other ions - skip
        continue
```

The problem: `ion_tree = cKDTree(np.array(ion_positions))` is rebuilt on EVERY iteration, even when the previous iteration rejected the placement (the `continue` branch). Since no ion was added to `ion_positions`, the tree is unchanged and the rebuild is wasted.

**Fix:** Initialize `ion_tree = None` before the loop. Inside the loop, only build/rebuild the tree when `ion_positions` has changed (i.e., after a successful placement). Before the overlap check, build the tree lazily:

```python
# Before the loop (around line 361-362):
ion_positions = []  # Track placed ion positions
ion_tree = None  # KDTree for ion-ion overlap checking (rebuilt only on successful placement)

# Inside the loop, replace the current block:
# Also check against previously placed ions
if ion_positions:
    ion_tree = cKDTree(np.array(ion_positions))
    min_ion_dist = ion_tree.query(water_pos, k=1)[0]
    if min_ion_dist < MIN_SEPARATION:
        continue

# With the optimized version:
if ion_tree is not None:
    min_ion_dist = ion_tree.query(water_pos, k=1)[0]
    if min_ion_dist < MIN_SEPARATION:
        continue

# Then, AFTER a successful placement (after `ion_positions.append(water_pos)` around line 387):
ion_tree = cKDTree(np.array(ion_positions))
```

The key insight: `ion_tree` starts as None. When the first ion is placed, the tree is built. On subsequent iterations, the tree is only rebuilt when an ion is successfully added to `ion_positions`. If the previous iteration rejected the placement, the tree is reused without rebuilding.

Verify: After the fix, `ion_tree = cKDTree(...)` should appear only ONCE inside the loop — right after `ion_positions.append(water_pos)`. The overlap check should use `if ion_tree is not None:` instead of `if ion_positions:`.

This is a performance optimization only — no correctness change. The functional behavior is identical.
