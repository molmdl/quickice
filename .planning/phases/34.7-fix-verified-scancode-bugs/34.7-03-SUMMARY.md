## Decisions Made
- **ion_tree = None before loop:** Replaces the `if ion_positions:` check which was always True after the first ion, causing unconditional rebuilds. None initialization makes the "no previous ions" case explicit.
- **Strictly-increasing sequence test instead of exact count:** Charge neutrality cleanup may remove excess ions after the placement loop, so `na_count + cl_count` can be less than the number of ions that passed overlap checks. Using strictly-increasing sizes [1, 2, ..., N] as the regression invariant is more robust than counting exact cKDTree calls.
- **100-size threshold to separate existing_atoms_tree from ion_tree calls:** The existing_atoms_tree uses thousands of ice+guest atoms, while ion_tree grows from size 1. A threshold of 100 cleanly separates the two.
