<objective>
Add an optional `custom_guest_info: dict | None = None` parameter to `write_multi_molecule_gro_file` so the GUI hydrate exporter can pass the custom guest residue name (`"MOL_H"`) instead of the writer falling through to `"UNK"`.

Purpose: EXPORT-04 (GRO residue <=5 chars with `_H`). Today the writer's `else` branch does `MOLECULE_TO_GROMACS.get(mol.mol_type, {"res_name": "UNK"})` → `"UNK"` for a custom `mol_type` like `"etoh_e2e"` (gromacs_writer.py:1544). The `.gro` then has `"UNK"` residues while the `.top [molecules]` lists `"MOL_H"` → grompp FATAL. Fix: when `custom_guest_info` maps `mol.mol_type`, use `custom_guest_info["residue_name"]`.

`custom_guest_info` shape (used identically by plans 41-02..41-05): `{"mol_type": str, "residue_name": str, "itp_path": Path}` — `itp_path` unused by this GRO writer (only the TOP writers use it for the atomtypes merge) but kept on the dict for a single consistent API.

Output: new param + tests (custom → `MOL_H`, built-in unchanged).
</objective>
