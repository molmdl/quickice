Create custom_molecule_panel.py following RESEARCH.md Pattern 4 and SolutePanel pattern:

1. Import dependencies:
   - from PySide6.QtWidgets import QWidget, QVBoxLayout, QHBoxLayout, QPushButton, QLabel, QFileDialog, QMessageBox, QComboBox, QLineEdit, QDoubleSpinBox
   - from PySide6.QtCore import Signal
   - from pathlib import Path

2. Create CustomMoleculePanel class:
   - Signals: files_uploaded(bool), generate_requested
   - Attributes: gro_path, itp_path, itp_info, validation_result, placement_mode

3. UI Layout (following CONTEXT.md decisions):
   - **File Upload Section**:
     - Horizontal layout for GRO: [Upload .gro] button + status label (shows filename)
     - Horizontal layout for ITP: [Upload .itp] button + status label (shows filename + atom count)
     - Validation status label (shows ✓ validated or ✗ error details)
   
   - **Molecule Info Section** (shown after validation):
     - Atom count label
     - Residue name label
   
   - **Placement Mode Section**:
     - Dropdown: "Random" | "Custom"
     - Dynamic controls below based on selection
   
   - **Random Mode Controls**:
     - QDoubleSpinBox for molecule count (range 1-1000, default 10)
   
   - **Custom Mode Controls**:
     - Position inputs: QLineEdit for X, Y, Z (3 fields in horizontal layout)
     - Rotation inputs: QDoubleSpinBox for α, β, γ (degrees, 0-360)
     - [Add Position] button
     - Position count label: "Positions added: N"
   
   - **Generate Button**:
     - Enabled only when files validated
     - Text: "Generate Custom Molecules"

4. File upload handlers:
   - _upload_gro(): QFileDialog for .gro, store path, call _validate_files()
   - _upload_itp(): QFileDialog for .itp, parse with parse_itp_file(), store itp_info, call _validate_files()

5. Validation handler:
   - _validate_files(): Call validate_custom_molecule(gro_path, itp_info)
   - If residue_name_mismatch: show QMessageBox dialog asking if ITP residue name should override
   - If valid: enable generate button, emit files_uploaded(True)
   - If invalid: show errors, disable generate button

6. Placement mode handler:
   - _on_placement_mode_changed(): Show/hide appropriate controls
   - Preserve values when switching modes (store in instance variables)

7. Position management:
   - _add_position(): Store current position/rotation values, increment count
   - positions_added list: list of (position, rotation) tuples

Reference: quickice/gui/solute_panel.py for exact UI structure and signal patterns.

  <files>quickice/gui/custom_molecule_worker.py</files>
  <action>
Create custom_molecule_worker.py following SoluteWorker pattern:

1. Import dependencies:
   - from PySide6.QtCore import QObject, Signal
   - from quickice.structure_generation.custom_molecule_inserter import CustomMoleculeInserter, InsertionError
   - from quickice.structure_generation.types import CustomMoleculeConfig, CustomMoleculeStructure, InterfaceStructure

2. Create CustomMoleculeWorker class:
   - Inherits from QObject
   - Signals:
     - finished(CustomMoleculeStructure)
     - error(str)
     - progress(int, str)  # percent, message
   
3. Constructor:
   - __init__(self, config: CustomMoleculeConfig, structure: InterfaceStructure, gro_path: Path, itp_path: Path)
   - Store all parameters

4. run() method:
   - Emit progress(0, "Loading custom molecule...")
   - Parse GRO file: positions, atom_names, cell = parse_gro_file(gro_path)
   - Emit progress(20, "Initializing placement...")
   - Create CustomMoleculeInserter with config
   - Emit progress(40, "Placing molecules...")
   - Based on config.placement_mode:
     - If "random": result = inserter.place_random(structure, config.molecule_count)
     - If "custom": result = inserter.place_custom(structure, config.positions, config.rotations)
   - Emit progress(100, "Complete")
   - Emit finished(result)
   - Catch InsertionError and emit error signal with message

5. Follow SoluteWorker pattern exactly for signal emission and error handling.

Reference: quickice/gui/workers.py or solute_worker.py pattern.
