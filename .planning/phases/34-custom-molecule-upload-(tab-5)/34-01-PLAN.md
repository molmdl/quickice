<task type="auto">
  <name>Task 2: Enhance molecule validator with residue name checking</name>
  <files>quickice/structure_generation/molecule_validator.py</files>
  <action>
Enhance molecule_validator.py following RESEARCH.md Pattern 2:

1. Extend ValidationResult dataclass with new fields:
   - residue_name_mismatch: bool = False
   - gro_residue_name: str | None = None
   - itp_residue_name: str | None = None

2. Create new function `validate_custom_molecule(gro_path: Path, itp_info: ITPMoleculeInfo) -> ValidationResult`
   - Parse GRO file using existing parse_gro_file() to get atom count
   - Extract residue name using new extract_residue_name_from_gro()
   - Compare atom count between GRO and ITP (BLOCKING error if mismatch)
   - Compare residue names between GRO and ITP (NON-BLOCKING warning if mismatch)
   - Check for [ atomtypes ] section in ITP (warning if missing)
   - Return ValidationResult with detailed errors/warnings

3. Validation logic:
   - Atom count mismatch: BLOCKING - add to errors list, is_valid=False
   - Residue name mismatch: NON-BLOCKING - add to warnings, set residue_name_mismatch=True
   - Missing atomtypes: WARNING - add to warnings list

4. Error message format (specific and actionable):
   - Atom count: "Atom count mismatch:\n  GRO file has X atoms\n  ITP file defines Y atoms"
   - Residue name: "Residue name mismatch:\n  GRO file uses 'X'\n  ITP file uses 'Y'\n  User must choose to proceed or re-select files."

Keep existing validate_gro_itp_consistency() function unchanged for backward compatibility.
  </action>
  <verify>
    pytest tests/test_molecule_validator.py -v
    # Should pass all validation tests including residue name mismatch scenarios
  </verify>
  <done>
ValidationResult has residue_name_mismatch, gro_residue_name, itp_residue_name fields. validate_custom_molecule() validates atom count and residue name consistency with specific error messages.
  </done>
