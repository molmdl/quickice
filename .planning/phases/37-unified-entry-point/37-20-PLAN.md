Create `scripts/hydrate-interface-custom-ion.sh` following the project's existing script pattern (see clean-test-output.sh for shebang, set -e, header comment style, flag parsing with `for arg in "$@"; do case`, --help, echo-based progress, summary at end).

The script is ACTUALLY RUNNABLE (not commented out like cli-examples.sh). It parses its own CLI options and constructs/runs `python -m quickice` commands.

Script structure:

1. `#!/bin/bash`
2. `set -e`
3. Header comment (lines 3-30ish):
   ```
   # QuickIce Hydrate-Interface-Custom Molecule-Ion Workflow
   # Generates a hydrate structure, creates ice-water interface,
   # inserts custom molecules and NaCl ions
   #
   # Usage: ./scripts/hydrate-interface-custom-ion.sh [OPTIONS]
   #
   # Required:
   #   --custom-gro PATH    Path to custom molecule .gro file
   #   --custom-itp PATH    Path to custom molecule .itp file
   #
   # Optional:
   #   --ion-conc FLOAT     Ion concentration in mol/L (default: 0.15)
   #   --custom-count INT   Number of custom molecules for random placement (default: 5)
   #   --temperature FLOAT  Temperature in K (default: 260)
   #   --pressure FLOAT     Pressure in MPa (default: 0.1)
   #   --lattice-type TYPE  Hydrate lattice: sI, sII, sH (default: sI)
   #   --guest TYPE         Guest molecule: CH4, THF (default: CH4)
   #   --output DIR         Output directory (default: hydrate-custom-ion-output)
   #   -h, --help           Show this help message
   #
   # Example:
   #   ./scripts/hydrate-interface-custom-ion.sh \
   #     --custom-gro quickice/data/custom/etoh.gro \
   #     --custom-itp quickice/data/custom/etoh.itp \
   #     --ion-conc 0.3
   ```

4. Default variable assignments:
   ```bash
   CUSTOM_GRO=""
   CUSTOM_ITP=""
   ION_CONC=0.15
   CUSTOM_COUNT=5
   TEMPERATURE=260
   PRESSURE=0.1
   LATTICE_TYPE="sI"
   GUEST="CH4"
   OUTPUT_DIR="hydrate-custom-ion-output"
   ```

5. Flag parsing with `for arg in "$@"; do case "$arg" in`:
   - `--custom-gro`) shift; CUSTOM_GRO="$1" ;;
   - `--custom-itp`) shift; CUSTOM_ITP="$1" ;;
   - `--ion-conc`) shift; ION_CONC="$1" ;;
   - `--custom-count`) shift; CUSTOM_COUNT="$1" ;;
   - `--temperature`) shift; TEMPERATURE="$1" ;;
   - `--pressure`) shift; PRESSURE="$1" ;;
   - `--lattice-type`) shift; LATTICE_TYPE="$1" ;;
   - `--guest`) shift; GUEST="$1" ;;
   - `--output`) shift; OUTPUT_DIR="$1" ;;
   - `-h|--help`) echo help text and exit 0 ;;
   - `*)` echo "ERROR: Unknown option: $arg" && exit 1 ;;

6. Validate required arguments:
   ```bash
   if [ -z "$CUSTOM_GRO" ]; then
       echo "ERROR: --custom-gro is required"
       echo "Run with --help for usage information."
       exit 1
   fi
   if [ -z "$CUSTOM_ITP" ]; then
       echo "ERROR: --custom-itp is required"
       echo "Run with --help for usage information."
       exit 1
   fi
   ```

7. Validate files exist:
   ```bash
   if [ ! -f "$CUSTOM_GRO" ]; then
       echo "ERROR: Custom GRO file not found: $CUSTOM_GRO"
       exit 1
   fi
   if [ ! -f "$CUSTOM_ITP" ]; then
       echo "ERROR: Custom ITP file not found: $CUSTOM_ITP"
       exit 1
   fi
   ```

8. Print configuration summary:
   ```bash
   echo "=== QuickIce Hydrate-Interface-Custom Molecule-Ion Workflow ==="
   echo ""
   echo "Configuration:"
   echo "  Temperature:     ${TEMPERATURE} K"
   echo "  Pressure:        ${PRESSURE} MPa"
   echo "  Hydrate lattice: ${LATTICE_TYPE}"
   echo "  Guest molecule:  ${GUEST}"
   echo "  Custom molecule: ${CUSTOM_GRO} / ${CUSTOM_ITP}"
   echo "  Custom count:    ${CUSTOM_COUNT}"
   echo "  Ion concentration: ${ION_CONC} mol/L"
   echo "  Output directory: ${OUTPUT_DIR}"
   echo ""
   ```

9. Construct and run the pipeline command. This uses `python -m quickice` with ALL the flags in a SINGLE command (the CLI pipeline supports chaining all steps in one invocation):
   ```bash
   echo "=== Running QuickIce pipeline ==="
   echo ""
   
   CMD="python -m quickice --hydrate --lattice-type ${LATTICE_TYPE} --guest ${GUEST} --interface --mode slab --box-x 5.0 --box-y 5.0 --box-z 10.0 --ice-thickness 3.0 --water-thickness 4.0 --custom-gro ${CUSTOM_GRO} --custom-itp ${CUSTOM_ITP} --custom-placement random --custom-count ${CUSTOM_COUNT} --ion-concentration ${ION_CONC} --gromacs --no-diagram --output ${OUTPUT_DIR} --temperature ${TEMPERATURE} --pressure ${PRESSURE}"
   
   echo "Command:"
   echo "  ${CMD}"
   echo ""
   
   eval $CMD
   PIPELINE_EXIT=$?
   
   if [ $PIPELINE_EXIT -ne 0 ]; then
       echo ""
       echo "ERROR: Pipeline exited with code ${PIPELINE_EXIT}"
       exit $PIPELINE_EXIT
   fi
   ```

   Note: Use `--temperature` and `--pressure` (long form) for clarity. The CLI parser accepts these in any position. The hydrate + interface flags chain correctly because the CLI pipeline handles the hydrate→interface workflow internally.

10. Print summary at end:
    ```bash
    echo ""
    echo "=== Summary ==="
    echo "Pipeline completed successfully."
    echo ""
    echo "Output files in ${OUTPUT_DIR}/:"
    if [ -d "$OUTPUT_DIR" ]; then
        ls -la "$OUTPUT_DIR"/ 2>/dev/null || echo "  (directory listing not available)"
    fi
    echo ""
    echo "The GROMACS files (.gro, .top, .itp) are ready for simulation."
    echo "Next steps: Run 'gmx grompp -f em.mdp -c <gro_file> -p <top_file>' for energy minimization."
    ```

IMPORTANT:
- The script uses `eval $CMD` to run the python command — this is acceptable for a helper script, not security-critical
- `set -e` ensures the script stops if the pipeline fails
- Use `--gromacs` and `--no-diagram` flags for the workflow (GROMACS output is the point, diagram is unnecessary for batch runs)
- The script does NOT separately run hydrate then interface — the CLI pipeline handles `--hydrate --interface` in a single invocation
- Make the file executable: `chmod +x scripts/hydrate-interface-custom-ion.sh`
