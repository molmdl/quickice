  <action>
Create `quickice/entry.py` with the following structure:

**Helper functions (all at module level, no PySide6 imports at top level):**

1. `_is_pyside6_available() -> bool` — Uses `importlib.util.find_spec('PySide6')`. Returns False on ImportError/ValueError/ModuleNotFoundError. CRITICAL: Never uses `import PySide6` — must use find_spec to avoid Qt crashes in headless.

2. `_has_display() -> bool` — Check platform via `platform.system()`. macOS/Darwin → True. Windows → True. Linux → check DISPLAY, WAYLAND_DISPLAY, QT_QPA_PLATFORM (values 'wayland', 'xcb', 'offscreen' count as True). Return False otherwise.

3. `_has_pipeline_flags(argv: list[str]) -> bool` — Returns True if any arg in argv[1:] starts with '-' AND is not in the router_flags set {'--cli', '--gui', '--help', '--version', '-h', '-V'}. This detects computation flags like -T, --interface, --hydrate, etc.

4. `_get_install_hint() -> str` — If `getattr(sys, 'frozen', False)` (PyInstaller binary) → "Reinstall or report bug — GUI should be included in binary distribution." Otherwise → "See environment.yml for GUI dependencies."

**main(argv: list[str] | None = None) -> int:**

CRITICAL DESIGN: `main()` must accept `argv` parameter so tests can pass args without modifying sys.argv. Default is None → uses sys.argv[1:].

Routing priority (matches CONTEXT.md locked decisions):
1. **No args** (empty argv) → `from quickice.cli.parser import create_parser; parser = create_parser(); parser.print_help(); return 0`
2. **--help/-h** in argv → delegate to `create_parser().parse_args(['--help'])` (exits with 0)
3. **--version/-V** in argv → delegate to `create_parser().parse_args(['--version'])` (exits with 0)
4. **--gui** explicit → check PySide6 via `_is_pyside6_available()`. If missing → `print("Error: --gui requested but PySide6 is not installed.", file=sys.stderr); print(_get_install_hint(), file=sys.stderr); return 1`. If PySide6 OK but no display → `print("Error: --gui requested but no display is available.", file=sys.stderr); return 1`. If both OK → `from quickice.gui.main_window import run_app; run_app(); return 0`.
5. **--cli** explicit → Strip --cli from argv, pass remaining to CLI. `sys.argv = [sys.argv[0]] + remaining; from quickice.main import main as cli_main; return cli_main()`.
6. **Pipeline flags detected** (via `_has_pipeline_flags`) → Strip --cli/--gui from argv if present (list comprehension filter), set `sys.argv = [sys.argv[0]] + clean_argv`, then `from quickice.main import main as cli_main; return cli_main()`.
7. **No pipeline flags, no explicit mode** → `from quickice.cli.parser import create_parser; parser = create_parser(); parser.print_help(); return 0`.

**Pre-parser for --cli/--gui detection:**
Use `argparse.ArgumentParser(add_help=False)` with `--cli` and `--gui` store_true arguments. Call `parse_known_args(argv)` to get `(known, remaining)`. This correctly handles cases like `--cli -T 300 -P 0.1 -N 100`.

**IMPORTANT notes:**
- All imports of quickice.main, quickice.gui.main_window, quickice.cli.parser must be LAZY (inside function branches, not at module top level). This avoids importing PySide6 when not needed.
- `sys.argv` modification: When routing to CLI, set `sys.argv = [sys.argv[0]] + remaining_args`. This is needed because quickice/main.py's `get_arguments()` calls `parser.parse_args()` which reads sys.argv.
- The function returns int exit codes: 0 success, 1 error, 2 for argparse errors (SystemExit).
