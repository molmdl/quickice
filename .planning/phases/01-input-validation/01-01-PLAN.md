## CRITICAL: Setup Workflow

The setup workflow is EXPLICIT and MUST be followed:

1. **ONE TIME ONLY:** `conda env create -f env.yml` — creates conda environment
2. **EVERY NEW SHELL:** `source setup.sh` — activates conda env + sets PYTHONPATH
3. **RUN:** `quickice` — executable script at project root

## DO NOT USE

- `pip install -e .` — NOT NEEDED, PYTHONPATH handles imports
- `pyproject.toml` console_scripts entry points — NOT NEEDED
- Poetry or any other package manager — NOT NEEDED

The `quickice` executable IS a Python script at the project root. There is NO naming conflict between:
- `quickice` (executable file at root)
- `quickice/` (package directory)

These coexist on Unix systems (file vs directory).
