<objective>
Extend `count_guest_atoms` (molecule_utils.py:21) to accept an explicit `guest_atom_count` parameter for custom guests, taking priority over the ch4/thf heuristic. This is the atom-counting fix that prevents the IndexError crash in interface generation.

Purpose: Currently `count_guest_atoms` only knows ch4 (5 atoms) and thf (13 atoms); any custom guest_type falls through to a fragile heuristic that miscounts (returns 1/2/13 randomly for ethanol atoms). When interface modes tile guests using the wrong atom count, accumulated indices overshoot the array → IndexError. An explicit atom_count fixes this for ANY custom guest.

Output: Extended count_guest_atoms signature + test proving correct count for custom ethanol.
</objective>
