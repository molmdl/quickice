## Decisions Made

- **Module-scoped fixture builds BOTH ch4 and thf chains ONCE:** `built_in_chains` returns `dict {"ch4": SimpleNamespace, "thf": SimpleNamespace}` indexed by the `@pytest.mark.parametrize` guest_type. Each chain carries `hydrate` (with `.config`), `iface`, `solute`, `custom`, `ion` — the 4 downstream structures the 4 GUI exporters + 4 CLI branches consume. Amortizes the GenIce2 + inserter calls (~3-5s each) across the 4 parametrized cases per AGENTS.md testing guidance. Uses `tempfile.mkdtemp` for the renamed `cmet.gro`/`cmet.itp` (module-level temp dir, cleaned up in the fixture finalizer via `yield` + `shutil.rmtree`).
- **Inline QFileDialog + QMessageBox patching (NOT the mock_save_dialog fixture):** The plan explicitly warns that `mock_save_dialog` from `tests/test_output/conftest.py` is directory-scoped to `tests/test_output/` and NOT available to root-level test files like this one. Each of the 4 GUI exporters is called under `with patch('quickice.gui.export.QFileDialog.getSaveFileName', return_value=(str(ws/'output.gro'), 'GRO Files (*.gro)')), patch('quickice.gui.export.QMessageBox'):` with `parent_widget=None`. QFileDialog is called ONCE per export flow (single return_value patch suffices — matches 44.1-09/11/13/15). Under `QT_QPA_PLATFORM=offscreen` the unpatched `getSaveFileName` returns `("", "")` → exporter returns `False` without writing files → assertions fail.
- **CLI test uses `_run_export_step` directly (NOT `CLIPipeline.execute`):** The plan allows the export-step-direct approach from plan 17's tests (`tests/test_e2e_custom_guest_cli_grompp.py`). Each of the 4 branches gets a fresh `CLIPipeline` with `_hydrate_result=hydrate` (carries `.config`) + one downstream structure set on the target `_*_result` attribute. `_run_export_step` reads `hydrate_config = getattr(self._hydrate_result, "config", None)` → the built-in `HydrateConfig` → `_build_custom_guest_info` returns `None` → `custom_guest_info=None` → built-in residue name + bundled ITP. No QFileDialog mocking needed (CLI writes files directly to `_output_dir`).
- **File-consistency always runs; grompp conditional on `shutil.which('gmx')`:** Rather than `@gmx_skipif` on the whole test (which would skip file-consistency too when `gmx` is absent), the `_assert_builtin_export` helper ALWAYS runs `assert_itp_completeness` + `assert_gro_top_consistent` and ONLY runs `run_gmx_grompp` when `shutil.which('gmx')` is truthy. This matches the plan's "file-consistency always runs; grompp when gmx available" requirement more precisely than the `@gmx_skipif`-whole-test convention. `gmx` IS on PATH so grompp runs (verified: rc==0).
- **Renamed `cmet.itp` (moleculetype `MOL`) for the custom-molecule inserter:** The `register_custom_molecule` default is `MOL`, but the raw `etoh.itp` moleculetype is `etoh` → `assert_gro_top_consistent` would fail on the `.gro` `MOL` vs `.top` `etoh` name mismatch (not even case-insensitive match: `MOL` ≠ `ETOH`). The rename (`etoh` → `MOL` in `cmet.itp`'s `[ moleculetype ]` line) makes `.top [molecules]` `MOL` match `.gro` residue `MOL` (direct match). This is a TEST-SETUP detail to avoid a PRE-EXISTING mismatch (NOT a 44.1 concern). For the built-in `ch4`/`thf` cage guest, the cage-guest ITP (`ch4_hydrate.itp` / `thf_hydrate.itp`) does NOT collide with `cmet.itp` — no filename collision (unlike the 44.1-13 custom-cage-guest test where both the cage guest and the custom molecule used `etoh.itp`).
- **Key regression assertion:** The `_assert_builtin_export` helper asserts the built-in guest ITP (`ch4_hydrate.itp` / `thf_hydrate.itp`) is staged and contains the built-in residue name (`CH4_H` / `THF_H`, NOT `MOL_H`); the `.top [molecules]` has the built-in residue (NOT `GUE` / `MOL_H`); the `.gro` residues match. This confirms the `hydrate_config=None` default path is the pre-44.1 built-in behavior — the core regression guard.
