## Decisions Made
- **Two keys, not one:** Carried BOTH `guest_descriptors` (full list with cage_key/guest_residue_name/is_custom for downstream exporters) AND `guest_atom_counts` (flat dict for the hot count_guest_atoms path). The dict is the lookup interface modes will read; the list preserves richer info plans 05-07+ may need. Intentional redundancy — the dict is cheap to derive and avoids interface modes re-iterating the list.
- **Defensive copy:** Used `list(self.guest_descriptors)` so the Candidate does not share the HydrateStructure's mutable list (consistent with the project's V-17 "all inserters return NEW structures" discipline; avoids aliasing surprises if either side mutates).
- **Built-ins covered too:** No special-casing for built-in vs custom guests — the generator already populates `guest_descriptors` with correct atom_count for ch4 (5) and thf (13), so the same dict-comprehension works for both. This is why the ch4 regression guard is meaningful: it proves the built-in path is NOT broken by the change.
