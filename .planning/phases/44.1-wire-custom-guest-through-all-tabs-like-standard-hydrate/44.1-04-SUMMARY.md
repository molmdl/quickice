## Decisions Made
- **Verbatim move, behavior preserved:** The function body and docstring are byte-identical to the 42-07 source. Only the module location changes. The `list[dict] | None` return type, the mol_type dedup, the `{guest_residue_name}_H` residue_name, and the `Path(assignment.guest_itp_path)` itp_path are all unchanged — so every existing call site and external importer keeps working.
- **Re-export (not a wrapper):** pipeline.py does `from quickice.output.guest_info import _build_custom_guest_info` (a direct name import) rather than defining a thin wrapper. This means `quickice.cli.pipeline._build_custom_guest_info IS quickice.output.guest_info._build_custom_guest_info` (identity) — no wrapping overhead, and `# noqa: F401` documents the re-export intent so linters don't flag the apparently-unused import. All existing call sites (pipeline.py:917) and external importers (test_pipeline_custom_guest_export.py:34 `from quickice.cli.pipeline import _build_custom_guest_info`) resolve the name unchanged.
- **`git add -f` for the gitignored output file:** `quickice/output/guest_info.py` matches the repo `.gitignore` `output/` pattern (same pre-existing interaction as quickice/output/gromacs_writer.py per STATE [41-02]); staged with `git add -f` so the new tracked file is committed despite the ignore rule.
- **Standalone test function (not a class):** The plan named a single test function `test_build_custom_guest_info_from_guest_info_module`; added it as a standalone function (with local imports of `_build_custom_guest_info` + `HydrateConfig`, matching the file's lazy-import style seen in TestCountGuestAtomsCustomExplicit) rather than a new class, since it is a single focused test for a different concern (the neutral module) from the existing classes (bridge/validation/injection/count_guest_atoms).
