## Decisions Made
- **Config-driven (Option A) over structure-driven (Option B):** Per 44.1-RESEARCH §5, the helper drives `custom_guest_info` from `_build_custom_guest_info(hydrate_config)`, NOT from `structure.guest_descriptors`. Option B (structure-driven) would require ~6 extra plans to propagate `GuestDescriptor` through `InterfaceStructure`/`SoluteStructure`/`IonStructure`/`CustomMoleculeStructure` + the inserters. Option A is safe for the interface path because you can't change the interface without regenerating from the hydrate (no 42-08-style desync). Documented limitation: a stale `_current_hydrate_config` after a hydrate change without interface regen (same class as 42-08; low risk for the 44.1 generate-once-export flow).
- **Built-in detection: structure first, config fallback:** `_detect_builtin_guest_type(structure, ...)` mirrors `quickice.gui.export._detect_guest_type_from_structure` (molecule_index + atom_names scan, heuristic fallback on guest_atom_count/guest_nmolecules). When `structure is None` (tests / config-only) OR detection returns None, fall back to the first built-in `cage_guest_assignment`'s `guest_type`. This handles real exports (structure present -> existing behavior), the IonStructure case (no `interface_structure` ref -> config fallback), AND tests (structure=None -> config fallback).
- **transform_guest_itp REUSED, not reimplemented (AGENTS.md):** The custom path imports and reuses `transform_guest_itp` from `quickice.output.gromacs_writer`. `ci['residue_name']` is already `'{base}_H'` (e.g. `MOL_H`); `removesuffix('_H')` recovers the base (`MOL`) which is passed with `suffix='_H'` to the transform -> `MOL_H`. Mirrors `hydrate_export.py:353-359` passing `descriptor.guest_residue_name` (base). comb-rule=2 is preserved (the transform's Step 1 only comments out `[atomtypes]`, never touches `[defaults]`).
- **Built-in path uses shutil.copy (no transform):** The bundled `{guest_type}_hydrate.itp` is pre-transformed (moleculetype `CH4_H`, [atoms] resname `CH4_H`); `transform_guest_itp` would be idempotent but `shutil.copy` matches the existing `export.py:185-187` built-in behavior exactly, keeping the built-in path byte-identical to today (regression-safe).
- **Presence gate (guest_atom_count<=0 OR guest_nmolecules<=0 -> (None, {})):** Both explicit kwargs (threaded from interface modes per 44.1-05/06/07) and the structure's attrs are consulted (explicit kwargs override). Short-circuits ion-only / ice-only exports without touching the filesystem.
- **itp_files keyed by filename -> full path:** Per the plan spec ("dict[str, str] of staged ITP file names -> paths, for the exporter to report"). The gro/top writers get `custom_guest_info` (list[dict] with `mol_type` + `itp_path`) directly; the exporter derives any `mol_type -> filename` mapping it needs for the `.top` `#include` list from `custom_guest_info` (custom path) or the detected `guest_type` (built-in path).
- **Two private module-level helpers (not inlined):** `_bundled_hydrate_guest_itp_path` and `_detect_builtin_guest_type` are separate named helpers (cleaner, testable) that document the neutral-mirror relationship to the GUI originals. The GUI copies (`quickice.gui.export._get_hydrate_guest_itp_path` / `quickice.gui.hydrate_export._get_hydrate_guest_itp_path` / `quickice.gui.export._detect_guest_type_from_structure`) are candidates for future consolidation onto these neutral helpers.
- **git add -f for guest_info.py:** The file is gitignored (`.gitignore:215:output/` matches `quickice/output/`) but tracked (force-added in 44.1-04). Plain `git add` is refused with "The following paths are ignored by one of your .gitignore files: quickice/output"; `git add -f` is required for modifications, per the plan's critical_constraints. The test file (`tests/test_custom_guest_bridge.py`) is not ignored -> plain `git add`.
