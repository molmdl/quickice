## Decisions Made
- **Piece-only change (no consolidation):** Per the plan, the identical change pattern was applied to piece.py only. slab.py (44.1-05) and pocket.py (44.1-06) already have their own fixes. Consolidating the three `_detect_guest_atoms` functions would be scope creep and risk breaking the other two modes mid-wave.
- **guest_atom_count threaded as kwarg with default None:** Both `_detect_guest_atoms` and `_count_guest_molecules` keep their existing positional args first; `guest_atom_count: int | None = None` is appended so any existing callers (tests, other modes) keep working unchanged. The new value is only populated on the hydrate path where candidate.metadata carries `guest_atom_counts`.
- **Lookup by _guest_type (single source of truth):** `_guest_atom_count = guest_atom_counts.get(_guest_type) if _guest_type else None` reuses the same `_guest_type` already extracted from `guest_type_counts`. This guarantees the atom count matches the identified guest type (no key-mismatch risk) and stays None when there is no guest.
- **Built-in path stays byte-identical:** For ch4/thf, `count_guest_atoms` returns the explicit constant (CH4=5, THF=13) and ignores `guest_atom_count` when `guest_type in ("ch4", "thf", None)` (44.1-03 logic). So threading `guest_atom_count=5` for ch4 changes nothing — verified by `test_assemble_piece_builtin_ch4_regression`.
- **Piece InterfaceConfig uses box_x/y/z (no piece-specific fields):** The plan's probe example `InterfaceConfig(mode='piece', box_x=3.0, box_y=3.0, box_z=8.0, seed=42, ice_thickness=2.0, water_thickness=4.0)` worked as-is — piece mode centers the ice piece in the water box using only box_x/y/z; the slab-only `ice_thickness`/`water_thickness` fields are accepted (defaults 0.0 in InterfaceConfig) but unused by `assemble_piece`. No probe adjustment needed (unlike pocket 44.1-06 which needed pocket_diameter).
