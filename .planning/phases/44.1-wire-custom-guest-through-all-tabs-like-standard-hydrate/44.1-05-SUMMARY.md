## Decisions Made
- **Slab-only change (no consolidation):** Per the plan, the identical ~3-line change was applied to slab.py only. pocket.py and piece.py have their own `_detect_guest_atoms` functions (lines 155-157 and 201-203 read guest_type_counts) and are fixed by separate plans 06 and 07. Consolidating the three helpers would be scope creep and risk breaking the other two modes mid-wave.
- **guest_atom_count threaded as kwarg with default None:** Both `_detect_guest_atoms` and `_count_guest_molecules` keep their existing positional args first; `guest_atom_count: int | None = None` is appended so any existing callers (tests, other modes) keep working unchanged. The new value is only populated on the hydrate path where candidate.metadata carries `guest_atom_counts`.
- **Lookup by _guest_type (single source of truth):** `_guest_atom_count = guest_atom_counts.get(_guest_type) if _guest_type else None` reuses the same `_guest_type` already extracted from `guest_type_counts`. This guarantees the atom count matches the identified guest type (no key-mismatch risk) and stays None when there is no guest.
- **Built-in path stays byte-identical:** For ch4/thf, `count_guest_atoms` returns the explicit constant (CH4=5, THF=13) and ignores `guest_atom_count` when `guest_type in ("ch4", "thf", None)` (44.1-03 logic). So threading `guest_atom_count=5` for ch4 changes nothing — verified by `test_assemble_slab_builtin_ch4_regression`.
