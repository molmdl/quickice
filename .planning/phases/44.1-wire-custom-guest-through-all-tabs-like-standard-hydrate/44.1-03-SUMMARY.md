## Decisions Made
- **Three disjoint paths, one function:** Custom (guest_atom_count not None AND guest_type not in ch4/thf/None) -> return guest_atom_count; built-in (guest_type in ch4/thf) -> return CH4/THF_ATOMS_PER_MOLECULE; heuristic (guest_type=None or unknown) -> pattern matching. The new branch is purely additive — no existing path's behavior changes.
- **Gate uses `not in ("ch4", "thf", None)`:** Explicit enumeration (not `guest_type not in GUEST_MOLECULES`) so the gate stays local to molecule_utils.py with no cross-module import. A custom guest_type like 'etoh_e2e' (or any non-builtin) trips the short-circuit; built-ins and None never do.
- **LAST kwarg placement:** Adding `guest_atom_count` after `guest_type` keeps every positional call site (`count_guest_atoms(atom_names, start, guest_type)`) byte-identical; only callers that opt into the new kwarg are affected. Minimizes diff and regression surface.
- **Heuristic-shown-wrong proof in the test:** The test asserts `first_step_heuristic != 9 OR first_step_none != 9` rather than just `!= 9` for the custom path — this documents WHY the explicit count is required (the heuristic actually miscounts) rather than just asserting the explicit count is correct. If a future heuristic improvement ever returns 9 for ethanol, the test would flag it as "explicit count would be redundant", prompting a re-evaluation rather than silently passing.
