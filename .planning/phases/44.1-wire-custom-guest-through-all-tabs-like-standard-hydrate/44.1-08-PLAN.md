Add a shared `_stage_hydrate_guest_itps` helper to `quickice/output/guest_info.py` that all 4 GUI exporters (and the CLI) call to stage hydrate-guest ITPs in a config-driven way. This DRYs the broken `_detect_guest_type + shutil.copy` pattern duplicated 4× in export.py and mirrors the 42-08 hydrate_export.py:147-359 pattern.

Purpose: All 4 GUI exporters currently use `_detect_guest_type_from_structure` (returns None for custom) + `shutil.copy({guest_type}_hydrate.itp)` (FileNotFoundError for custom). This helper replaces that with a config-driven approach: build `custom_guest_info` from the hydrate config, transform+write custom ITPs, copy built-in `_hydrate.itps`. Returns `(custom_guest_info, itp_files)` so the exporter passes `custom_guest_info` to the gro/top writers AND stages the ITPs.

Output: Shared helper in guest_info.py + unit test.

```python
def _stage_hydrate_guest_itps(output_dir, hydrate_config, structure, guest_atom_count=None, guest_nmolecules=None):
    """Stage hydrate-guest ITPs for any interface-derived exporter (interface,
    solute, custom-molecule, ion). Config-driven — safe for the interface path.

    Mirrors hydrate_export.py:253-316 (42-08) but drives from the hydrate config
    (not the structure's molecule_index, which interface-derived structures
    don't carry the same way as the hydrate structure).

    Returns ``(custom_guest_info, itp_files)`` where:
    - ``custom_guest_info``: list[dict] for the gro/top writers' custom_guest_info
      kwarg, or ``None`` when there are no custom guests (built-in path).
    - ``itp_files``: dict[str, str] of staged ITP file names -> paths, for the
      exporter to report.

    ``guest_atom_count`` / ``guest_nmolecules`` gate presence: if the structure
    has no guest atoms (e.g. ion-only export with no hydrate guests), returns
    (None, {}) without staging any guest ITPs.
    """
```

Implementation (config-driven Option A from the research):
1. Call `_build_custom_guest_info(hydrate_config)` → `custom_guest_info` (list[dict] or None).
2. Gate on guest presence: if `guest_atom_count` and `guest_nmolecules` are both provided and either is 0 (or the structure has no guests), return `(None, {})` — nothing to stage. If the structure carries guests (atom_count > 0 and nmolecules > 0), proceed.
3. If `custom_guest_info` is not None (custom guests): for each `ci` in `custom_guest_info`, use `transform_guest_itp` (import from `quickice.output.gromacs_writer`) to read-transform-write the custom ITP to `output_dir` with the `_H` suffix applied (mirror hydrate_export.py:355). The residue name for the transform is `ci["residue_name"]` (already has `_H` suffix; the base is `removesuffix("_H")`).
4. If `custom_guest_info` is None (built-in path): detect the built-in guest_type from the structure using the EXISTING `_detect_guest_type_from_structure` logic (or `detect_guest_type_from_atoms`), and copy the built-in `{guest_type}_hydrate.itp` to `output_dir` (mirror the existing export.py built-in path). Return `(None, {f"{guest_type}_hydrate.itp": ...})`.
5. Return `(custom_guest_info, itp_files)`.

Use lazy imports for `transform_guest_itp` (inside the function body) to keep guest_info.py free of heavy module-level deps. Import `shutil` and `Path` as needed.

IMPORTANT: Do NOT reimplement `transform_guest_itp` — import and reuse it (AGENTS.md). Preserve comb-rule=2 (the transform already does).
