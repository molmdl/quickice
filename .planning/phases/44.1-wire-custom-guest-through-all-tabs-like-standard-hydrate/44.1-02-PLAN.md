<objective>
Fix `HydrateStructure.to_candidate()` (types.py:1217-1231) to carry `guest_descriptors` and `guest_atom_counts` in the returned Candidate's metadata. This is the root-cause fix for the interface generation IndexError crash — without the atom count, interface modes fall back to the ch4/thf heuristic and miscount custom guest atoms.

Purpose: `to_candidate()` currently drops `guest_descriptors` (which carry `atom_count=9` for ethanol). Interface modes (slab/pocket/piece) then can't identify custom guest atoms → IndexError. This plan threads the descriptors through candidate metadata so plans 05-07 can read them.

Output: Extended to_candidate metadata + test verifying descriptors are carried.
</objective>

Add two new keys (carry guest descriptors so interface modes can identify custom guest atoms precisely via atom_count instead of the ch4/thf heuristic):
```python
metadata={
    "lattice_type": lattice_id,
    "lattice_description": self.lattice_info.description,
    "water_count": self.water_count,
    "guest_count": self.guest_count,
    "guest_type_counts": guest_type_counts,
    "original_hydrate": True,
    # NEW (44.1): carry guest descriptors so interface modes can identify
    # custom guest atoms precisely (atom_count) instead of the ch4/thf heuristic.
    "guest_descriptors": list(self.guest_descriptors),
    # NEW (44.1): fast lookup mol_type -> atom_count for count_guest_atoms.
    "guest_atom_counts": {gd.mol_type: gd.atom_count for gd in self.guest_descriptors},
}
```

`GuestDescriptor` is a dataclass (types.py:469) and `Candidate.metadata` is `dict[str, Any]` (types.py:251), so carrying the list directly is safe. `self.guest_descriptors` is already an attribute of `HydrateStructure` (types.py:1166). For built-in ch4/thf hydrates, `guest_descriptors` is populated by the generator with the correct atom_count (ch4=5, thf=13), so the lookup works for built-ins too.
