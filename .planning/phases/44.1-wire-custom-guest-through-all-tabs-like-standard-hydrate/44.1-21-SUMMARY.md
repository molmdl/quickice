## Decisions Made
- **Called the interface writers DIRECTLY (NOT the GUI exporter):** The plan's step 5 explicitly says `write_interface_gro_file` + `write_interface_top_file` with `custom_guest_info=cgi`. This tests the writer-level contract (custom_guest_info threading → one MOL_H line) in isolation, complementing the 44.1-09 test which exercises the EXPORTER METHOD end-to-end via QFileDialog mocking. Both paths are needed: 44.1-09 proves the exporter wiring; this test proves the 2-cage aggregation semantics at the writer level (the acceptance-test concern). - **Used cage_guest_assignments explicitly (NOT the legacy single-guest_type shim):** The plan's step 1 specifies `HydrateConfig(cage_guest_assignments={...})` with the same etoh_e2e in both. This exercises the Pitfall 6 relaxation (44.1-01) directly — the explicit-dict path (types.py:666-743) is where the relaxed `_seen_residue_name_to_guest_type` check lives. This directly satisfies the plan's step 8 "exactly ONE MOL_H moleculetype entry in [molecules]".
- **Aggregation proof via `mols['MOL_H'] == iface.guest_nmolecules`:** The MOL_H count in `[molecules]` must equal the AGGREGATED `iface.guest_nmolecules` (both cages summed into one number by `assemble_slab`). For sI 1x1x1 with 100% occupancy in both cages: 8 hydrate guests (2 small + 6 large) tiled across the slab → 288 aggregated guest molecules → `MOL_H` count 288 == `iface.guest_nmolecules` 288. This is the "aggregating both cages' molecule counts" requirement, mirroring ch4-in-all-cages which produces one `CH4_H` line with the summed count.
- **@gmx_skipif on the WHOLE test (matches repo convention):** file-consistency + grompp both run when gmx is present, both skipped when absent. gmx IS on PATH so the test runs fully (verified: grompp rc==0, em.tpr produced).
