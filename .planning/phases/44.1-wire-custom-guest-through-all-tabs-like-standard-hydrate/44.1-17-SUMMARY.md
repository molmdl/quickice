## Decisions Made
- **Call-site update split into Task 2 (not Task 1):** The plan's Task 1 action only threads `custom_guest_info=cgi` to the writers; the `copy_itp_files_for_structure(..., hydrate_config=...)` call-site update belongs to Task 2. This keeps the Task 1 commit independently functional (passing an unsupported `hydrate_config` kwarg before Task 2 lands would `TypeError` the existing `_run_export_step` tests). Each task commit is independently green.
- **`cgi` built once before dispatch:** `hydrate_config` + `cgi` are computed once before the writer dispatch and threaded to the 4 non-hydrate branches. The hydrate branch keeps its own local `custom_guest_info` (the working reference, untouched). The minor redundancy (cgi is also computed for ice/hydrate branches) is harmless and matches the plan's specified code.
- **Custom-guest block at the TOP of each branch:** Placed before the existing `_detect_guest_type` built-in logic so custom guests are staged when present. `_detect_guest_type` returns None for custom (mol_type 'etoh_e2e' is not CH4/THF; 9 atoms/mol is between the 5/10 heuristic thresholds), so the built-in block is a safe no-op fallback for custom and unchanged for built-ins.
- **`_run_export_step` direct (not full CLI subprocess):** The plan explicitly allows exercising the export step directly by constructing the structure + calling `_run_export_step`. This directly validates the Task 1+2 threading/staging, distinct from the GUI exporter tests (44.1-09/11/13/15) which test the GUI exporters. A module-scope fixture amortizes the GenIce2 cost.
- **Renamed cmet.itp for the custom-molecule test:** Both the cage guest and the custom molecule use the same source etoh.itp; the renamed cmet.itp (moleculetype 'MOL') avoids the filename collision (the custom-molecule ITP copy would overwrite the transformed cage-guest ITP) and a duplicate `#include` — same workaround as the GUI 44.1-13 test.
- **Absolute guest_gro_path/guest_itp_path in the fixture:** Uses `str(ETOH_GRO)`/`str(ETOH_ITP)` (absolute via e2e_export_helpers `.resolve()`) so ITP staging is CWD-independent, matching `tests/test_cli/test_pipeline_custom_guest_export.py`.
