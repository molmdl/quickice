## Decisions Made
- **Ion built from solute (not interface directly):** The real CLI workflow order is interface -> solute -> ion; the per-branch ion test (44.1-17) inserts ions from the interface directly, which doesn't exercise the solute->ion hand-off. This full-chain test uses `_insert_ions_from_solute` -> `_solute_to_ion_source` (propagates solute attributes onto `solute.interface_structure` via duck-typing) so the ion export must still stage the custom cage-guest ITP after the solute step. This is the integration aspect the plan targets.
- **Dedicated module-scope fixture (not the shared one):** `_solute_to_ion_source` mutates `solute.interface_structure` (duck-typing). To avoid contaminating the per-branch tests' shared `_custom_ethanol_hydrate_and_iface` iface, this file owns its own `_full_chain_structures` fixture where the mutation is part of the fixture's established state. No cross-file contamination.
- **Per-step isolated subdirs:** `_run_export_step` only exports the MOST downstream structure, so to export each intermediate step each call must have only that step's structure set (others None). Separate `tmp_path/<step_name>/` subdirs also keep each grompp's ITP set isolated (no cross-step overwrite of etoh.itp/tip4p-ice.itp). Subdirs are `mkdir`'d before `_run_export_step` since `execute()` (which creates `_output_dir`) is bypassed.
- **Shared `_assert_step_output` helper:** Factored the per-step assertions (file existence + MOL_H staging + itp completeness + gro/top consistency + grompp + molecules/residues checks) into one helper with optional `extra_mols`/`extra_itp` so the 3 step exports share one assertion path; ion species (NA/CL) asserted only when actually placed (`na_count`/`cl_count` > 0), matching the per-branch ion test's conditional assertion.
- **No custom-molecule export step:** The plan's explicit chain is hydrate -> interface -> solute -> ion (no custom-molecule link); the plan marks custom-molecule as "if applicable" (optional). The custom-molecule branch is already covered by the per-branch test (44.1-17) with the renamed cmet.itp collision workaround; adding it here would complicate the chain without adding integration coverage.
