<task type="auto">
  <name>Task 2: Create .gro/.itp creation guide</name>
  <files>docs/gro-itp-guide.md</files>
  <action>
Create comprehensive user guide for creating valid .gro and .itp files for custom molecules.

**Target length:** 400-500 lines

**Structure:**

1. **Introduction**
   - Purpose: Guide for preparing custom molecule files for QuickIce
   - Prerequisites: GROMACS installed, molecule structure available

2. **GRO File Format**
   - Structure overview (title, atom count, coordinates, box)
   - Fixed-width column format (residue name cols 6-10, atom name cols 11-15, coords cols 21-45)
   - Units (coordinates in nm, box in nm)
   - Example GRO file with annotations

3. **Creating GRO Files**
   - Method 1: Using GROMACS editconf (convert from PDB)
   - Method 2: Manual creation (for simple molecules)
   - Method 3: Using molecular editors (Avogadro, PyMOL)
   - Common errors and fixes

4. **ITP File Format**
   - Required sections: [ atomtypes ], [ moleculetype ], [ atoms ]
   - Optional sections: [ bonds ], [ angles ], [ dihedrals ], [ pairs ]
   - Format specifications for each section
   - Example ITP file with annotations

5. **Creating ITP Files**
   - Method 1: Using ACPYPE (recommended for GAFF/GAFF2)
   - Method 2: Using CHARMM-GUI (for CHARMM force field)
   - Method 3: Manual creation (for expert users)
   - Force field parameter sources (literature, databases)

6. **Validation**
   - What QuickIce checks (atom count, residue name, required sections)
   - How to validate files before uploading (gmx check)
   - Common validation errors and fixes

7. **Examples**
   - Example 1: Small molecule (ethanol)
   - Example 2: Drug molecule (aspirin)
   - Example 3: Polymer fragment

8. **Troubleshooting**
   - "Atom count mismatch" error
   - "Residue name not found" error
   - "Missing [ atomtypes ] section" error
   - GROMACS simulation failures after export

9. **References**
   - GROMACS manual (file formats)
   - ACPYPE documentation
   - CHARMM-GUI website
   - Force field databases (ATB, LigParGen)

**Style:** Tutorial-focused with step-by-step examples, not exhaustive reference.
