# Phase 34.1: Ion Source Dropdown - Context

**Gathered:** 2026-05-05
**Status:** Ready for planning

<domain>
## Phase Boundary

User can select ion insertion source (Interface/Custom/Solute) and receives warning when charge is not neutralized. This phase delivers source selection UI and charge warnings. It does NOT implement automatic charge neutralization (user warned only).

</domain>

<decisions>
## Implementation Decisions

### Dropdown UI placement & behavior (follow Interface tab pattern)

- **Placement:** Top of Ion tab left column, before other controls (matches Interface tab's Source dropdown placement)
- **Labels:** Simple text labels - "Interface", "Custom Molecule", "Solute" (no status in dropdown text, no "Interface (no data)")
- **Immediate UI effect:** Selection changes control enablement right away (like Interface tab disables candidate dropdown for Hydrate source)
- **Generate-time application:** Source is used when "Generate Ions" is clicked (not immediate load)
- **HelpIcon:** Include tooltip explaining each source type (like Interface tab's source dropdown)

### Charge warning display (OpenCode's discretion with rationale)

- **Format:** Inline label near Generate button (non-blocking warning, not a modal dialog)
- **Timing:** Show when Custom Molecule source selected AND user has configured ions (real-time feedback without premature warnings)
- **Information:** Show total charge value - "Charge: +2 (non-neutral)" (user understands exact imbalance magnitude)
- **Persistence:** Warning stays visible as long as charge imbalance exists (cannot dismiss - ensures user doesn't forget the issue)
- **Rationale for decisions:**
  - Inline label (vs dialog): Does not block workflow, allows user to proceed if they choose
  - Real-time timing: Avoids surprise warnings at generate time, gives early feedback
  - Show charge value: Helps user understand imbalance magnitude, not just that there IS an imbalance
  - Persistent: Prevents user from dismissing and forgetting about a real issue

### Source switching behavior

- **Ion configuration:** Preserve concentration and ion types when switching sources (only structure data changes, not ion config)
- **3D viewer:** Clear immediately when source changes (show placeholder - clean slate for new source)
- **Info panel:** Log source change (e.g., "Source switched to Custom Molecule") - rationale: helps user track state changes in workflow
- **Generate button:** Follow Interface pattern - enable/disable based on source data availability (like Interface tab enables generate for Hydrate but disables for Ice if no candidates)

### Default source selection (follow Interface pattern)

- **Default source:** "Interface" (index 0 in dropdown) - matches current behavior, users already familiar with this flow
- **No data state:** If Interface has no structure, show "No source available" message in area, disable Generate button, show tooltip "Generate Interface structure first"
- **No auto-selection:** Do NOT auto-select another source if default is unavailable (user must generate valid source structure first)
- **Empty state UI:** Guidance message + disabled Generate button (matches Interface tab's behavior when no ice candidates exist)

### OpenCode's Discretion

- Exact warning label styling (color: red/orange, icon placement, font weight)
- Exact log message format for source changes (e.g., "Source: Interface → Custom Molecule" vs "Source switched to Custom Molecule")
- Tooltip text details for source dropdown
- Whether to check all sources on tab open or just default source

</decisions>

<specifics>
## Specific Ideas

- **Pattern reference:** Follow Interface tab's source selection pattern exactly (see `quickice/gui/interface_panel.py` lines 254-273, 532-564)
- **Behavior consistency:** Interface tab accepts Ice/Hydrate sources; Ion tab accepts Interface/Custom/Solute sources with same UI patterns
- **Charge warning scope:** Warning is informational only - does NOT prevent generation (user can choose to proceed with non-neutral system)
- **Workflow expectation:** User generates Interface structure first (Tab 3), then goes to Ion tab (Tab 6) - default source matches this typical flow

</specifics>

<deferred>
## Deferred Ideas

- **Automatic charge neutralization:** Explicitly out of scope per ROADMAP.md - user is warned but not auto-corrected (future phase if needed)

</deferred>
