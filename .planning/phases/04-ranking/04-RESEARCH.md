## Summary

This phase implements scoring and ranking of ice structure candidates generated by Phase 3. Candidates are ranked by relevance to user's temperature (T) and pressure (P) conditions using three scoring components: energy-based ranking, density matching, and diversity scoring.

**Key findings:**
- GenIce provides geometric data (positions, cell) but NOT energy calculations
- Energy can be estimated from hydrogen bond network quality using O-O distance heuristics
- Density scoring uses deviation from expected phase density at user's T,P
- Diversity rewards different polymorphs (different phase_ids)
- Combined ranking uses weighted sum with documented weights

**Primary recommendation:** Use weighted sum scoring with equal weights (1:1:1) as baseline, normalized components, and explicit breakdown for transparency. Energy estimation based on O-O distance deviation from ideal 0.276nm is practical for "vibe" approach.

### Scoring Component Patterns

#### Energy Score (Lower is Better)

```python
# Energy estimation from H-bond geometry
# Ideal O-O distance in ice: ~0.276 nm
# Typical H-bond energy: ~20 kJ/mol

def estimate_energy_score(positions, atom_names, cell):
    """Estimate energy from O-O distance distribution.
    
    Lower score = better structure (more ideal H-bonds)
    """
    # Extract oxygen positions
    o_indices = [i for i, name in enumerate(atom_names) if name == 'O']
    o_positions = positions[o_indices]
    
    # Calculate O-O distances with PBC
    oo_distances = calculate_oo_distances(o_positions, cell)
    
    # Ideal distance
    ideal_dist = 0.276  # nm
    
    # Score based on mean absolute deviation from ideal
    mean_deviation = np.mean(np.abs(oo_distances - ideal_dist))
    
    # Convert to pseudo-energy (arbitrary units for ranking)
    # Larger deviation = higher energy = worse score
    energy_score = mean_deviation * 100  # Scale for visibility
    
    return energy_score
```

#### Density Match Score (Lower Deviation is Better)

```python
def density_match_score(candidate, target_temperature, target_pressure):
    """Score how well candidate density matches expected at T,P.
    
    Returns: score where 0 = perfect match, higher = worse match
    """
    # Target density from Phase 2 metadata
    target_density = candidate.metadata.get('density')
    
    # Calculate actual density from cell volume
    cell_volume_nm3 = np.abs(np.linalg.det(candidate.cell))
    mass_per_molecule = 18.01528  # g/mol (water)
    avogadro = 6.022e23
    
    # Convert: nm^3 to cm^3, molecules to moles
    actual_density = (
        candidate.nmolecules * mass_per_molecule / 
        (avogadro * cell_volume_nm3 * 1e-21)
    )
    
    # Score: absolute deviation (g/cm³)
    density_deviation = abs(actual_density - target_density)
    
    return density_deviation
```

#### Diversity Score (Different Polymorphs Rewarded)

```python
def diversity_score(candidate, all_candidates):
    """Score based on phase diversity relative to other candidates.
    
    Different polymorphs should have higher diversity scores
    to encourage representation of different ice phases.
    """
    # Get all unique phase_ids in candidate set
    unique_phases = set(c.phase_id for c in all_candidates)
    
    # Count how many candidates share this phase_id
    same_phase_count = sum(
        1 for c in all_candidates if c.phase_id == candidate.phase_id
    )
    
    # Score: inversely proportional to frequency
    # Less common phases get higher scores
    diversity_score = 1.0 / same_phase_count
    
    return diversity_score
```

#### Combined Ranking Score

```python
def calculate_combined_score(energy_score, density_score, diversity_score,
                             energy_weight=1.0, density_weight=1.0, 
                             diversity_weight=1.0):
    """Combine scores into single ranking value.
    
    Lower combined score = better candidate
    """
    # Normalize each score to 0-1 range (for weighting)
    # Use min-max normalization based on candidate set
    
    # Combined: weighted sum
    combined = (
        energy_weight * energy_score +
        density_weight * density_score +
        diversity_weight * diversity_score
    )
    
    return combined
```

### Anti-Patterns to Avoid

1. **Don't use raw energy values directly** - Different phases have different baseline energies; normalize for fair comparison

2. **Don't ignore diversity completely** - Without diversity scoring, all candidates may be same polymorph

3. **Don't score density without considering rounding** - GenIce rounds nmolecules to complete unit cells; actual vs requested affects density

4. **Don't combine scores without normalization** - Energy (~10), density (~0.01), diversity (~0.1) have different scales; normalize before combining

## Don't Hand-Roll

Problems with existing solutions:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Distance with PBC | Custom loop | scipy.spatial.cdist with cell | Handles periodic boundary conditions correctly |
| Normalization | Hardcoded ranges | Min-max per candidate set | Adapts to actual score distribution |
| Phase lookup | Parse JSON each time | Already cached in Phase 2 | Phase info already in metadata |
| Sorting | Custom sort | sorted() with key | Python built-in is efficient |

## Common Pitfalls

### Pitfall 1: O-O Distance Calculation Without PBC

**What goes wrong:** Distances calculated naively ignore periodic images, giving incorrect distances for molecules near cell boundaries.

**Why it happens:** Ice structures wrap around; O atoms at cell edge are close to O atoms on opposite edge.

**How to avoid:** Use minimum image convention or scipy's pbc-aware distance calculation.

**Warning signs:** Very large or very small distances in distribution; mean O-O distance far from 0.276nm for bulk ice.

### Pitfall 2: Treating All Candidates as Same Phase

**What goes wrong:** Diversity score assumes all candidates from same generation run have same phase_id.

**Why it happens:** Phase 3 generates candidates for ONE phase at a time.

**How to avoid:** Note that for single-phase generation, diversity scoring may need adjustment OR this only matters when generating candidates across multiple T,P conditions (different phases).

**Clarification:** From context - Phase 2 maps specific T,P to ONE phase, so Phase 3 generates candidates for ONE phase. Diversity scoring may need to measure structural variety within phase (different H-bond orientations) not polymorph diversity. This should use seed-based diversity.

### Pitfall 3: Score Scale Mismatch

**What goes wrong:** Combined score dominated by energy component (~10-100) while density component is tiny (~0.01).

**Why it happens:** Different scoring methods have different natural scales.

**How to avoid:** Normalize each component to 0-1 range before combining, or use percentage-based scoring.

### Pitfall 4: Ignoring Cell Volume Rounding

**What goes wrong:** Density score computed on rounded nmolecules doesn't match user's requested nmolecules.

**Why it happens:** GenIce rounds up to complete unit cells; actual_nmolecules may differ significantly from requested.

**How to avoid:** Document this in metadata; account for it in density scoring interpretation.

```python
# quickice/ranking/scorer.py
import numpy as np
from scipy.spatial.distance import cdist
from collections import Counter

from quickice.structure_generation.types import Candidate
from quickice.ranking.types import RankedCandidate, RankingResult

# Constants
IDEAL_OO_DISTANCE = 0.276  # nm, ideal O-O distance in ice
HBOND_ENERGY_KJ = 20  # kJ/mol, approximate H-bond energy


def calculate_oo_distances_pbc(positions, cell, cutoff=0.35):
    """Calculate O-O distances with periodic boundary conditions.
    
    Uses minimum image convention for PBC.
    """
    # Get cell matrix
    inv = np.linalg.inv(cell)
    
    # Fractional coordinates
    frac = positions @ inv
    
    # Calculate pairwise distances in fractional space
    n_atoms = len(frac)
    distances = []
    
    for i in range(n_atoms):
        for j in range(i + 1, n_atoms):
            # Minimum image in fractional space
            delta = frac[j] - frac[i]
            delta = delta - np.floor(delta + 0.5)
            
            # Convert back to Cartesian
            cart_delta = delta @ cell
            dist = np.linalg.norm(cart_delta)
            
            if dist < cutoff:  # Only consider H-bond distances
                distances.append(dist)
    
    return np.array(distances)


def energy_score(candidate):
    """Estimate energy from hydrogen bond network quality.
    
    Returns lower score for more ideal H-bond geometry.
    """
    positions = candidate.positions
    atom_names = candidate.atom_names
    cell = candidate.cell
    
    # Extract oxygen positions
    o_indices = [i for i, name in enumerate(atom_names) if name == 'O']
    o_positions = positions[o_indices]
    
    # Calculate O-O distances with PBC
    oo_distances = calculate_oo_distances_pbc(o_positions, cell)
    
    if len(oo_distances) == 0:
        return float('inf')  # No valid H-bonds
    
    # Mean absolute deviation from ideal distance
    deviations = np.abs(oo_distances - IDEAL_OO_DISTANCE)
    mean_deviation = np.mean(deviations)
    
    # Scale to get pseudo-energy (arbitrary units for ranking)
    energy_score = mean_deviation * 100
    
    return energy_score


def density_score(candidate):
    """Calculate density deviation score.
    
    Returns absolute deviation from expected density.
    """
    # Expected density from metadata
    expected_density = candidate.metadata.get('density', 0.9167)
    
    # Calculate actual density
    cell_volume_nm3 = np.abs(np.linalg.det(candidate.cell))
    mass_per_molecule = 18.01528  # g/mol
    avogadro = 6.022e23
    
    actual_density = (
        candidate.nmolecules * mass_per_molecule / 
        (avogadro * cell_volume_nm3 * 1e-21)
    )
    
    # Absolute deviation
    deviation = abs(actual_density - expected_density)
    
    return deviation


def diversity_score(candidate, all_candidates):
    """Calculate diversity score based on unique phases.
    
    For single-phase generation, measures structural diversity
    via seed-based variation (different hydrogen bond networks).
    """
    # Get all seeds (proxy for H-bond network diversity)
    all_seeds = [c.seed for c in all_candidates]
    seed_counter = Counter(all_seeds)
    
    # Score: unique seeds get higher diversity
    # Use inverse frequency
    same_seed_count = seed_counter.get(candidate.seed, 1)
    diversity_score = 1.0 / same_seed_count
    
    return diversity_score


def normalize_scores(scores):
    """Normalize scores to 0-1 range using min-max scaling."""
    scores = np.array(scores)
    min_s = np.min(scores)
    max_s = np.max(scores)
    
    if max_s == min_s:
        return np.zeros_like(scores)
    
    return (scores - min_s) / (max_s - min_s)


def rank_candidates(candidates, weights=None):
    """Rank candidates by combined scoring.
    
    Args:
        candidates: List of Candidate objects from Phase 3
        weights: Dict of component weights (default: equal weights)
    
    Returns:
        RankingResult with ranked candidates and score breakdown
    """
    if weights is None:
        weights = {'energy': 1.0, 'density': 1.0, 'diversity': 1.0}
    
    # Calculate raw scores for each component
    energy_scores = [energy_score(c) for c in candidates]
    density_scores = [density_score(c) for c in candidates]
    diversity_scores = [diversity_score(c, candidates) for c in candidates]
    
    # Normalize each component
    energy_norm = normalize_scores(energy_scores)
    density_norm = normalize_scores(density_scores)
    diversity_norm = normalize_scores(diversity_scores)
    
    # Calculate combined scores
    combined = []
    for i in range(len(candidates)):
        score = (
            weights['energy'] * energy_norm[i] +
            weights['density'] * density_norm[i] +
            weights['diversity'] * diversity_norm[i]
        )
        combined.append(score)
    
    # Create RankedCandidate objects
    ranked = []
    for i, cand in enumerate(candidates):
        ranked.append(RankedCandidate(
            candidate=cand,
            energy_score=energy_scores[i],
            density_score=density_scores[i],
            diversity_score=diversity_scores[i],
            combined_score=combined[i],
            rank=0  # Will be set after sorting
        ))
    
    # Sort by combined score (lower is better)
    ranked.sort(key=lambda x: x.combined_score)
    
    # Assign ranks
    for i, rc in enumerate(ranked):
        rc.rank = i + 1
    
    return RankingResult(
        ranked_candidates=ranked,
        scoring_metadata={
            'n_candidates': len(candidates),
            'ideal_oo_distance': IDEAL_OO_DISTANCE,
        },
        weight_config=weights
    )
```

## Open Questions

   - Should energy scores be phase-normalized?
   - **Recommendation:** Keep unnormalized for now; phases are sorted by user T,P anyway

2. **Diversity metric for single-phase generation**
   - If all candidates from same phase, diversity based on phase_id = constant
   - Fall back to seed-based structural diversity
   - **Recommendation:** Use seed-based diversity; document this in code

3. **Weight tuning**
   - Equal weights (1:1:1) suggested as baseline
   - User mentioned density-weighted "given T,P input context"
   - **Recommendation:** Start with equal weights, allow CLI flag for adjustment
