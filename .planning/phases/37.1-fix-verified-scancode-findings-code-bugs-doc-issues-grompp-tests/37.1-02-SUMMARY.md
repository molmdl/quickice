# Phase 37.1 Plan 02: GRO Coordinate Bugs Summary

**One-liner:** Fix MW recomputation for 4-atom ice/water (AN-02) and add PBC wrapping for solute/custom molecule positions (AN-03)

## What Was Done

### Task 1: Fix MW recomputation for 4-atom ice (AN-02)

In `write_interface_gro_file`, the MW position was always recomputed via `compute_mw_position(o_pos, h1_pos, h2_pos)` regardless of whether the ice had 3 or 4 atoms per molecule. For 4-atom hydrate ice (OW, HW1, HW2, MW), the MW already exists at `wrapped_positions[base_idx + 3]`, placed correctly by molecule-aware wrapping. The recomputed MW could differ from the existing MW in edge cases near PBC boundaries.

**Fix applied to all 4 writer functions:**
- `write_interface_gro_file`: 4-atom ice uses `wrapped_positions[base_idx + 3]`
- `write_ion_gro_file`: ice with `mol.count == 4` uses `wrapped_positions[start + 3]`; water always uses `wrapped_positions[start + 3]`
- `write_custom_molecule_gro_file`: same pattern for ice and water
- `write_solute_gro_file`: same pattern for ice and water

3-atom classic ice still computes MW from O/H1/H2 positions since no existing MW exists.

### Task 2: Add PBC wrapping for solute/custom positions (AN-03)

In `write_ion_gro_file`, solute_positions and custom_molecule_positions were written directly to GRO output without PBC wrapping. Coordinates could fall outside [0, box_size), producing invalid GRO files.

**Fix:** Added modulo wrapping using `positions % np.diag(ion_structure.cell)` for both solute and custom molecule positions, then updated all write references to use the wrapped arrays.
