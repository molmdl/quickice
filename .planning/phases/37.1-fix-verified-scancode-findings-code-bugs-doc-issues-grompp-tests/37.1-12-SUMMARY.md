## Changes Made

### Task 1: EH-01 + EH-05

**EH-01: GRO file write operations catch OSError and ValueError**

Five pipeline-called GRO writer functions in `gromacs_writer.py` now have try/except around their file-writing sections:
- `write_gro_file` (ice candidates)
- `write_interface_gro_file` (interface)
- `write_ion_gro_file` (ions)
- `write_custom_molecule_gro_file` (custom molecules)
- `write_solute_gro_file` (solutes)

Pattern applied to each:
- Only the file-writing portion is wrapped (validation/wrapping stays outside)
- On `OSError` (disk full, bad path) or `ValueError` (data validation): logger.error with filepath
- Partial file deleted via `Path(filepath).unlink()` if it exists
- Exception re-raised so callers can handle it

**EH-05: Pipeline export catches both OSError and ValueError**

The `_run_export_step` method's except clause changed from `except OSError as e:` to `except (OSError, ValueError) as e:` so that `ValueError` from gromacs_writer (e.g., invalid atom counts) is caught and reported instead of causing an unhandled crash.

### Task 2: EH-02

**Hydrate wrapper atom count assertion**

Added `assert water_atom_count + guest_atom_count == len(hydrate.positions)` in the hydrate export path, between computing `guest_atom_count` and creating the `InterfaceStructure` wrapper. This catches bugs where `water_count * 4` doesn't match actual water atoms in positions, preventing silent downstream atom count corruption.
