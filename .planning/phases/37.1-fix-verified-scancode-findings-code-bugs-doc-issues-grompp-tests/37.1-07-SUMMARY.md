# Phase 37.1 Plan 07: PBC Wrapping Integration Test Summary

PBC wrapping functional integration test confirms all GRO coordinates within [0, box_size)

## One-liner

Functional integration test verifying PBC wrapping in write_ion_gro_file produces coordinates within [0, box_size) for SOL, guest, custom, solute, and ion molecules.

## What Was Done

Created `tests/test_pbc_wrapping.py` with `TestPBCWrapping` class containing 4 integration test methods and a `_parse_gro_coordinates` helper function.

### Test Coverage

| Test | Chain | Molecule Types | Purpose |
|------|-------|---------------|---------|
| test_full_chain_gro_coordinates_in_box | F1: interface→custom→solute→ion | SOL, custom, solute, NA, CL | Most comprehensive: all molecule types |
| test_solute_only_coordinates_in_box | interface→solute→ion | SOL, solute, NA, CL | Solute-specific PBC wrapping |
| test_custom_only_coordinates_in_box | interface→custom→ion | SOL, custom, NA, CL | Custom molecule PBC wrapping |
| test_interface_only_coordinates_in_box | interface→ion | SOL, NA, CL | Simplest: SOL + ion wrapping |

### Helper Functions

- **`_parse_gro_coordinates(gro_path)`**: Parses GRO v3.3 format file, extracts atom positions from fixed-width columns [20:28], [28:36], [36:44], and box dimensions from last line. Returns `(positions_array [N,3], box_dims [3])`.

### Key Finding: Molecule-Center Wrapping Deviation

During test development, discovered that `wrap_molecules_into_box` can produce coordinates slightly outside [0, box_size) (observed max -0.009 nm for HW1/HW2 atoms). This occurs because:

1. The function unwraps atoms split across PBC (keeping molecule intact)
2. Wraps the molecule center into [0, box_size) using `np.mod`
3. Atoms near the reference atom can extend slightly beyond 0 when the molecule center is near a boundary

This is **expected behavior** — GROMACS handles it gracefully during simulation. The test uses 0.01 nm tolerance (10× GRO format precision) to account for this known property while still catching truly broken wrapping.
