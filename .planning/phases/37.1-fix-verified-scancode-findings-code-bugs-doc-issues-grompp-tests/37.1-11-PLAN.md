    - --ion-concentration (line 339-342): type=float, no range check

    Add range validation using a custom type function pattern (common in argparse):

    ```python
    def validate_concentration_range(value_str):
        """Validate concentration is in reasonable range [0.0, 5.0] mol/L."""
        value = float(value_str)
        if not (0.0 <= value <= 5.0):
            raise argparse.ArgumentTypeError(
                f"concentration must be between 0.0 and 5.0 mol/L, got {value}"
            )
        return value

    def validate_occupancy_range(value_str):
        """Validate cage occupancy is in [0.0, 100.0] percent."""
        value = float(value_str)
        if not (0.0 <= value <= 100.0):
            raise argparse.ArgumentTypeError(
                f"cage occupancy must be between 0.0 and 100.0%, got {value}"
            )
        return value
    ```

    Then update the add_argument calls:
    - --solute-concentration: type=validate_concentration_range (instead of type=float)
    - --ion-concentration: type=validate_concentration_range
    - --custom-concentration: type=validate_concentration_range
    - --cage-occupancy-small: type=validate_occupancy_range
    - --cage-occupancy-large: type=validate_occupancy_range

    Range rationale:
    - Concentration 0.0-5.0 mol/L: Covers practical range (seawater ~0.6, saturated ~5)
    - Occupancy 0.0-100.0%: Cage occupancy is a percentage

    Place the validator functions near the top of parser.py (after imports, before the build_parser function).
