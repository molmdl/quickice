key-decisions:
  - id: v03-d1-combined-tree-data
    decision: Replace base_existing_data + all_placed double-vstack with single combined_tree_data array
    rationale: >
      combined_tree_data is initialized from existing_tree.data.copy() and grows incrementally
      via np.vstack([combined_tree_data, solute_positions]) after each successful placement.
      This eliminates the conditional if/else on base_existing_data, the redundant reassembly
      of placed_positions list via np.vstack on every rebuild, and simplifies the code from
      8 lines to 3 lines in the success branch.
  - id: v03-d2-conditional-rebuild
    decision: cKDTree rebuild only after successful placement, matching ion_inserter pattern
    rationale: >
      The continue statement on overlap rejection (line 786) already skips the rebuild.
      Moving the rebuild into the explicit success branch (after placed=True, before break)
      makes the conditional pattern self-documenting and prevents future regressions.
  - id: v03-d3-regression-test
    decision: TestTREE03 tracks cKDTree call sizes to detect redundant rebuilds
    rationale: >
      Strictly-increasing size sequence proves no redundant rebuilds. Rebuild count
      equals placed molecule count. Each rebuild increment matches CH4_ATOMS_PER_MOLECULE=5.
      This matches the TestTREE01 pattern from 34.7-03.
