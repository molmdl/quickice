# Phase 37.1 Plan 01: Data Model Bugs Summary

**One-liner:** Fix critical TIP4P-ICE atom count (3→4), complete InterfaceStructure dataclass with 13 optional fields, replace hardcoded AVOGADRO with shared constant import.

## Tasks Completed

| 2 | Import AVOGADRO from ion_inserter instead of hardcoding (UM-01) | 0ad9bb3 | pipeline.py |

## Decisions Made

1. **ice atom count = 4 (TIP4P-ICE), not 3 (TIP3P):** MOLECULE_TYPE_INFO["ice"]["atoms"] was 3 from initial write, but the project uses TIP4P-ICE (4 atoms: OW, HW1, HW2, MW) throughout. This is a critical correctness fix.

2. **Add 13 optional fields to InterfaceStructure dataclass:** Duck-typing (setting undeclared attributes on a dataclass at runtime) breaks `dataclasses.fields()` and `dataclasses.asdict()`. IonStructure already declares these fields properly; InterfaceStructure should too for consistency and correctness.

3. **Import AVOGADRO from ion_inserter at module top level:** The `ion_inserter` module already defines `AVOGADRO = 6.02214076e23` (CODATA 2017). No circular dependency exists. Importing at module level is safe since it's just a constant, not a heavy dependency.
