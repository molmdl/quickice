## Decisions Made

1. **detect_atoms_per_molecule in types.py (single definition):** The function was identical in all 3 modes files (slab.py:24, pocket.py:24, piece.py:31). Moving to types.py alongside WATER_ATOMS_PER_MOLECULE ensures a single source of truth. The 3 modes files now import from types.py.

2. **Create new InterfaceStructure instead of mutating input (V-17 fix):** When `removed_count==0`, the old code set 5 custom_molecule_* attributes directly on the input `structure` object. This mutated the caller's InterfaceStructure, causing bugs if the same structure was reused. The fix creates a new `InterfaceStructure` with the custom molecule attributes set on the new object, not the input. This matches the pattern used in the `removed_count > 0` branch (lines 573-607), which already creates a new InterfaceStructure.
