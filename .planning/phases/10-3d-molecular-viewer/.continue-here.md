<context>

**Implementation approach:**

The 3D molecular viewer is fully implemented using VTK with PySide6 integration. The architecture follows a clean separation:

- **vtk_utils.py**: Low-level VTK conversion utilities (no Qt dependencies)
- **molecular_viewer.py**: Single viewer widget with all visualization features
- **dual_viewer.py**: Container for two synchronized viewers
- **view.py**: ViewerPanel and InfoPanel UI components
- **main_window.py**: Integration layer connecting ViewModel to viewer

**Why VTK crashes in SSH X11:**

VTK requires a proper OpenGL context with direct rendering. SSH X11 forwarding provides indirect rendering which works for simple apps (glxgears) but not for VTK's initialization. This is a known limitation, not a code bug.

**Fallback mode strategy:**

Rather than failing silently or crashing, the GUI now:
1. Detects likely SSH X11 forwarding (localhost in DISPLAY)
2. Skips VTK initialization
3. Shows informative message to user
4. Allows rest of GUI to function normally
5. Provides override for edge cases (QUICKICE_FORCE_VTK=true)

This ensures the application is usable in development environments while still requiring proper display for 3D visualization testing.
