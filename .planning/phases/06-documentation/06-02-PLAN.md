<task type="auto">
  <name>Create docs/ranking.md with scoring methodology</name>
  <files>docs/ranking.md</files>
  <action>
Create detailed ranking methodology documentation. Include:

1. **Title and intro**: "Ranking Methodology"

2. **Overview** section:
   - Candidates are ranked by combined score (lower = better)
   - Three components: energy, density, diversity

3. **Energy Score** section:
   - Based on O-O distance deviation from ideal (0.276 nm)
   - Formula: `energy_score = mean(|d_oo - 0.276|) * 100`
   - Lower = better (closer to ideal H-bond length)
   - Note: This is heuristic, not actual energy calculation

4. **Density Score** section:
   - Compares actual density to expected phase density
   - Formula: `density_score = |actual_density - expected_density|`
   - Lower = better match
   - Expected densities from phase metadata (e.g., Ice Ih: 0.9167 g/cm³)

5. **Diversity Score** section:
   - Rewards unique seeds (structures from less common seeds)
   - Formula: `diversity_score = 1.0 / seed_count`
   - Higher = more unique
   - Note: Single-phase fallback approach

6. **Combined Score** section:
   - Normalizes each component to 0-1 range
   - Formula: `combined = w_energy * norm_energy + w_density * norm_density + w_diversity * (1 - norm_diversity)`
   - Default weights: equal (1:1:1)
   - Lower combined = better overall candidate

7. **Limitations** section:
   - Honest note: scoring is heuristic, not physics-based
   - For real energy calculations, use MD simulations with force fields
   - Diversity is seed-based, not structural fingerprint

Reference the actual scorer.py implementation for accurate formulas.
  </action>

   - Uses GenIce2 library for actual structure generation
   - Provides heuristic ranking to prioritize candidates

3. **Why QuickIce Exists** section:
   - "Pure vibe coding project" - created as a coding exercise
   - Explore ice structures quickly without full MD simulations
   - Educational tool for understanding ice polymorphs

4. **How It Works** section:
   - Phase 1: Validate inputs (T, P, n_molecules)
   - Phase 2: Map T,P to ice polymorph using boundary curves
   - Phase 3: Generate candidates using GenIce2
   - Phase 4: Rank candidates by heuristic scores
   - Phase 5: Output PDB files and phase diagram

5. **Key Design Decisions** section:
   - Curve-based phase lookup (not polygon containment)
   - IAPWS R14-08 melting curves (high confidence)
   - Linear interpolation for solid-solid boundaries (medium confidence)
   - Heuristic ranking (not physics simulations)

6. **What QuickIce Does NOT Do** section:
   - No energy minimization
   - No molecular dynamics simulations
   - No validation against experimental data
   - Results are candidates, not validated structures
