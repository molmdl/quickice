Fix P2: Shapely centroid computed every render.

**Problem:** Lines 747-749 in phase_diagram.py compute shapely centroid for each phase on every render. With 11+ phases, this is 11 shapely polygon constructions per render.

**Current code (lines 747-761):**
```python
# Label using shapely centroid
shapely_poly = ShapelyPolygon(vertices)
centroid = shapely_poly.centroid
label = PHASE_LABELS.get(phase_id, phase_id)
ax.text(
    centroid.x, centroid.y,  # x=T, y=P (correct orientation)
    label,
    ...
)
```

**Fix:** Pre-compute and cache centroids at module level.

Add a centroid cache dictionary near the top of the file (after imports):
```python
# Cache for phase region centroids (computed once, reused)
_CENTROID_CACHE: dict[str, tuple[float, float]] = {}
```

Create a helper function to get or compute centroids:
```python
def _get_phase_centroid(phase_id: str, vertices: list) -> tuple[float, float]:
    """Get cached centroid for a phase region, computing if needed.
    
    Args:
        phase_id: Phase identifier
        vertices: List of (T, P) vertices defining the phase region
    
    Returns:
        (x, y) tuple of centroid coordinates
    """
    if phase_id not in _CENTROID_CACHE:
        shapely_poly = ShapelyPolygon(vertices)
        centroid = shapely_poly.centroid
        _CENTROID_CACHE[phase_id] = (centroid.x, centroid.y)
    return _CENTROID_CACHE[phase_id]
```

Replace lines 747-761 with:
```python
# Label using cached centroid
centroid_x, centroid_y = _get_phase_centroid(phase_id, vertices)
label = PHASE_LABELS.get(phase_id, phase_id)
ax.text(
    centroid_x, centroid_y,  # x=T, y=P (correct orientation)
    label,
    fontsize=14,
    fontweight='bold',
    ha='center',
    va='center',
    color='black',
    alpha=0.8,
    zorder=5,
)
```

**Why this works:**
- Centroids are computed once per phase per process lifetime
- No external cache invalidation needed (phase boundaries are static)
- Memory footprint is minimal (11 floats pairs)
