Fix S2: Path traversal vulnerability in --output argument.

**Problem:** Lines 48-49 in orchestrator.py. No validation of `output_dir`. User can write to arbitrary paths like `/etc/passwd` or `../../etc/shadow`.

**Current code (lines 47-49):**
```python
# Create output directory
output_path = Path(output_dir)
output_path.mkdir(parents=True, exist_ok=True)
```

**Fix:** Add validation to ensure output directory is within the current working directory:
```python
# Create output directory with path traversal protection
output_path = Path(output_dir).resolve()
allowed_base = Path.cwd().resolve()

# Security check: output must be within current working directory
if not str(output_path).startswith(str(allowed_base)):
    raise ValueError(
        f"Security error: Output path must be within {allowed_base}. "
        f"Got: {output_path}"
    )

output_path.mkdir(parents=True, exist_ok=True)
```

**Why this approach:**
- Uses `resolve()` to canonicalize paths (resolves symlinks, `..`, etc.)
- Prevents writing to system directories or arbitrary locations
- Provides clear error message explaining the restriction
- Still allows subdirectories like `./output/structures/`

**Alternative consideration:** Could also allow `--output /tmp/quickice` for temporary files, but for security, restricting to cwd is safer. Users can symlink or cd if needed.

Fix S3: Global numpy random state pollution.

**Problem:** Line 83 in generator.py uses `np.random.seed(seed)` which modifies GLOBAL random state. This affects all other code using numpy's random functions, causing:
- Non-reproducible results in other parts of the application
- Race conditions in multi-threaded environments
- Side effects that are hard to debug

**Current code (line 83):**
```python
# Set random seed for reproducible generation
np.random.seed(seed)
```

**Fix:** Use a local random generator instead:
```python
# Create local random generator for reproducible, thread-safe generation
rng = np.random.Generator(np.random.PCG64(seed))
```

**Update the GenIce call:** GenIce may need the random generator. Check if GenIce accepts a `rng` parameter. If not, the numpy Generator should still be used for any random operations within this function.

Looking at the code, the random seed is used for hydrogen bond network diversity in GenIce. The fix should create a local generator:

Note: If GenIce internally uses `np.random`, it may still use global state. Check GenIce documentation. For now, removing `np.random.seed(seed)` prevents the most obvious global state pollution.

Fix Q2: Overly broad exception handling in GenIce wrapper.

**Problem:** Line 123 in generator.py catches `Exception` which masks errors and makes debugging harder.

**Current code (lines 123-127):**
```python
except Exception as e:
    # Wrap GenIce errors in our custom exception
    raise StructureGenerationError(
        f"Failed to generate ice structure: {e}"
    ) from e
```

**Fix:** Catch specific GenIce exceptions. Check what exceptions GenIce raises:

Common GenIce exceptions (verify by checking GenIce source):
- `genice2.GenIceError` - base GenIce error
- `ValueError` - invalid parameters
- `ImportError` - lattice/molecule not found (from safe_import)

Update to catch specific exceptions:
```python
except (ValueError, ImportError) as e:
    # Configuration or import errors
    raise StructureGenerationError(
        f"Failed to generate ice structure: {e}"
    ) from e
except Exception as e:
    # Unexpected errors - log and re-raise with context
    # Use type(e).__name__ to identify the specific exception
    raise StructureGenerationError(
        f"Unexpected error generating ice structure: {type(e).__name__}: {e}"
    ) from e
```

Actually, for a hotfix, the safest approach is to keep catching Exception but improve the error message to include the exception type:

```python
except Exception as e:
    # Wrap any GenIce or internal errors with full context
    raise StructureGenerationError(
        f"Failed to generate ice structure ({type(e).__name__}): {e}"
    ) from e
```

This provides better debugging info without risking missed exceptions.

For a more comprehensive fix, investigate GenIce's exception hierarchy and catch specific exceptions.
