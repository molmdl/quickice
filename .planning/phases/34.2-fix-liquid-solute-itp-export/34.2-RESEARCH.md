# Phase 34.2: Fix Liquid Solute ITP Export - Research

**Researched:** 2026-05-05
**Domain:** GROMACS topology file generation, moleculetype naming
**Confidence:** HIGH

## Summary

This phase addresses a critical GROMACS compatibility issue: liquid solutes (CH4_LIQ, THF_LIQ) currently export with incorrect ITP files that use "ch4" and "thf" moleculetype names, causing mismatches with the registry-generated names. The fix requires creating separate ITP files for liquid solutes with correct moleculetype and residue names, and updating the export logic to select the appropriate ITP file based on molecule source.

**Primary recommendation:** Create ch4_liquid.itp and thf_liquid.itp files with moleculetype names "CH4_LIQ" and "THF_LIQ", update SoluteGROMACSExporter to copy these files for liquid solutes, ensure .gro residue names match.

### Pattern 1: Moleculetype Name Consistency

**What:** GROMACS requires moleculetype name in .itp file to match [molecules] section name in .top file

**Critical requirement:**
```
In .itp file:
[ moleculetype ]
molecule_name   3

In .top file:
[ molecules ]
molecule_name   N_molecules
```

**Why it matters:** GROMACS will fail with "Invalid moleculetype" error if names don't match

**Current violation:**
- Registry generates: `CH4_LIQ`
- ITP file contains: `ch4`
- Result: **MISMATCH** → GROMACS error

### Pattern 2: Residue Name Consistency

**What:** Residue name in .itp [atoms] section should match residue name in .gro file

**Example:**
```gro
# In .gro file (fixed-width columns):
    1CH4_LIQ    C    1   1.234   2.345   3.456
    1CH4_LIQ    H    2   1.456   2.567   3.678

# In .itp file:
[ atoms ]
;  Index   type   residue  resname   atom  ...
     1     c3         1   CH4_LIQ    C    ...
     2     hc         1   CH4_LIQ    H    ...
```

**Why it matters:** GROMACS uses residue names to identify molecules. Mismatched names cause topology errors.

### Anti-Patterns to Avoid

- **Same ITP file for different moleculetype names:** Cannot reuse ch4.itp for both "CH4" and "CH4_LIQ"
- **Hardcoded residue names:** .gro writer must use registry-provided names, not hardcoded "CH4"/"THF"
- **Assuming case-insensitivity:** GROMACS is case-sensitive; "ch4" ≠ "CH4_LIQ"

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| ITP file creation | String formatting/templating | Copy existing .itp and modify moleculetype/residue names | Avoid syntax errors, preserve bond/angle/dihedral definitions |
| Residue name lookup | Hardcoded dictionary | MoleculetypeRegistry.get_gromacs_name() | Registry tracks source context (hydrate vs liquid) |
| ITP file selection | Conditional based on solute type | Conditional based on source (hydrate/liquid) | Same solute type can have different sources |

**Key insight:** The parameters (bonds, angles, dihedrals, charges) are identical between hydrate guests and liquid solutes. Only the moleculetype name and residue name differ. This means we can create liquid ITP files by copying and modifying the header sections.

## Common Pitfalls

### Pitfall 1: Moleculetype Name Mismatch

**What goes wrong:** .top file uses "CH4_LIQ" from registry, but .itp file defines moleculetype "ch4"

**Why it happens:** SoluteGROMACSExporter.copy_solute_itp() copies ch4.itp directly without modification

**How to avoid:** Create separate ch4_liquid.itp with correct moleculetype name, update export logic to select based on source

**Warning signs:**
- GROMACS error: "Invalid moleculetype in topology"
- GROMACS error: "moleculetype not found"

**Current code (INCORRECT):**
```python
# In SoluteGROMACSExporter.export_solute_gromacs()
if solute_type == "CH4":
    solute_itp_name = "ch4.itp"  # ← WRONG for liquid solutes!
else:
    solute_itp_name = "thf.itp"  # ← WRONG for liquid solutes!
```

### Pitfall 2: Residue Name Mismatch Between .gro and .itp

**What goes wrong:** .gro file uses "CH4_LIQ" residue name, but .itp defines residue "CH4"

**Why it happens:** gromacs_writer.write_multi_molecule_gro_file() uses registry name, but ch4.itp has hardcoded "CH4"

**How to avoid:** Create ch4_liquid.itp with "CH4_LIQ" residue name in [atoms] section

**Warning signs:**
- GROMACS error: "Residue 'CH4_LIQ' not found in topology"
- Atoms appear in wrong molecule type during simulation

**Example violation:**
```python
# In gromacs_writer.py write_ion_gro_file()
# Lines 1383-1396:
solute_res_name = ion_structure.solute_registry.get_gromacs_name(
    f"liquid_{ion_structure.solute_type}"
)  # Returns "CH4_LIQ"

# But ch4.itp has:
# [ atoms ]
#      1     c3         1      CH4    C  ...  ← "CH4" not "CH4_LIQ"!
```

### Pitfall 3: Both Hydrate and Liquid Solutes in Same System

**What goes wrong:** System has both CH4 hydrate guests and CH4_LIQ liquid solutes, but only one ch4.itp file

**Why it happens:** Current design doesn't distinguish ITP files by source

**How to avoid:** Create separate ITP files:
- ch4.itp (or ch4_hydrate.itp) for hydrate guests with moleculetype "CH4_HYD"
- ch4_liquid.itp for liquid solutes with moleculetype "CH4_LIQ"
- Include both in .top file when both present

**Warning signs:**
- GROMACS error: "Duplicate moleculetype definition" (if same name)
- GROMACS error: "moleculetype not found" (if missing one)
- Incorrect simulation results (if one overwrites the other)

```python

# Liquid solutes use _liquid.itp files
solute_type_lower = solute_type.lower()
if solute_structure.registry:
    # Check if this is a liquid solute
    liquid_key = f"liquid_{solute_type}"
    if liquid_key in solute_structure.registry._registered:
        # Liquid solute: use ch4_liquid.itp or thf_liquid.itp
        solute_itp_name = f"{solute_type_lower}_liquid.itp"
    else:
        # Fallback to standard ITP (shouldn't happen for solutes)
        solute_itp_name = f"{solute_type_lower}.itp"

    # Fallback if no registry (shouldn't happen)
    solute_itp_name = f"{solute_type_lower}_liquid.itp"
```

## Open Questions

### 1. Should Hydrate Guest ITP Files Be Renamed?

**Question:** Should ch4.itp be renamed to ch4_hydrate.itp for consistency?

**Options:**
- **Option A:** Keep ch4.itp for hydrates (backward compatible, less change)
- **Option B:** Rename to ch4_hydrate.itp (consistent naming, breaking change)

**Recommendation:** Option A - Keep ch4.itp for hydrates
- Current code uses "ch4.itp" and "thf.itp" for hydrate guests
- Changing would require updating multiple export paths
- "ch4.itp" is clear enough for the primary use case (hydrate guests)
- "ch4_liquid.itp" clearly distinguishes liquid solutes

**Impact on Phase 34.2:**
- No change needed to existing hydrate export paths
- Only add new liquid ITP files
- Lower risk, faster implementation

### 2. Should Moleculetype Names Match Residue Names Exactly?

**Question:** ITP moleculetype "CH4_LIQ" vs residue name "CH4_LIQ" - should they match exactly?

**Current THF.itp:**
- Moleculetype: "thf" (lowercase)
- Residue name: "THF" (uppercase)

**Current CH4.itp:**
- Moleculetype: "ch4" (lowercase)
- Residue name: "CH4" (uppercase)

**GROMACS behavior:**
- Moleculetype name is for topology matching (.top [molecules])
- Residue name is for coordinate matching (.gro residue column)
- They don't need to match each other

**Recommendation:** Follow registry naming convention
- Moleculetype: "CH4_LIQ" (from registry)
- Residue name: "CH4_LIQ" (from registry)
- Consistency reduces confusion

### 3. System with Both Hydrate Guests and Liquid Solutes

**Scenario:** User generates hydrate with CH4 guests, then inserts CH4_LIQ solutes

**Required .top structure:**
```
#include "ch4.itp"         ← For CH4_HYD
#include "ch4_liquid.itp"  ← For CH4_LIQ

[ molecules ]
SOL          1000
CH4_HYD      50    ← Uses ch4.itp
CH4_LIQ      25    ← Uses ch4_liquid.itp
```

**Question:** How does current system handle this?

**Current behavior:**
- Hydrate export: Uses ch4.itp, registers "CH4_HYD"
- Solute export: Would use ch4.itp, registers "CH4_LIQ"
- **PROBLEM:** Both would use same ITP file, moleculetype mismatch

**Recommendation for Phase 34.2:**
- Create separate ITP files (ch4.itp and ch4_liquid.itp)
- Ensure both can be included in same .top file
- Each has distinct moleculetype name
- Update export logic to select correct ITP

**Future consideration:**
- Update hydrate export to use "CH4_HYD" moleculetype name
- Create ch4_hydrate.itp if needed
- For Phase 34.2, focus on liquid solutes only
