# Phase 34.4: Solute Source Dropdown - Research

**Researched:** 2026-05-07
**Domain:** GUI source selection pattern, Custom → Solute workflow integration
**Confidence:** HIGH

## Summary

This phase adds a source dropdown to the Solute Insertion tab (Tab 4) to enable the Custom → Solute workflow chain planned in the architecture phase. The implementation follows the exact pattern established in Phase 34.1 (Ion source dropdown) with key adaptations for the solute insertion context.

**Core requirements:**
1. Add source dropdown with "Interface" and "Custom Molecule" options (NO "Solute" option - solutes can't insert solutes)
2. Follow IonPanel source dropdown pattern exactly (Phase 34.1)
3. Handle Custom Molecule data passing from Tab 3 to Tab 4
4. Validate source availability and disable Insert button when unavailable
5. Pass custom molecule ITP/GRO data to SoluteInserter for GROMACS export

**Primary recommendation:** Add source dropdown to SolutePanel following IonPanel pattern exactly, with "Interface" (default) and "Custom Molecule" sources. MainWindow must check source availability and pass appropriate data structure.

### Pattern 2: Source Change Handler

**What:** Handle source selection change with immediate UI effect
**When to use:** When source dropdown selection changes
**Example:**
```python
def _on_source_changed(self, index: int):
    """Handle source selection change.
    
    Follows IonPanel._on_source_changed pattern:
    - Map index to source name
    - Update internal state
    - Clear 3D viewer
    - Log source change
    - Update Insert button state
    - Emit configuration_changed signal
    """
    # Map index to source name
    source_names = ["Interface", "Custom Molecule"]
    source_name = source_names[index] if 0 <= index < len(source_names) else "Interface"
    
    # Update internal state
    self._current_source = source_name
    
    # Clear 3D viewer
    if hasattr(self.solute_viewer, 'clear'):
        self.solute_viewer.clear()
    
    # Log source change
    self.log_message(f"Source switched to {source_name}")
    
    # Update Insert button state based on source availability
    self._update_insert_button_state()
    
    # Emit configuration_changed signal
    self.configuration_changed.emit()
```
**Source:** quickice/gui/ion_panel.py:233-274 (adapted for 2 sources)

### Pattern 3: Source Availability Management

**What:** Track and validate source availability
**When to use:** When enabling/disabling Insert button based on source
**Example:**
```python
def _update_insert_button_state(self):
    """Update Insert button enabled state and tooltip based on source availability."""
    if self._current_source == "Interface" and not self._interface_available:
        # Interface source but no structure generated
        self.insert_button.setEnabled(False)
        self.insert_button.setToolTip("Generate Interface structure first (Tab 2)")
    elif self._current_source == "Custom Molecule" and not self._custom_molecule_available:
        # Custom Molecule source but no molecules generated
        self.insert_button.setEnabled(False)
        self.insert_button.setToolTip("Generate Custom Molecules first (Tab 3)")
    else:
        # Source available
        self.insert_button.setEnabled(True)
        self.insert_button.setToolTip(
            "Insert solute molecules into the liquid water region.\n"
            "Calculated from concentration and liquid volume."
        )

def set_custom_molecule_available(self, available: bool):
    """Set whether Custom Molecule source has data available.
    
    Called by MainWindow when custom molecules are generated.
    """
    self._custom_molecule_available = available
    self._update_insert_button_state()
```

### Anti-Patterns to Avoid

- **DON'T add "Solute" source option**: Solute tab inserting solutes into solute structure doesn't make sense
- **DON'T skip source availability checks**: Must validate source exists before allowing insertion
- **DON'T forget to clear 3D viewer**: When source changes, viewer should show placeholder
- **DON'T assume CustomMoleculeStructure has interface_structure**: May need to add this field or handle differently

## Don't Hand-Roll

Problems that look simple but have established patterns:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Source dropdown UI | Custom dropdown logic | IonPanel source dropdown pattern | Proven pattern, consistent UX |
| Source change handling | Custom signal handling | `_on_source_changed` pattern from Ion | Clear separation of concerns |
| Button state management | Ad-hoc enable/disable | `_update_insert_button_state` pattern | Consistent with other tabs |
| Source tracking | Multiple flags | Single `_current_source` string | Simpler state management |
| Custom molecule getters | Direct attribute access | Add `get_gro_path()`, `get_itp_path()` methods | Encapsulation, consistency |

**Key insight:** The Ion source dropdown pattern (Phase 34.1) is the reference implementation. Copy it exactly with 2 sources instead of 3.

## Common Pitfalls

### Pitfall 1: Missing Getter Methods in CustomMoleculePanel

**What goes wrong:** MainWindow calls `custom_molecule_panel.get_gro_path()` and `get_itp_path()` but these methods don't exist
**Why it happens:** CustomMoleculePanel stores `self.gro_path` and `self.itp_path` as attributes but has no getters
**How to avoid:** Add getter methods to CustomMoleculePanel:
```python
def get_gro_path(self) -> Path | None:
    """Get current GRO file path."""
    return self.gro_path

def get_itp_path(self) -> Path | None:
    """Get current ITP file path."""
    return self.itp_path
```
**Warning signs:** AttributeError when MainWindow tries to call getters

### Pitfall 2: CustomMoleculeStructure Missing Interface Reference

**What goes wrong:** MainWindow needs to access the interface structure that custom molecules were added to, but CustomMoleculeStructure doesn't store it
**Why it happens:** CustomMoleculeStructure was designed for standalone use, not as a source for other tabs
**How to avoid:** Either:
1. Add `interface_structure: InterfaceStructure` field to CustomMoleculeStructure, OR
2. MainWindow stores `_current_interface_result` separately and uses it when source="Custom Molecule"
**Warning signs:** Need to access interface data but only have CustomMoleculeStructure

### Pitfall 3: Forgetting to Update Source Availability

**What goes wrong:** Custom Molecule source shows as available even before molecules are generated
**Why it happens:** MainWindow doesn't call `solute_panel.set_custom_molecule_available(True)` after custom molecule generation
**How to avoid:** Follow Ion pattern - call `set_custom_molecule_available(True)` in `_on_custom_finished()` handler
**Warning signs:** Insert button enabled when no custom molecules exist

### Pitfall 4: Not Clearing Viewer on Source Change

**What goes wrong:** Old structure visible in 3D viewer after switching sources
**Why it happens:** Source change handler doesn't call `solute_viewer.clear()`
**How to avoid:** Include viewer clearing in `_on_source_changed()` handler
**Warning signs:** Mixed structures visible in viewer

### Pitfall 5: GROMACS Export Not Bundling Custom ITP

**What goes wrong:** GROMACS export fails or missing custom molecule ITP when using Custom Molecule source
**Why it happens:** SoluteInserter doesn't know about custom molecules, GROMACS exporter doesn't bundle custom ITP
**How to avoid:** Pass custom molecule ITP path to SoluteInserter when source="Custom Molecule", update GROMACS exporter to bundle it
**Warning signs:** Export fails or missing topology files

@Slot()
def _on_insert_solutes(self):
    """Handle solute insertion request."""
    from quickice.structure_generation.solute_inserter import SoluteInserter
    
    # Get current source
    current_source = self.solute_panel.get_current_source()
    
    # Get appropriate structure based on source
    if current_source == "Interface":
        if not hasattr(self, '_current_interface_result') or self._current_interface_result is None:
            self.solute_panel.log_message("Error: No interface structure available.")
            return
        interface = self._current_interface_result
        custom_molecule_data = None  # No custom molecules
        
    elif current_source == "Custom Molecule":
        if not hasattr(self, '_current_custom_molecule_result') or self._current_custom_molecule_result is None:
            self.solute_panel.log_message("Error: No custom molecule structure available.")
            return
        
        custom_structure = self._current_custom_molecule_result
        
        # Option A: If CustomMoleculeStructure stores interface_structure
        interface = custom_structure.interface_structure
        
        # Option B: If not stored, use _current_interface_result
        # interface = self._current_interface_result
        
        # Pass custom molecule data for GROMACS export
        custom_molecule_data = {
            'gro_path': self.custom_molecule_panel.get_gro_path(),
            'itp_path': self.custom_molecule_panel.get_itp_path(),
            'residue_name': custom_structure.residue_name,
            'moleculetype_name': custom_structure.moleculetype_name
        }
    
    # Get configuration and insert
    config = self.solute_panel.get_configuration()
    self.solute_panel.log_message(f"Inserting {config.solute_type} solutes at {config.concentration_molar} M...")
    
    try:
        inserter = SoluteInserter(config)
        solute_structure = inserter.insert_solutes(interface, config, custom_molecule_data=custom_molecule_data)

## Open Questions

### Question 1: CustomMoleculeStructure Interface Reference

**What we know:**
- CustomMoleculeStructure has positions, atom_names, cell, molecule_index
- MainWindow stores `_current_custom_molecule_result`
- Solute inserter needs InterfaceStructure (ice + water + custom molecules)

**What's unclear:**
- Does CustomMoleculeStructure already have `interface_structure` field?
- If not, should we add it or just use `_current_interface_result` from MainWindow?

**Recommendation:** Check CustomMoleculeStructure dataclass. If it doesn't have `interface_structure` field, add it during custom molecule insertion. Alternative: MainWindow can use separate `_current_interface_result` when source="Custom Molecule".

### Question 2: SoluteInserter Custom Molecule Handling

**What we know:**
- SoluteInserter currently only accepts InterfaceStructure and SoluteConfig
- Need to pass custom molecule ITP/GRO for GROMACS export

**What's unclear:**
- Does SoluteInserter need to be modified to accept custom molecule data?
- How does GROMACS export bundle custom molecule ITP?

**Recommendation:** Check SoluteInserter implementation. May need to add `custom_molecule_data` parameter to `insert_solutes()` method. GROMACS exporter should bundle custom ITP when present.

### Question 3: GROMACS Export Topology

**What we know:**
- GROMACS export needs to include custom molecule ITP
- Custom molecules need unique moleculetype names (CUSTOM_MOL_1, etc.)

**What's unclear:**
- Does current export handle custom molecules from Interface structure?
- Does SoluteStructure track custom molecules separately?

**Recommendation:** Check export.py to see how CustomMoleculeGROMACSExporter works. SoluteStructure may need to carry custom molecule information for export.
