Fix the confirmed TIP4P-ICE LJ parameter bug in all 6 TOP-writing functions.

**BUG DETAILS:**

The bundled template `quickice/data/tip4p-ice.itp` has CORRECT values (line 4):
```
;OW_ice      8      15.9994  0.0000  A   0.31668  0.88211
```
These match the Abascal et al. 2005 TIP4P-ICE paper (DOI: 10.1063/1.1931662):
- σ_O = 3.1668 Å = 0.31668 nm
- ε_O/k_B = 106.1 K → ε = 106.1 × 0.00831446 = 0.882 kJ/mol

But the writer at 6 locations emits catastrophically wrong values:
- σ = `0.31668e-3` (0.000317 nm) — 1000× too small
- ε = `0.88216e-6` (0.000000882 kJ/mol) — 10⁶× too small

For comparison, all OTHER atomtypes in the same functions are correct:
- NA: `2.21737e-1` (0.222 nm) ✅
- CL: `4.69906e-1` (0.470 nm) ✅
- c3 (CH4): `3.39771e-1` (0.340 nm) ✅
- os (THF): `3.15610e-1` (0.316 nm) ✅

**COMB-RULE ANALYSIS:**

All 6 functions use `comb-rule = 2` (Lorentz-Berthelot). With comb-rule 2, the `[atomtypes]` V and W columns are C6 and C12, NOT sigma and epsilon. However:

- The column header at line 1278 says `sigma (nm)` and `epsilon (kJ/mol)` — confirming the WRITER INTENDED sigma/epsilon format
- The values `0.31668e-3` and `0.88216e-6` are NEITHER correct sigma/epsilon NOR correct C6/C12
- Expected C6 = 4εσ⁶ = 3.559e-03; Expected C12 = 4εσ¹² = 3.590e-06
- Neither written value matches either format

**FIX APPROACH — Option A (RECOMMENDED):**

Switch all 6 functions to `comb-rule = 1` and write sigma/epsilon directly:
1. Change `[defaults]` line: `1  2  yes  0.5  0.8333` → `1  1  yes  0.5  0.8333`
2. Change OW_ice line: `0.31668e-3    0.88216e-6` → `3.16680e-1    8.82110e-1`
3. All other atomtypes (NA, CL, c3, hc, os, c5, h1, etc.) are ALREADY in sigma/epsilon format with correct values — no change needed
4. Update column header comments from `V`/`W` to `sigma (nm)`/`epsilon (kJ/mol)` where missing

**Pro:** Human-readable, matches tip4p-ice.itp format, matches GROMACS tutorials, matches ALL existing guest/ion atomtypes already written in sigma/epsilon format

**Con:** Changes comb-rule. But comb-rule 1 (geometric) is the standard for mixed force fields (TIP4P-ICE + GAFF2 + Madrid2019). GROMACS documentation recommends comb-rule 1 for heterogeneous atomtypes.

**FIX APPROACH — Option B (alternative):**

Keep `comb-rule = 2` and compute correct C6/C12:
1. Keep `[defaults]` line unchanged: `1  2  yes  0.5  0.8333`
2. Change OW_ice line: `0.31668e-3    0.88216e-6` → `3.559026e-3    3.589674e-6`
3. ALL guest/ion atomtypes must ALSO be converted to C6/C12 format (they're currently in sigma/epsilon format which is WRONG for comb-rule 2)
4. This means computing C6/C12 for every atomtype: NA, CL, c3, hc, os, c5, h1, etc.

**Pro:** No comb-rule change
**Con:** MUCH more work (must convert ALL atomtypes), values are opaque/hard to verify, GROMACS tutorials rarely use comb-rule 2 with sigma/epsilon

**RECOMMENDED:** Option A. It's simpler, more human-readable, and corrects an existing inconsistency (guest atomtypes are already in sigma/epsilon format despite comb-rule 2).
