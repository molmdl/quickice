**VERDICT: REAL_BUG** — The TIP4P-ICE LJ parameters written to ALL .top files are catastrophically wrong.

### Executive Summary

The OW_ice atom type in every `.top` file generated by QuickIce has TIP4P-ICE LJ parameters that are **1000× too small** for σ and **10⁶× too small** for ε. This effectively eliminates ALL water-water van der Waals interactions. The bug is present in all 6 TOP-writing functions and affects every simulation ever run with QuickIce-generated topologies.

- σ_O = 3.1668 Å → 0.31668 nm
- ε_O/k_B = 106.1 K → ε = 106.1 × 0.008314462618 = 0.882164 kJ/mol

- **GROMACS grompp**: Does not warn about abnormally small LJ parameters — it just uses whatever values are provided

## Root Cause Analysis

The root cause is a **scientific notation error** in the hardcoded OW_ice atom type definition. The values appear to have been copied from the commented-out [atomtypes] section of the original tip4p-ice.itp.bak file but with incorrect exponents added:

- Original ITP (no exponent): `0.31668` and `0.88216`
- Written to .top with wrong exponents: `0.31668e-3` and `0.88216e-6`

The `e-3` and `e-6` exponents were likely added during a mistaken unit conversion attempt (possibly confusing nm with pm for σ, and kJ/mol with mJ/mol for ε), or were simply a typo when transcribing values into the f.write strings.

This same string was then copy-pasted across all 6 TOP-writing functions, propagating the bug uniformly.

## Impact Scope

**All 6 TOP-writing functions are affected:**
1. `write_top_file` (ice candidate export) — line 550

### Physical Impact on Simulations

With σ ≈ 0.0003 nm and ε ≈ 0 kJ/mol:
- Water-water LJ interactions are effectively zero
- Water molecules have no van der Waals repulsive core → can overlap unphysically
- Water-guest van der Waals (primary hydrate stabilization force) is ~0
- PME electrostatics still provides some cohesion, making simulations appear to "work"
- **All thermodynamic properties** (density, enthalpy of vaporization, melting point, dissociation pressure, hydration free energy, cage occupancy) are catastrophically wrong
