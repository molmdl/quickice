### Auto-fixed Issues

**1. [Rule 1 - Bug] Research §3 "[defaults] byte-identical across all 6 TOP writers" claim was WRONG — actual source has 3 distinct format variants; parameterized the helper to preserve byte-identity**
- **Found during:** Task 1 (read each of the 6 inline `[defaults]` blocks before editing, per the plan's explicit instruction "VERIFY the inline block is byte-identical to the helper... If ANY writer's `[defaults]` block differs... DO NOT replace that writer's block — investigate first")
- **Issue:** Research §3 of the plan claimed the 6 `[defaults]` blocks are "byte-identical across all 6 TOP writers" and specified a no-args helper `_write_top_defaults(f)` that emits ONE format. Direct source reading (via Python `repr()` extraction) revealed **3 distinct format variants**:
  - Format A (write_top_file L185-190 + write_interface_top_file L605-610): 6 f.write calls — includes `; Defaults compatible with the Amber forcefield\n` header line + compact `; nbfunc  comb-rule  gen-pairs  fudgeLJ  fudgeQQ\n` comment (single-space separators)
  - Format B (write_multi_molecule_top_file L1044-1048): 5 f.write calls — NO `; Defaults compatible` header line, compact `; nbfunc` comment (same as Format A's comment)
  - Format C (write_ion_top_file L1691-1695 + write_custom_molecule_top_file L2242-2246 + write_solute_top_file L2922-2926): 5 f.write calls — NO `; Defaults compatible` header line AND multi-space aligned `; nbfunc        comb-rule       gen-pairs       fudgeLJ fudgeQQ\n` comment (different from A/B)

- **Fix:** Parameterized the helper with 2 boolean keyword-only flags:

  The 6 call sites:
  - `write_top_file`: `_write_top_defaults(f)` (defaults → Format A)
  - `write_interface_top_file`: `_write_top_defaults(f)` (defaults → Format A)
  - `write_multi_molecule_top_file`: `_write_top_defaults(f, include_amber_header=False)` (Format B)
