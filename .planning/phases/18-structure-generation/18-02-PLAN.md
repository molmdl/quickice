**modes/slab.py** — Slab mode: ice-water-ice sandwich along Z-axis.

Function signature: `assemble_slab(candidate: Candidate, config: InterfaceConfig) -> InterfaceStructure`

Layout along Z-axis:
- Bottom ice: [0, ice_thickness]
- Water: [ice_thickness, ice_thickness + water_thickness]
- Top ice: [ice_thickness + water_thickness, box_z]

Implementation steps:
1. Get ice cell diagonal dimensions: `ice_cell_dims = np.array([candidate.cell[0,0], candidate.cell[1,1], candidate.cell[2,2]])`. Only support orthogonal boxes for v3.0.

2. **Tile ice for bottom layer**: Use `tile_structure(candidate.positions, ice_cell_dims, np.array([config.box_x, config.box_y, config.ice_thickness]))` from water_filler. This returns (bottom_ice_positions, bottom_ice_nmolecules). Note: tile_structure works for ice too — it just tiles any structure to fill a target region.

3. **Tile ice for top layer**: Same call but shift Z positions after tiling. `top_ice_positions = bottom_ice_tiled_positions.copy()`, then `top_ice_positions[:, 2] += config.ice_thickness + config.water_thickness`. However, we need to tile independently because the positions after tiling are filtered to be within [0, ice_thickness] in Z. So tile the same way as bottom, then shift.

Actually, simpler approach: tile ice for bottom layer with target [box_x, box_y, ice_thickness], tile ice for top layer with same target, then shift top layer Z by (ice_thickness + water_thickness).

4. **Fill water in middle region**: Call `fill_region_with_water(np.array([config.box_x, config.box_y, config.water_thickness]), config.overlap_threshold)` from water_filler. This returns (water_positions, water_atom_names, water_nmolecules). Then shift water Z by ice_thickness: `water_positions[:, 2] += config.ice_thickness`.

5. **Detect overlaps**: Extract ice O positions and water O positions. For ice from GenIce (3 atoms per molecule: O, H, H), O atoms are at indices [0, 3, 6, ...] → `ice_o_positions = combined_ice_positions[::3]`. For water from tip4p.gro (4 atoms: OW, HW1, HW2, MW), O atoms are at indices [0, 4, 8, ...] → `water_o_positions = water_positions[::4]`. Call `detect_overlaps(ice_o_positions, water_o_positions, np.array([config.box_x, config.box_y, config.box_z]), config.overlap_threshold)`.

6. **Remove overlapping water molecules**: Call `remove_overlapping_molecules(water_positions, overlapping_mol_indices, atoms_per_molecule=4)`.

7. **Combine all positions**: Ice atoms FIRST, then water atoms. `all_positions = np.vstack([combined_ice_positions, trimmed_water_positions])`. Same order for atom_names. Compute `ice_atom_count = len(combined_ice_positions)`, `water_atom_count = len(trimmed_water_positions)`.

8. **Build cell matrix**: `cell = np.diag([config.box_x, config.box_y, config.box_z])`.

9. **Build report string**: Following gmx solvate convention — report molecules present. Format:
   ```
   Generated slab interface structure
     Ice molecules: {N_ice}
     Water molecules: {N_water_kept}
     Total molecules: {N_total}
     Box: {box_x:.2f} x {box_y:.2f} x {box_z:.2f} nm
   ```

10. **Create InterfaceStructure** with all fields populated.

CRITICAL: Ice atoms come from GenIce with 3 atoms per molecule (O, H, H). Do NOT normalize or add MW atoms. That is an export concern for Phase 20.

CRITICAL: All coordinates in nm. The overlap_threshold in InterfaceConfig is already in nm (0.25 default).

**modes/pocket.py** — Pocket mode: water cavity in ice matrix.

Function signature: `assemble_pocket(candidate: Candidate, config: InterfaceConfig) -> InterfaceStructure`

Implementation steps:
1. Get ice cell diagonal dimensions (same as slab).

2. **Tile ice to fill entire box**: `tile_structure(candidate.positions, ice_cell_dims, np.array([config.box_x, config.box_y, config.box_z]))`. This gives (ice_positions, ice_nmolecules). Replicate ice_atom_names by the tiling factor.

3. **Remove ice molecules within spherical cavity**:
   - Cavity center: `center = np.array([config.box_x, config.box_y, config.box_z]) / 2.0`
   - Cavity radius: `radius = config.pocket_diameter / 2.0`
   - For each ice molecule, check if its O atom is within the cavity: `ice_o_positions = ice_positions[::3]` (GenIce uses 3 atoms/molecule)
   - `distances = np.linalg.norm(ice_o_positions - center, axis=1)` — NO PBC wrapping needed here because cavity is fully inside the box (not near boundaries). The pocket is at the center and smaller than the box.
   - `ice_inside_cavity = set(np.where(distances < radius)[0])` — these molecules are INSIDE the cavity and should be REMOVED
   - Remove these ice molecules: `keep_mask = np.ones(ice_nmolecules, dtype=bool)`, set False for each index in ice_inside_cavity, expand with `np.repeat(keep_mask, 3)` (3 atoms per ice molecule), filter positions.
   - This is the ONLY case where we remove ice molecules — to make room for the water cavity.

4. **Fill entire box with water**: `fill_region_with_water(np.array([config.box_x, config.box_y, config.box_z]), config.overlap_threshold)`. Returns (water_positions, water_atom_names, water_nmolecules).

5. **Remove water molecules OUTSIDE the cavity**:
   - `water_o_positions = water_positions[::4]`
   - `water_distances = np.linalg.norm(water_o_positions - center, axis=1)`
   - `water_outside = set(np.where(water_distances >= radius)[0])` — remove these (keep only water IN cavity)
   - Use remove_overlapping_molecules with water_outside set and atoms_per_molecule=4

6. **Detect overlaps between remaining ice and cavity water**:
   - Get remaining ice O positions and remaining water O positions
   - `detect_overlaps(remaining_ice_o, remaining_water_o, box_dims, config.overlap_threshold)`

7. **Remove overlapping water molecules** from the cavity water (not ice — ice at the cavity boundary is preserved).

8. **Combine**: ice (outside cavity) FIRST, then water (in cavity).

9. **Build InterfaceStructure** with cell, report, etc.

NOTE on pocket_shape: InterfaceConfig stores pocket_shape ("sphere" or "ellipsoid") but for v3.0 MVP, only implement sphere. If pocket_shape == "ellipsoid", raise InterfaceGenerationError with mode="pocket" saying ellipsoid is not yet supported. This leaves the API clean for future extension.

**modes/pocket.py** — IMPORTANT: For pocket mode, we're removing ice molecules to create the cavity, then placing water inside. The remaining ice at the cavity boundary forms the ice-water interface. Overlap resolution removes water molecules too close to this boundary ice.

**modes/piece.py** — Piece mode: ice crystal embedded in water box (box/cube only for v3.0).

Function signature: `assemble_piece(candidate: Candidate, config: InterfaceConfig) -> InterfaceStructure`

Implementation steps:
1. Get ice piece dimensions from candidate cell diagonal: `ice_dims = np.array([candidate.cell[0,0], candidate.cell[1,1], candidate.cell[2,2]])`.

2. **Validate**: box dimensions must be larger than ice piece. If `config.box_x <= ice_dims[0]` or `config.box_y <= ice_dims[1]` or `config.box_z <= ice_dims[2]`, raise `InterfaceGenerationError(f"Box must be larger than ice piece ({ice_dims[0]:.2f} x {ice_dims[1]:.2f} x {ice_dims[2]:.2f} nm)", mode="piece")`.

3. **Tile ice to fill piece region**: `tile_structure(candidate.positions, ice_dims, ice_dims)` — this tiles the candidate to fill exactly the piece region. Since the candidate cell IS the ice dimensions, this is effectively just the candidate itself (possibly with one tiling copy if there's slight mismatch). Actually, simpler: the candidate positions are already in a box of size ice_dims. We just need to center them.

4. **Center the ice piece in the box**: `offset = np.array([config.box_x, config.box_y, config.box_z]) / 2.0 - ice_dims / 2.0`. Shift all ice positions: `centered_ice_positions = tiled_ice_positions + offset`. The ice piece now spans from `offset` to `offset + ice_dims` in each dimension.

5. **Fill entire box with water**: `fill_region_with_water(np.array([config.box_x, config.box_y, config.box_z]), config.overlap_threshold)`. Returns (water_positions, water_atom_names, water_nmolecules).

6. **Detect overlaps**: `ice_o_positions = centered_ice_positions[::3]`, `water_o_positions = water_positions[::4]`, `detect_overlaps(ice_o_positions, water_o_positions, box_dims, config.overlap_threshold)`.

7. **Remove overlapping water molecules**: `remove_overlapping_molecules(water_positions, overlapping_mol_indices, atoms_per_molecule=4)`.

8. **Combine**: ice FIRST (centered), then water.

9. **Build InterfaceStructure** with cell, report, etc.

**interface_builder.py** — Main orchestrator with validation and mode routing.

Two public functions:

1. `validate_interface_config(config: InterfaceConfig, candidate: Candidate) -> None`:
   - Raises `InterfaceGenerationError` if config is invalid. Fail fast.
   - Common checks:
     - box_x, box_y, box_z must be positive (> 0)
   - Mode-specific checks:
     - **slab**: `abs(config.box_z - (2 * config.ice_thickness + config.water_thickness)) > 0.01` → error with message showing expected vs actual box_z
     - **pocket**: `config.pocket_diameter > min(config.box_x, config.box_y, config.box_z)` → error "Pocket diameter must be smaller than box dimensions"
     - **pocket**: `config.pocket_diameter < 1.0` → warning in report (not an error) that very small pockets may contain zero water molecules
     - **piece**: box dimensions must be larger than candidate cell diagonal
     - **piece**: `config.pocket_shape != "sphere"` for pocket mode → error "Ellipsoid pockets not yet supported"
   - Check candidate exists and has valid positions/cell
   - If any validation fails, raise `InterfaceGenerationError(message, mode=config.mode)`

2. `generate_interface(candidate: Candidate, config: InterfaceConfig) -> InterfaceStructure`:
   - Call `validate_interface_config(config, candidate)` first
   - Route to mode: `if config.mode == "slab": return assemble_slab(candidate, config)`, etc.
   - If unknown mode: raise `InterfaceGenerationError(f"Unknown interface mode: {config.mode}", mode=config.mode)`
   - Wrap any unexpected errors from mode functions: catch Exception, re-raise as InterfaceGenerationError
