# Phase 5: Output - Context

**Gathered:** 2026-03-27
**Status:** Ready for planning

<domain>
## Phase Boundary

Users receive 10 usable PDB files ready for molecular visualization or analysis. Optional phase diagram visualization shows user's T,P point. This phase handles the final output step after ranking is complete.

**In scope:**
- Writing 10 ranked candidates to PDB files
- Phase diagram generation (optional, on by default)
- Validation of output structures (spglib, overlap checks)

**Out of scope:**
- Creating/modifying ranking logic (Phase 4)
- Interactive visualization or GUI
- Analysis or post-processing of output files

</domain>

<decisions>
## Implementation Decisions

### File Organization

- **Output directory:** Sensible default location with `--output` flag for user-specified path
- User can override where files are written

### Phase Diagram

- **Formats:** Both PNG and SVG available, plus a text-based data file for external plotting tools (gnuplot or similar)
- **Content:** Full detailed diagram with grid lines, axes labels, phase boundaries visible, and user's T,P point marked
- **Activation:** Opt-out model — diagram generated by default, `--no-diagram` flag to disable

### OpenCode's Discretion

The following areas are left to OpenCode's judgment with documented rationale during implementation:

**PDB File Content:**
- Metadata level (minimal vs rich)
- Atom naming convention (TIP3P style, standard PDB, or GenIce default)
- CRYST1 record details (full crystal info, cell only, or GenIce provided)
- Chain/residue naming scheme

**File Organization Details:**
- Default output directory location (current dir, output/, or timestamped)
- File naming pattern for the 10 PDB files
- Collision handling when files exist (overwrite, auto-rename, or fail)
- Whether to include a summary file alongside PDBs

**Phase Diagram Details:**
- Image size/resolution
- Text-based data file format choice

**Validation Handling:**
- Behavior when spglib space group validation fails (warn/skip/fail)
- Behavior when atomic overlap is detected (warn/skip/fail)
- Whether to output validation results separately
- Success threshold (how many valid candidates needed)

</decisions>
