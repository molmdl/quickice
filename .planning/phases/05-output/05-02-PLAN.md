<feature>
  <name>PDB Writer with CRYST1</name>
  <files>quickice/output/pdb_writer.py, tests/test_output/test_pdb_writer.py</files>
  <behavior>
    Write Candidate structures to PDB format with CRYST1 record.
    
    Cases:
    1. write_pdb_with_cryst1(candidate, filepath) -> writes valid PDB file
    2. CRYST1 record contains: a, b, c, alpha, beta, gamma in Angstrom
    3. Coordinates converted from nm to Angstrom (multiply by 10.0)
    4. ATOM records follow PDB format specification
    5. write_ranked_candidates(ranking_result, output_dir, base_name) -> writes 10 PDB files with rank suffix
    
    Critical conversion: Candidate.positions and Candidate.cell are in nm, must multiply by 10.0 for Angstrom.
  </behavior>
  <implementation>
Implementation steps after tests pass:

1. **write_pdb_with_cryst1(candidate, filepath)**:
   - Convert cell from nm to Angstrom: cell_angstrom = cell * 10.0
   - Calculate cell parameters:
     - a = np.linalg.norm(cell[0])
     - b = np.linalg.norm(cell[1])
     - c = np.linalg.norm(cell[2])
     - alpha = angle between b and c (in degrees)
     - beta = angle between a and c (in degrees)
     - gamma = angle between a and b (in degrees)
   - Write CRYST1 record following PDB format (columns 1-80)
   - Write ATOM records for each atom with converted positions
   - Write END record
   - Atom naming: Use atom_names from Candidate (O, H format from GenIce)

2. **write_ranked_candidates(ranking_result, output_dir, base_name)**:
   - Create output directory if needed
   - Iterate over ranking_result.ranked_candidates[:10]
   - For each, write PDB file with filename: {base_name}_{rank:02d}.pdb
   - Return list of output file paths

3. **PDB Format Notes**:
   - CRYST1: "CRYST1" + a(9.3) + b(9.3) + c(9.3) + alpha(7.2) + beta(7.2) + gamma(7.2) + "  1"
   - ATOM: "HETATM" + serial(5) + atom_name(4) + resName(3) + chain(1) + resSeq(4) + x(8.3) + y(8.3) + z(8.3) + occ(6.2) + temp(6.2) + element(2)
   - Use HETATM for water molecules (standard for non-standard residues)
   - Element: right-justified in columns 77-78
  </implementation>
</feature>
