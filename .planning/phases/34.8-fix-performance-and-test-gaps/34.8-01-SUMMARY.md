## Decisions Made
- Used minimum image convention (`delta - cell_dims * np.round(delta / cell_dims)`) instead of direct Euclidean for boxsize distances — direct Euclidean gives wrong distances for atoms near PBC boundaries (e.g., atom at x=0.1 and x=1.4 in 1.5 nm box has direct distance 1.3 nm but minimum image 0.2 nm)
- Fixed supercell deduplication with set-based canonical pair tracking — old filter (`i < n_oxygen`, `i < j_original`) missed pairs where the first image block didn't contain the canonical (lower-indexed) atom for the pair
- Memory test threshold set to 50 MB (generous for KDTree overhead; a 27× supercell of 10k atoms would use > 100 MB)

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed minimum image distance computation in boxsize branch**
- **Found during:** Task 2 (regression test development)
- **Issue:** Plan specified `dist = np.linalg.norm(o_wrapped[j] - o_wrapped[i])` for boxsize pairs, but this computes direct Euclidean distance between wrapped coordinates. For atoms near opposite PBC boundaries, direct distance can be >> cutoff while minimum image distance is < cutoff (e.g., atoms at x=0.1 and x=1.4 in 1.5 nm box: direct=1.3 nm, minimum image=0.2 nm)
- **Fix:** Replaced with minimum image convention: `delta = delta - cell_dims * np.round(delta / cell_dims)`
- **Files modified:** quickice/ranking/scorer.py
- **Verification:** All 11 tests pass; boxsize distances all <= cutoff; boxsize matches supercell
- **Committed in:** c83f129 (Task 2 commit)

**2. [Rule 1 - Bug] Fixed supercell pair deduplication (pre-existing bug)**
- **Found during:** Task 2 (debugging boxsize vs supercell pair count mismatch)
- **Issue:** Old supercell filter `i < n_oxygen` + `i < j_original` missed cross-block PBC pairs. The first image block has offset (-1,-1,-1) for all dimensions. For atoms near opposite PBC boundaries (e.g., x=0.1 and x=1.4), the minimum image requires different x-offsets for each atom, but both are shifted together in the first block. Cross-block pairs (atom 85 in first block, atom 79 in central block) are found by query_pairs but rejected by the `i < j_original` filter because the canonical ordering flips across image blocks.
- **Fix:** Replaced with set-based canonical pair deduplication: `seen_pairs.add((min(i_orig, j_orig), max(i_orig, j_orig)))`. This correctly counts ALL unique atom pairs regardless of which image block they appear in.
- **Files modified:** quickice/ranking/scorer.py
- **Verification:** All 11 tests pass; boxsize and supercell produce identical pair counts and distances
- **Committed in:** c83f129 (Task 2 commit)
