<objective>
Add regression test verifying that [molecules] section names in exported TOP files match [moleculetype] names in their corresponding ITP files. A mismatch causes GROMACS to crash with a fatal error, so this test closes a critical compatibility gap.

Purpose: GROMACS requires that every molecule name listed in the [molecules] section of a .top file exactly matches a [moleculetype] name in a referenced .itp file. Currently no test verifies this consistency. The e2e-compute-export phase (Plan 06) already fixed a bug where moleculetype names were mismatched, but no regression test prevents this from recurring.

Output: New test file tests/test_gromacs_moleculetype_names.py with comprehensive name-matching assertions
</objective>

**Helper functions (module-level):**

1. `_parse_molecules_section(top_path: Path) -> list[str]`:
   - Read the TOP file content
   - Find the `[ molecules ]` section using regex: `\[\s*molecules\s*\](.*?)(?=\[\s*\w+\s*\]|\Z)` with DOTALL
   - Parse each non-comment, non-empty line: first field is the moleculetype name
   - Return list of moleculetype names

2. `_parse_moleculetype_name_from_itp(itp_path: Path) -> str`:
   - Use `parse_itp_file(itp_path)` from `quickice.structure_generation.itp_parser`
   - Return `itp_info.molecule_name`

**Test class structure:**

1. **TestIceCandidateTopNames** — Ice candidate export (uses `write_top_file`):
   - Create a synthetic Candidate (96 molecules, ice_ih, orthorhombic cell)
   - Write TOP file to tmp_path
   - Parse [molecules] section → should contain "SOL"
   - The ice TOP has inline [moleculetype] SOL, so verify "SOL" appears in [molecules] AND inline [moleculetype]

2. **TestInterfaceExportNames** — Interface export (uses `write_interface_top_file` + #include tip4p-ice.itp):
   - Create a synthetic InterfaceStructure using e2e_export_helpers patterns
   - Write TOP file + ITP files to tmp_path
   - Parse [molecules] section from TOP
   - Parse [moleculetype] name from tip4p-ice.itp (use `get_tip4p_itp_path()`)
   - Assert every name in [molecules] has a matching [moleculetype] in at least one referenced ITP

3. **TestMultiMoleculeExportNames** — Multi-molecule export (uses `write_multi_molecule_top_file`):
   - Build a molecule_index with multiple molecule types: ice, water, ch4 (hydrate guest)
   - Write TOP file with `write_multi_molecule_top_file` including #include for tip4p-ice.itp and ch4.itp
   - Parse [molecules] names from TOP
   - Parse [moleculetype] from each referenced ITP file
   - Assert each [molecules] name matches a [moleculetype] name from some ITP

4. **TestSoluteExportNames** — Solute export (uses `write_solute_top_file`):
   - Build a synthetic SoluteStructure (follow e2e_export_helpers patterns)
   - Write TOP + ITP files to tmp_path
   - Parse [molecules] and verify each name matches a [moleculetype] in referenced ITPs
   - Specifically verify CH4_LIQ or THF_LIQ (liquid solute names) match their ITP [moleculetype] names

5. **TestIonExportNames** — Ion export (uses `write_ion_top_file`):
   - Build a synthetic IonStructure
   - Write TOP + ITP files (ion.itp is generated, not copied)
   - Parse [molecules] and verify "NA" and "CL" match ion.itp [moleculetype] names
   - Verify ion.itp [moleculetype] name matches [molecules] entry

6. **TestCustomMoleculeExportNames** — Custom molecule export (uses `write_custom_molecule_top_file`):
   - Build a synthetic CustomMoleculeStructure with etoh.itp
   - Write TOP + ITP files
   - Parse [molecules] and verify custom molecule name matches etoh.itp [moleculetype] name
   - This is the exact scenario that caused GROMACS crashes before (moleculetype_name mismatch)

**Key testing pattern for each class:**
```python
# 1. Write export files to tmp_path
# 2. Parse [molecules] names from TOP
molecules_names = _parse_molecules_section(top_path)
# 3. For each #include directive, parse [moleculetype] name from ITP
itp_names = set()
for itp_path in referenced_itps:
    itp_names.add(_parse_moleculetype_name_from_itp(itp_path))
# 4. If inline [moleculetype] exists in TOP, add that name too
# 5. Assert: every name in molecules_names is in itp_names (or inline moleculetype)
for name in molecules_names:
    assert name in itp_names, f"[molecules] name '{name}' has no matching [moleculetype]"
```
