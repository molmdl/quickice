## Resolution

root_cause: The .top file in tmp/thf/ was generated by a code path (likely manual script or intermediate version) that computed water_nmolecules via subtraction using the old THF atom count of 12: water_nmolecules = (total_atoms - guest_nmolecules * 12) / 4 = (648 - 96) / 4 = 138. The correct calculation uses THF=13: (648 - 104) / 4 = 136. All current code paths are correct and use hydrate.water_count (136) directly, NOT subtraction. The MOLECULE_TYPE_INFO thf atoms value was corrected from 12→13 in commit f217c6d (May 22), but was never actually used in any computation (dead import). The stale .top file was the only artifact of the old computation.
fix: Regenerated tmp/thf files using current correct code (write_multi_molecule_gro_file + write_multi_molecule_top_file with registry). Added regression test test_top_sol_count_matches_water_count that asserts .top SOL count equals structure.water_count. Also fixed minor vtkActor type annotation bug in ion_viewer.py that was blocking test execution in headless environments.
verification: grompp succeeds on regenerated files with zero atom-count errors. New regression test passes. All 8 hydrate export tests pass.
