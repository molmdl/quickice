# Milestone v3.5: Interface Enhancements

**Status:** ✅ SHIPPED 2026-04-13
**Phases:** 22-27
**Total Plans:** 20

## Overview

This roadmap delivers four tightly-scoped features for QuickIce v3.5: Ice Ih IAPWS density (temperature-dependent, replacing hardcoded 0.9167 g/cm³), water density calculation from T/P via IAPWS (with correct spacing for interface generation), triclinic-to-orthogonal cell transformation (enabling Ice II, V, VI in interface generation), and CLI interface generation (`--interface` flag). The phase structure follows the research-recommended build order: Ice Ih Density → Water Density → Triclinic → CLI → Integration.

## Milestone Summary

**Key Decisions:**

- Decision: Use IAPWS library for density calculations (Rationale: Already in environment, scientifically accurate)
- Decision: Direct iapws._iapws._Ice usage (Rationale: Already implements IAPWS R10-06(2009))
- Decision: @lru_cache(maxsize=256) for density (Rationale: Performance for IAPWS iterative solver)
- Decision: Native triclinic handling instead of transformation (Rationale: Transformation creates gaps during tiling)
- Decision: Ice II blocked for interfaces (Rationale: Rhombohedral crystal structure incompatible)
- Decision: GROMACS atom number wrapping at 100000 (Rationale: Standard convention for large systems)

**Issues Resolved:**

- Fixed GROMACS atom number overflow for large systems (>99k atoms)
- Fixed crystal system documentation inaccuracies (Ice VI is tetragonal, not triclinic)
- Removed flawed triclinic-to-orthogonal transformer approach

**Deferred to v4.0:**

- Water density integration (Tab 1 display, Tab 2 interface spacing) - see Phase 23 UAT

**Technical Debt Incurred:**

- Screenshots deferred (user choice - will capture manually later)
